# 竞品调研 + 诚实对位(2026-06-08 · 报税/做账设计前对齐)

> Zihao 要求:别交作业,调研别人有什么,我们到底有没有真超越 Paypers、真更智能。本页 = 市场全景 + 诚实自评 + 怎么才是真更好。报税/做账/智能入口设计照此补全。

## 一、市场分三类(谁干啥)

| 类 | 代表 | 干什么 | 对位我们 |
|---|---|---|---|
| **采集整理→交会计** | **Paypers**(泰·给老板) · Dext/Hubdoc(全球·喂会计软件) | 拍票→抽取→分类→生成替代收据→归档(Drive/月)→**交给会计** | = 我们进项 capture 那块 |
| **全套账+出税表** | **FlowAccount · PEAK**(泰 SME) · Xero/QBO(全球) | 开票/费用/GL/AR/AP/库存 + 自动 VAT/WHT + **生成 PP30/PND 交税局** | = 我们做账 + 报税 |
| **事务所侧 AI 记账** | **AccRevo**(泰) | 老板拍票→事务所在平台上做账(5 类日记账)+ OCR + API | = 我们事务所栈 |

## 二、Paypers 到底是什么(官网核实)

给**freelancer/小店老板**(非会计)。采集:拍票/发票/税单/**银行转账单/手写现金单/电商截图(Shopee/Lazada/TikTok)**/PDF多语言 → 抽 类型/科目/金额/VAT/WHT → **自动生成替代收据** + 凭单(ใบสำคัญจ่าย·按日期金额/按公司月归类) → 月度/年度费用汇总 + 仪表盘 + **Google Sheets/Drive 归档** → 比对泰国工商库 → **整理好交给会计**。
**关键:Paypers 不自动做账、不报税(无 PP30/PND 申报)** —— 它是"老板的智能鞋盒",整理好交给会计,会计还得自己做账报税。

## 三、★ 我们真超越 Paypers 了吗?(诚实)

- **愿景上:超越了。** Paypers 止于"采集→整理→交会计";我们要 **采集→自动做账(真借贷)→自动报税(PP30/PND)闭环**,把会计那步吃掉。这是真·更进一步。
- **执行上:还没。** ① 我们进项 capture 现在**比成熟 Paypers 更糙**(Codex 抓的 WHT/供应商空/替代收据no-op/糊图/设置空一堆 bug)② 做账/报税**还没建**。**所以"超越"目前只在 PPT 上,执行补上才算数。**
- **Paypers 有、我们没有的(真实高频入口)**:电商截图(Shopee/Lazada/TikTok)· 银行转账单 · 手写现金单 · Google Drive 归档交会计。泰国小店天天用这些。

## 四、智能到没到位?(采集→分类→该去哪)

- **全球金标(Dext / Xero JAX)**:抽取(99.9%)+ 自动分类 + **从纠正中学习(2-3月越来越准)** + **用户可见规则** + **高置信才自动·低置信留人工复核** + 多通道(App/邮件/**银行feed**)。
- **好消息**:我们做账的「置信门槛→自动过账·否则待审」+ review_learned,**正好跟 Xero JAX 同路数**(国际金标)。方向对。
- **差的**:① 智能分流(上传/LINE→该去哪)还有冲突没贯通(吃饭票进了识别记录)② **没有银行流水 feed/对账**(Xero/QBO/PEAK 的核心)③ 没电商平台对接。

## 五、做账会不会"做出来全是错的"?(Zihao 担忧)

行业让账不出错靠:**银行对账** + 规则 + 干净科目表 + **高置信自动·其余人工复核** + 借贷必平 + 月度对账校验。
我们设计已有:置信门槛/待审、借贷必平断言、review_learned、审计留痕 —— **设计是稳的**。
**但缺一块致命的:银行对账 loop**(账与银行流水对上,是"账对不对"的最终校验)。不补 → 账可能跟银行对不上 → 老板不敢信。**做账图纸要加银行对账。**

## 六、怎么才是真更简单/快/智能(设计指引)

1. **报税**:别只"生成 PP30/PND"(FlowAccount/PEAK 都有=标配)。真差异化 = **不用编表(从做账自动来)+ 异常前置提醒(报前抓错)+ e-Tax 直接提交(不只出表)+ 一站看所有税种与截止**。
2. **做账**:补**银行对账 loop**;把置信/学习做成**用户看得见的"规则"**(像 Xero bank rules·"这供应商以后都记XX")。
3. **智能入口**:先把**统一分流 + 待归类**做扎实(这才是超越 Paypers"整理"的根基);考虑加**电商(Shopee/Lazada/TikTok)+ 银行 feed** 采集口(泰国小店刚需)。
4. **执行**:进项那堆 bug 先收尾 —— 否则 capture 比 Paypers 还糙,谈不上超越。

## 七、结论一句话
**我们的牌是"闭环"(做账+报税)——这是 Paypers/采集工具到不了的地方,也是真正的护城河。但前提是:① 进项 capture 做到不输 Paypers(收尾 bug + 补电商/银行/Drive 入口)② 做账真准(补银行对账+人工复核)③ 报税真省(e-Tax 直报+不用编表)。三者都做到,才是真超越、真更智能 —— 不是交作业。**
