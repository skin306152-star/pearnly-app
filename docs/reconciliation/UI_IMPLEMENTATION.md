# Pearnly 对账中心 UI 精确落地任务

> ⚠️ **先读 `docs/reconciliation/00_CURRENT_STATE_AND_CORRECTIONS.md`(权威现状与修正)。**
> 与本文件冲突时以 00 号文件为准:模板用后端对齐的那套(UI 稿里的模板标签作废)、
> 收入对账输入是 GL+税表VAT、下载模板走后端按语言生成、后端端点清单见 00 号。
> 后端已就绪(解析器认全 4 语 + 发票 Excel 路 + 模板下载端点),本任务**只做前端**。

## 任务性质

把交互原型 **按现有产品技术栈落地到真实项目中**。目标:最终页面在布局、层级、尺寸、间距、
文案、状态变化和按钮交互上与参考 HTML 尽可能一致,同时保留并正确接入现有业务逻辑。

参考原型:`design-reference/pearnly_reconciliation_redesign_v2.html`

---

## 一、最高优先级约束

1. **禁止重新设计**:不改页面结构/配色/文案;不擅加渐变/插画/大图标/卡片/动画;不删区域;
   不合并按钮;不用占位交互冒充真实交互。
2. **参考 HTML 是视觉和交互事实来源**:先完整读其 HTML/CSS/JS,提取结构/状态/颜色/圆角/
   边框/阴影/间距/字号/交互;原型里的原生 JS 仅用于理解行为,落地改写为项目框架与状态管理。
3. **不得破坏现有产品**:先读对账中心实现/路由/权限/国际化/上传/解析/任务状态/下载/导出/
   结果列表;优先复用现有接口与组件;不改无关模块;不全局样式重置;不借机大重构;不改后端契约。
4. **禁止面向用户出现** `OCR`/`AI`/`人工智能`/`大模型`;统一用:`标准模板读取`/`普通表格读取`/
   `文件内容读取`/`自动选择读取方式`/`需要确认的内容`/`清晰度`/`读取结果`。内部代码/已有 API/
   字段名不要求改名。

## 三、页面状态机(不是静态图)

- **A 初始**:标题/三标签/模板引导横幅/四步流程/两上传区/余额预检空态/清空/禁用的开始对账。
  此时禁示:匹配率/已匹配/待处理差异/未匹配/虚假 0 结果卡。
- **B 只上传一个文件**:显已传文件信息+真实处理状态;开始对账仍禁用。
- **C 两份已传但有待确认**:显文件名/格式/大小/读取方式/数据行数/需确认项数/确认入口;
  全部确认前开始对账禁用。
- **D 通过导入前检查**:显真实余额预检;开始对账启用。
- **E 对账中**:隐藏上传工作区,显"正在执行对账"+当前处理说明+真实进度(无精确进度则用
  不虚构百分比的进行中态)+已处理数量(仅接口提供时)。**禁定时器伪造进度。**
- **F 完成**:才显四指标(匹配率/已匹配/待处理差异/未匹配)+ 优先处理分类 + 结果明细 +
  导出 + 开始处理差异。**指标可点击筛选明细。**
- **G 失败/中断**:明确错误原因 + 重试入口 + 返回文件检查入口 + 不丢已传文件与已完成确认。

后端状态来源:`GET /api/recon/jobs/{job_id}` 的 `status`/`progress`;`needs_mapping` 带 `mapping`
载荷(弹字段确认),`needs_review` 带 `review` 载荷(弹内容确认)。

## 四、读取方式与引导

1. **标准模板读取**(推荐):文案 `标准模板读取`/`推荐`/`按字段直接读取`/`需要确认 0 项`(仅真实
   校验通过才显)。识别**不只靠文件名**:用模板签名(后端模板 sheet 名 `Pearnly-<TYPE>`)/固定表头
   /后端识别结果。现有列头匹配机制可兜底,代码注释与报告中说明。
2. **普通表格读取**(用户自己的 Excel/CSV):文案 `普通表格读取`/`确认字段对应关系`/
   `记住该企业 / 银行的字段设置`/`下次可直接使用`。必须接现有字段映射保存逻辑(见 §十.字段映射),
   不得只弹一个不保存的窗口。
3. **文件内容读取**(PDF/图片/Word):文案 `文件内容读取`/`确认需要关注的内容`/`系统只列出需要
   确认的内容`/`读取结果`/`清晰度`。只展示需确认字段,不要求复核整份文件。

## 十四、按钮交互逐条规范(20 项 · 权威清单)

### 1. 顶部三个对账类型
切换 **银行对账** 同步更新:左=银行账单、右=总账 GL、模板中心内容、下载模板内容、字段映射规则、
对账结果与任务状态。
切换 **销项税报告核查**:销项税报告 + 销售发票明细 + 对应两个真实模板。
切换 **收入对账**:总账 GL + 税表VAT + 对应两个真实模板(注:以后端为准,非"收款流水/销售收入")。
**最易漏**:只换标题没换真实接口/模板/数据;或切换后还保留上一类型已上传的文件。
**正确规则**:切换时不能把上一类型的文件和结果带过来;存在未提交数据时按产品规范提醒用户。

### 2. 导入说明
打开说明弹窗解释三种方式(标准模板读取 / 普通表格读取 / 文件内容读取);只负责说明,不改上传状态。
关闭方式:右上角关闭 / "知道了" / Esc /(可选)点遮罩。

### 3. 模板中心
右侧抽屉(不跳页);内容随当前对账类型变化;每个模板含名称/用途/下载/查看示例/真实版本号
(有才显);关闭抽屉不清空已传文件。

### 4. 下载本页模板
一次下载当前对账需要的两个模板。需考虑浏览器拦截连续下载:推荐后端打包 ZIP,或第一次点击给出含
两个明确下载项的弹窗。不照搬原型连续触发两个浏览器下载的写法。

### 5. 暂不提示
关闭顶部标准模板推荐横幅:本次会话隐藏或存为用户偏好;模板中心/设置里仍可重新看到。不得永久消失无恢复。

### 6. 上传卡片"选择文件"
按左右位调真实文件选择器;依次:格式检查→大小检查→上传→判断读取方式→读取内容→导入前检查→
返回文件状态。必须有:上传中/失败/重试/防重复上传/文件过大提示/不支持格式提示。

### 7. 拖拽文件
与"选择文件"走**同一个真实处理函数**(不能点击有校验、拖拽绕过)。处理:拖入多个/文件夹/不支持/
拖到错误上传位。

### 8. 下载标准模板(左右上传区各一)
左右下载的模板不同(银行对账:左=银行账单模板、右=GL 模板),不能两个都下同一文件。必须用项目真实
.xlsx 模板(走后端端点),不临时生成 CSV。

### 9. 上传后"移除文件"
同时清除:上传文件 / 读取结果 / 字段映射临时状态 / 用户改了未提交的字段 / 余额预检 / 该文件相关
错误 / 开始对账可用状态。例:两份都就绪后移除任一份 → 开始对账立即重新禁用。

### 10. 普通 Excel"查看字段对应"
弹字段对应:原文件列名 / Pearnly 字段 / 示例数据 / 忽略该列 / 必填缺失提醒 / 重复映射提醒 / 金额
与日期格式预览 / "记住该企业／银行的字段设置"。勾选后**真实保存**(按 企业+银行或来源+表头特征+
当前对账类型 组合识别),下次同结构直接复用。**最易做假**:只关弹窗没存库。

### 11. 字段弹窗"确认并导入"
验证必填映射完整 → 保存映射 → 重读表格 → 更新数据行数 → 更新余额预检 → 文件状态改"已通过检查" →
判断是否启用开始对账。点击期间禁用防重复;失败不直接关弹窗。

### 12. PDF/图片"确认需要关注的内容"
只打开系统认为不清晰/有歧义的字段(金额/日期/参考编号/余额);每项尽量给:当前读取值 / 原文件所在
页 / 局部预览 / 清晰度 / 修改输入框。不要求复核整份文件。

### 13. "确认 N 项并导入"
保存修正值 → 更新该文件数据 → 重跑必要校验 → 更新需确认数量 → 更新余额预检 → 满足条件启用开始
对账。**不能只把"N 项"改成"0 项"却没真更新后台数据。**

### 14. 预览文件
用项目现有预览能力:PDF 内置预览器 / 图片放大 / Excel 预览前若干行列 / Word 可读内容或安全下载。
无预览能力时**不做点击只弹"预览成功"的假按钮**。

### 15. 清空
清本次临时状态(左右文件/读取结果/字段确认/余额预检/错误/对账结果/当前筛选)。已完成对账或已大量
手改时二次确认:`确定清空本次对账内容吗?未保存的修改将丢失。` 不删用户保存过的长期字段映射。

### 16. 开始对账
全满足才可点:左右文件都在 / 上传完成 / 读取完成 / 必填完整 / 必须确认项已处理 / 余额预检达业务允许
条件 / 无正在执行任务。点击:防重复 → 调真实对账接口 → 进对账中态 → 轮询/实时状态 → 成功显结果 /
失败可重试。**禁定时器假进度**;无精确进度显 `正在执行对账,请稍候……`,不编 67% 或"已处理 864 条"。

### 17. 完成后四个指标(仅完成后显)
匹配率→显全部;已匹配→筛已匹配;待处理差异→筛(金额/日期有差异、需人工确认、规则无法自动决定);
未匹配→只显未找到对应记录。被选指标有选中态,表格分页与总数同步变化。

### 18. "开始处理 N 项差异"
进真实差异处理流程(页面下方切差异列表 / 打开差异抽屉 / 跳现有差异页带任务 ID)。支持:单项确认/
批量确认/忽略/建立匹配/修改数据/返回结果页/更新待处理数量与匹配率。不能只弹"已进入处理流程"。

### 19. 导出结果
调真实导出:导出全部还是当前筛选 / XLSX/CSV/PDF / 生成中 / 失败 / 文件名 / 下载权限。建议默认导出
完整结果,文件中区分已匹配/待处理差异/未匹配。

### 20. 原型底部四个演示按钮(标准模板/普通Excel/PDF图片/直接查看结果)
仅预览用,正式项目**全部删除**。同样不得照搬:假文件名 / 固定 1,286 行 / 固定金额 / 固定匹配率 97% /
定时器模拟进度 / 点击只弹 Toast / 临时生成 CSV 模板。

### 九个最易被做假/漏掉的点(逐一自查)
1. 按钮能点但调的是假提示。2. 字段映射没真存。3. 模板下载错误或仍生成 CSV。4. 对账进度是定时器
模拟。5. 切换对账中心后残留旧数据。6. 移除文件后没重置余额和开始按钮。7. KPI 只是卡片没真实筛选。
8. 异步按钮没失败态和防重复。9. 移动端/Esc 关闭/焦点/键盘被忽略。
