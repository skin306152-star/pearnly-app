# 竞品 UI 参考 · PEAK 权限/用户管理(2026-06-10 截图调研)

> 来源:Zihao 实测 PEAK 账号(子号皮 5KS828AB1)截图。**仅作设计参考,合法的产品体验调研,非源码抄袭。** 对位我们 /console。

## PEAK 权限体系动线(6 屏)

### 屏1 · 设置左树 + 用户列表
- 左侧设置长目录(分组可折叠):升级/扩展套餐(包裹/历史/信用卡)· 组织环境 · **用户设置(用户/角色)** · 文档设置 · 会计政策制定 · 连接应用设置 · 注册会计师事务所。
- 右侧用户列表:`当前用户 1/10`(席位计量)· tab[用户 | 受邀用户]· 列(姓名/用户权限/使用权)· 使用权 = 一排彩色模块图标(账户/工资/PEAK板/资产/税 五个产品)· owner 带皇冠 · 顶部「+ 添加 PEAK 支持」「+ 添加新用户」· 底部分页(每页 10)。

### 屏2 · 添加新用户(邀请)
- 输邮箱 → **选角色卡**(8/10 张:行政/管理会计/财务会计/财务会计和工资/管理所有文档/文件草稿/管理收入文件/管理报价,每卡带描述)+「+ 添加角色」入口。
- 角色卡下方实时显示该角色的:使用权(模块图标)+ **额外权限勾选**(升级套餐/添加用户/管理角色,带说明气泡)。
- 再下方「角色详情」可展开 5 个产品分组(账户/工资/PEAK板/资产/税)看细权限。

### 屏3 · 确认邀请弹窗
- 发邀请前二次核对:组织 / 邮箱 / 角色 / 额外权限(×订阅 ×添加用户 ×管理角色)· 取消 / 确认。

### 屏4 · 角色列表(默认角色 8/10)
- 列(角色/用户数量)· 每行带彩色图标 + 「默认角色」标签 · 行政(1人)其余 0 人 · 顶部「+ 添加角色」。

### 屏5 · 添加自定义角色 · 步骤1 名称及描述
- 三步进度条:① 名称及描述 ② 设置用户权限 ③ 确认。
- 角色名(泰/英双语·各 40 字)+ 描述(100 字)+ 可编辑角色图标。

### 屏6 · 自定义角色 · 步骤2 权限矩阵(★核心)
- 左侧产品列表(账户/工资/PEAK板/资产/税)可逐个点。
- 右侧**逐单据 × 5 动作矩阵**:行=单据类型(引述/存款收据/发票/收据/销售税发票/贷项凭证…)× 列=[看法/创造/编辑/批准/删除] 全复选框 + 表头「显示更多/更少」。
- 底部 取消 / 下一个。

## 对位我们 /console(诚实)

| 能力 | PEAK | 我们 /console | 判断 |
|---|---|---|---|
| 用户列表 | ✅ 带席位 1/10、使用权图标列 | ✅ 成员列表(角色/作用域/状态) | 对齐;**缺席位计量显示**(我们后端有 seats 概念没在 UI 露) |
| 邀请流 | ✅ 邮箱+角色卡+确认弹窗 | ✅ 邀请弹窗(email+LINE+角色卡+作用域) | **我们更强**:多 LINE 通道、按套账作用域;PEAK 无作用域 |
| 预设角色 | ✅ 8 个(行政/管理会计/财务会计…按职能切) | ✅ 6 个(老板/管理员/会计/录入员/只读/收银员) | 路数相同;PEAK 角色更细分(工资/文档/报价单独成角色) |
| **自定义角色** | ✅ 三步向导+逐单据矩阵(高配) | ❌ P1 不做(图纸留位 roles.tenant_id) | **这是 P2,不是缺** — 对标结论:Xero/Stripe 干脆没有 |
| 权限矩阵粒度 | ✅ 单据×动作复选 | 后端有 62 码,前端用角色卡代替勾选 | 我们故意:动作数预算(选角色1下 vs 勾几十框) |
| 额外权限叠加 | ✅ 角色基础上勾「添加用户/管理角色」 | ❌ 单角色制 | P2(Stripe 式叠加) |
| 用户详情页 | ✅ 信息+角色+使用权+角色详情展开 | ✅ 行内展开 | 对齐 |

## 值得吸收的(排进 backlog,不急)

1. **席位计量上 UI**(`当前用户 1/10`):后端 seats_max 已有,前端 console 成员列表头加一行 —— 小、有"专业感"、且是套餐变现的钩子(满了提示升级)。低成本高感知。
2. **角色卡带"使用权图标 + 额外权限"预览**:我们角色卡现在是纯文字说明,可加一行该角色能碰哪些模块的图标(复用业态模块色)—— 让"选角色"更直观。
3. **自定义角色三步向导 + 单据×动作矩阵** = 我们 P2 自定义角色的**现成形态参考**:施工 P2 时照这个三步(名称→矩阵→确认),矩阵的行用我们的资源、列用我们的八动词。**图纸已留位,到时照搬此布局。**
4. **"默认角色"标签 + 用户数量列**:角色列表显示每个角色多少人在用 —— 治理可见性,顺手能加。

## 不吸收的

- PEAK 把工资/文档/报价拆成独立角色 = 它的模块划分,我们按 6 角色×作用域已覆盖,不跟。
- 逐单据勾几十个框的默认交互 = 老 ERP 路数,我们 P1 角色卡更省事,坚持。
