# 套账入口与新建向导 · 设计稿(2026-06-10)

> Zihao 拍板:登录后全屏选套账(照合伙人屏2形态·双端同步)+ 新建套账独立向导(零业务功能)+ **个人模式整体处决** + 手机端模块卡主页。
> 施工窗口:排做账前端(窗口B)收尾后 —— 同属 src/home 领地,不并行。
> **状态:草案 · 拍板点见末尾。**

## 一、动线(0/1/N 分流 · 动作数预算)

```
登录成功
 ├─ 0 个套账 → 新建向导(全屏·零业务功能·建好才放进系统)
 ├─ 1 个套账 → 直接进系统(不加一步·顶栏可切可新建)
 └─ ≥2 个套账 → 全屏选套账页 → 进系统
手机端进系统后首页 = 模块卡主页(合伙人屏3形态);桌面端保持侧栏,不强加卡片层
```

- 选套账页:公司列表(名称/总分公司/最后活动时间)+ 搜索 + 「+ 新建公司」入口;**只此一处有新建**(原"只选不建"拍板更新:选择页可新建,业务内仍不可)。
- 记住上次:登录后默认预选上次使用的套账(≥2 时高亮置顶,回车直进)。

## 二、新建套账向导(大厂级 · 三步收拢)

**步1 · 税号一键带出(杀手锏)**:输 13 位税号 → 调现有 RD 核验接口 → 自动回填公司泰文名/注册地址/总分公司+分支号 → 用户确认可改。查不到/个体户无税号 → 降级手填,四态齐。
**步2 · 公司资料**:泰/英文名 · 地址 · 电话/邮箱 · logo(票面用·可跳过)· **VAT 登记状态(已登记/未登记 · 新字段·决定能否开税票/要不要报 PP30·做账报税消费它)**。
**步3 · 账务设定(全部带默认值·可一路下一步)**:财年起始月(默认1月)· 本位币(锁 THB·展示位)· 单据号前缀(默认自动)。

- 完成 → 落 workspace_clients(+新列 vat_registered)→ 进系统。
- 全程无侧栏无业务导航,只有向导壳 + 退出登录。
- 经营类型不在向导里问(业态是租户级 onboarding 的事,见接缝)。

## 三、个人模式处决令(Zihao 2026-06-10:头脑发热产物,清干净不留屎山)

- 语义:`pearnly_work_mode=personal`(无套账做业务)整体删除 —— **套账是硬边界,业务必须有套账**(与 workspace-isolation 地基一致)。
- 施工法:grep 全仓 `work_mode|personal` 拉清单逐处删(localStorage 键/ws-modal 分支/销项免套账分支/E2E 里设 personal 的前置)·删干净后 ws-modal 半路弹窗整体退役(被本设计的全屏入口页替代)。
- 存量兜底:老用户若真有"无套账已做业务"的数据,迁移脚本给建默认套账归挂(先查数,可能为零)。
- E2E 同步改:所有用 personal 消弹窗的 spec 改走"预选套账"前置。

## 四、与业态/套餐的接缝(本设计不施工,只立规)

- 业态 onboarding **保留**,职责降级 = 只管预置(模块默认开关/科目表/POS 形态),时点不变(注册首进)。
- "开通"语义将来归套餐权益层(blueprint 07·波3):付费自动解锁 + 超管代开通双通道;模块 toggle 过渡期保留为偏好层。
- 本设计不动 tenant_modules 机制。

## 五、四态+边界

| 场景 | 行为 |
|---|---|
| RD 接口超时/查无 | 降级手填,提示人话,不阻塞 |
| 税号重复(本租户已有同税号套账) | 422 人话拦 |
| 向导中途退出 | 不留半套账(最后一步才落库) |
| 0 套账用户想逛系统 | 没有这条路 —— 向导是唯一出口(Zihao 拍板) |
| 受邀成员登录(权限体系) | 看到的列表 = scope 内套账;0 个被分配 = 空态"请联系管理员",**无新建入口**(新建 = owner/admin 权限码) |

## 六、六类完整性

① 操作面=选套账页/向导 ② 配置后台=无(套账资料编辑沿用现有客户管理) ③ 接入流=本体 ④ 四态=上表 ⑤ 角色=新建挂 settings.workspace.manage·成员只见分配内 ⑥ 联动=VAT 登记状态喂做账报税·RD 带出复用销项核验·替代 ws-modal。
