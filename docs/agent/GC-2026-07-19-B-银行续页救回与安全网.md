## 2. 改动点

### 2.1 分类判据扩一条腿（`sort.py`）

1. 新增窄谓词（放 `_is_statement_title` 旁）：
   ```python
   def _category_is_bank(fields: dict) -> bool:
       """OCR 管线自己的粗分类字段(category)明说 bank。单独不作准,与标题白名单双锁用。"""
       return (str(fields.get("category") or "").strip().lower()) == "bank"
   ```

### 2.4 三道确定性完整性闸（`bank_sales_suggest.py` + `stmt_totals.py` + `api.py`）

数据源全部来自既有 `item_bank_parsed` 事件回放（payload 含 item_id + rows[date/deposit/withdrawal/balance]），**零模型成本、不重读文件**。

1. **签名扩参**：`coverage_check(events)` 与 `suggest(events)`（`bank_sales_suggest.py:272/:358`）加关键字参数 `period: str | None = None`；调用点 `services/workorder/api.py:232-233`（order_detail 投影）从 `store.get_work_order(...)["period"]` 传入。佛历转公历 helper（纯函数）：`"2569-05" → (2026, 5)`，非法格式返 None 判据不启用。
2. **闸① 日期覆盖**：账期月的首日/末日 vs 全部行日期 min/max。`max_date < 月末` → 缺尾天列表（如 `["2026-05-29","2026-05-30","2026-05-31"]`）；`min_date > 月初+1` → 缺头天。命中 → coverage `reliable=False`，降级原因新常量 `DEGRADE_DATE_GAP`，缺哪几天进 message。**注意**：只缺 1 天不降级（周休/节假日单日无动账常见），缺 ≥2 连续天才降级；单日缺口进 message 作提示不拦。
3. **闸② 页数自报**：现状 `_statement_block`（:227-257）已消费 totals；扩：解析页数 = item_bank_parsed 中行数 ≥5 的 item 数（把 IMG_2586 这种单行片段排除在"页"外），与 `total_pages` 比对的缺页数进 message（现状只判 incomplete 不点名数字的话补点名）。
4. **闸③ 跨文件余额交接**：每 item 按事件内行序取段 `{first_date, last_date, first_balance, last_balance, first_signed}`（行数 <3 或余额全 0 的段跳过）；段按 `(first_date, first_balance)` 排序；相邻段判 `abs(next.first_balance - (prev.last_balance + next.first_signed)) <= 1.0`。断链 → `reliable=False`，原因 `DEGRADE_CHAIN_BREAK`，message 带断点两侧日期。
5. 降级 message 全部走现有 degrade 卡通道（前端 `ai-bank-sales-render.js` degradeCardHtml 已渲染 message，本单后端拼好人话文案即可；文案照 `stmt_totals.incomplete_message` 现有语种处理方式）。三个新降级原因常量与既有 `DEGRADE_INCOMPLETE`/`DEGRADE_COVERAGE` 并列同风格。
