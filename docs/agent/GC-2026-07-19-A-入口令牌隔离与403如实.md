## 1. 目标

同一浏览器同开 /ai 与 /pos 互不串权；401/403/校验失败/断网四种失败各说各话；AI 门户有零副作用会话探针。

## 2. 改动点

### 2.1 AI 门户令牌独立槽（根治覆盖）

1. 新 key：`mrpilot_token_ai`。在 `static/ai/ai-api.js` 顶部（apiFactory 之前）加单一事实源 helper 并挂 `AI.token`：
   ```js
   var TOKEN_KEY = 'mrpilot_token_ai';
   AI.token = {
       get: function () { try { return root.localStorage.getItem(TOKEN_KEY); } catch (_) { return null; } },
       set: function (v) { try { root.localStorage.setItem(TOKEN_KEY, v); } catch (_) {} },
       clear: function () { try { root.localStorage.removeItem(TOKEN_KEY); } catch (_) {} }
   };
   ```
   `apiFactory` 的 `getToken` 默认改走 `AI.token.get`。
2. 替换 AI 侧全部散落读写（上面清单逐处）：`ai-gate.js:242` 改 `AI.token.set(data.access_token)`；`ai.js`/`ai-pkg.js`/`ai-review.js`/`ai-review-inbox.js` 的 get/remove 全走 `AI.token`。替换后 `grep -rn "mrpilot_token'" static/ai/` 只允许 ai-api.js 一处定义命中（`mrpilot_lang`、`QUEUE_KEY_PREFIX` 等非 token key 不动）。
3. **不做旧 key 迁移采认**：新 key 为空即视为未登录 → 走 AI 登录卡。上线后 /ai 用户重登一次，属预期（邀请制用户少），REPORT 写明。`ai-gate.js:245` 的 `pearnly_entry='ai'` 提示写入保留不动（preboot 分流仍用）。
4. 构建：跑 `npm run build`，同 commit 带 `static/dist/ai.js`；`ai.html` 的 `ai.js?v=66` bump 到 67。
