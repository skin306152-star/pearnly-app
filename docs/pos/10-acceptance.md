## 3. 核心流程 E2E(端到端真跑)
1. **开班**:PIN 登录→开班输备用金→进收银主屏。
2. **卖一单(在线)**:扫码+点选→改量/折扣→收款现金(验找零)→出小票→断言:小票落库、库存按 base_unit 扣、连号正确。
3. **PromptPay**:收款选扫码→断言二维码 payload/CRC 正确(复用 promptpay 测试)。
4. **拆零**:卖"粒"(多单位)→断言库存扣 base_unit = qty×factor。
5. **批次 FEFO**:有两批不同效期→卖出→断言扣的是近效期批。
6. **升级税票**:小票→填买方→开全式→断言 sales_documents 生成、连号/冻结快照、`full_invoice_id` 回填、不重复计 VAT。
7. **退货**:扫原小票→选行→退款→断言负额小票 + 原批库存回补 + 红冲。
8. **挂单/取单**:挂起→取回→结算正常。
9. **交班**:点钞→提交→断言 expected/counted/diff + 日结汇总正确。

## 4. 离线(治断网丢单/双扣/号乱 · 08)
1. 飞行模式→连开 5 单(扫码+收款+出小票)→断言每单本地出票、UI"待同步 5"。
2. **关页面重开**→断言 outbox 5 单仍在(未丢)。
3. 恢复网络→断言自动同步 5 单全成、库存只扣一次、终端连号无冲突。
4. **双终端超卖**:两台离线都卖最后一件→联网→断言不报错落库 + 库存转负 + 老板端告警。
5. **重复补传**:人为重发同 `client_uuid`→断言 `deduped:true`、不双扣。

## 5. 权限/隔离/模块
