# Codex · Pearnly 销项开票模块 · 全量 UI 实测

## 目标
对线上 pearnly.com 的销项开票模块做**真浏览器全量实测**:每个屏、每个按钮都点一遍,**证明功能真的能用**(不是看着像);视觉也逐项核对草稿。遇到问题**记录+定位,然后继续下一项**,最后出报告。

## 账号与红线
- 用 **E2E 测试账号**(凭据按运行注入·不写进任何文件·不进 git)。
- **绝不碰真付费客户 mrerp 的数据/余额。**
- 开票会占用测试账号自己的发票连号(不可改·正常);可用"作废"清理状态,别狂开几百张。
- 邮件发送测到**你能查看的测试邮箱**,别发陌生地址;LINE 只生成链接、不真发给人。
- 不触发真实充值/扣费。

## 测试基准
- 功能契约:`docs/sales-module/docs/17` §3「按钮→接口」矩阵(每个按钮该调哪个 `METHOD /path`)。
- 视觉基准:桌面 `Pearnly开票UI预览/app.html`、`index.html`(布局/颜色/文案/位置/四态)。

## 每个按钮都验这 7 条
1. 点了**有反应**(不是死按钮)。
2. **调对接口**(Network 里就是矩阵那个 path+方法)。
3. 返回**正常**(2xx;预期的业务错误码也算对)。
4. **0 console error**。
5. **视觉对草稿**(位置/主色 #2563EB/文案)。
6. **四态**(加载/空/错/正常)该有的有。
7. **4 语**(中泰英日)切换不乱版、不露原文 key。
- 外加:手机视口(iPhone/Pixel)不溢出。

## 逐屏逐钮清单(按这个跑·一个不漏)
**A · 发票工作台**:开票按钮 / 筛选(全部·草稿·已开·已作废)/ 搜索 / 点行进详情 / 汇总卡 / 右上齿轮(开票设置)。
**B · 开票向导(5 步)**:① 单据类型切换 + 常见场景一键 ② 卖方下拉、买方类型切换(公司/个人/外国/匿名·字段随变)+ **税号验真·带出**(公司输 13 位→带出名称/地址/分店)③ 商品**菜单点选** + 加自定义行 + 数量增减 + 折扣 + 「存入商品库」勾选 ④ 收款状态/方式/日期 + 佛历切换 ⑤ 输出设置(纸张 A4/A5/热敏·语言·省纸两联)+ 合规检查清单 + 发票预览 + **开出** → 成功面板(下载/打印/发送)。
**C · 发票详情**:下载 PDF / 打印 / 发送(邮件·LINE 两档)/ 作废 / 红冲 / 补开 / 报价转换 / 复制再开 / PromptPay 付款码(仅未收款)。
**D · 商品管理**:列表 / 新增 / 编辑 / 删除 / Excel 导入 / 搜索。
**E · 账套·开票资料**:选账套 / 改资料 / **6 套模板切换 + 主色 + logo/印章/签名/页脚** / 保存 / 实时预览。
**F · 客户管理**:列表 / 新增·编辑(**买方类型切换 + 分店 + PromptPay**)/ 删除。
**G · 开票设置弹窗(右上齿轮)**:连号前缀/重置/起始号 / 审批模式 / 价内外默认 / WHT 档 / 默认语言·纸张·省纸 / 保存。
**H · 全程抽查**:语言切换(中泰英日)+ 手机视口。

## 出报告(每个问题一条)
格式:`【屏】【按钮/元素】【操作步骤】【现象】【期望】【疑似定位:接口 path 或 文件】【console 错误原文】【截图】【严重度 高/中/低】`
- 严重度:**高**=功能不能用/报错/死按钮;**中**=视觉明显错/四态缺/调错接口;**低**=文案/像素小瑕。
- 通过的项标 **PASS**。
- 报告顶部汇总:共测 X 屏 Y 钮 · PASS Z · 问题 W(高_ 中_ 低_)。
- **遇问题记录+定位后继续下一项,绝不中途停。**

跑完把报告整理好贴回窗口。
