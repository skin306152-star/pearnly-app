# 09 · LINE 记账对话 · 测试用例文案(给测试窗口照着跑)

> 用途:这是**可直接执行的测试脚本**——一条条"发什么 / 期望什么 / 怎么判 PASS"。
> 方法论看 `08-fuzz-conversation-test-construction.md`;本页是落地用例。
> 机器跑批用 `tests/fuzz/line_fuzz_corpus.py`(`expand()` 出 ~19,800 变体);本页是其中
> 121 条 **curated 硬用例**的人话版,供真机/真大脑(Tier B)逐条核对。
>
> 判定 = **端状态**(不是看回复文字漂不漂亮):①有没有建采购单 ②建的话金额多少 ③回复语言
> ④回复有没有禁词。每条标了它必须满足的不变量(INV-*,定义见 08 §2 + 语料文件顶部)。

---

## 0. 跑前必读(红线)

- **隔离租户 + 隔离 LINE 用户**,绝不碰真客户账套。每条跑完清掉测试单。
- Tier A(确定性·brain OFF·秒级·跑全量):`python -m unittest tests.fuzz.test_line_fuzz_deterministic`(窗口建)。
- Tier B(真大脑·brain ON·隔离租户·prod venv):`scripts/_line_fuzz_run.py`,真机/真 Gemini 逐条发,抓端状态。
- 判定只用确定性(`amount_extract.money_numbers` / DB 端状态),**不要用 LLM 当裁判**。
- 报告按不变量聚合 + FAIL 明细(模板见 §4)。存疑(疑似标注错而非产品 bug)的列出来回 PM,别静默跳过。

图例:**不记**=不该建任何单 · **记**=该建单且金额接地 · **问价**=不建单·回追问多少钱 · **拒绝**=合规拒绝不照做。

## 3. 扰动批次(每条再 ×10 · 鲁棒性)

对上面任一"记"或"不记"用例,套这 10 个扰动,端状态应**不变**(`corpus.PERTURBATIONS` 已实现):
去空格 / 叠字 / 句尾 `555` / 句尾 `ค่ะ`·`ครับ` / emoji / 大写拉丁 / 泰数字 / 全角数字 / 原样。
> 重点回归:`trailing_555`(句尾 555 不得顶替真额)、`thai_digits`/`fullwidth_digits`(泰/全角数字)。

机器跑全量:`from tests.fuzz.line_fuzz_corpus import expand; rows = expand()` → ~19,800 条(含上面 168 curated + 模板×槽位×扰动)。

## 4. 报告模板(回报 PM 用这个格式)

```
样本: N 条  ·  brain: ON/OFF  ·  PASS x (xx%)  ·  FAIL y
按不变量(从严重到轻):
  REFUSE_FRAUD     FAIL ?   ← 合规·最先看
  NO_RECORD        FAIL ?   ← 伤账
  AMOUNT_GROUNDED  FAIL ?
  INCOME_NOT_EXPENSE FAIL ?
  HONEST_TAX / NO_MODEL_LEAK / FOLLOW_LANG / FX_HANDLED / CONFIRM_DESTRUCTIVE ...
按类热区:V类型号 ?/?  · R类量词 ?/?  · BB类欺诈 ?/? ...
FAIL 明细(每条):# / 发什么 / 期望 / 实际端状态(建几单·金额·回复摘要) / origin
存疑(疑似标注错非产品bug):列出来回 PM,别静默改期望
```

## 5. 跑哪批先(优先级)

1. **BB 欺诈**(5 条 + 中泰变体)— 合规红线,最高。
2. **B/R/S/V 噪声&型号&量词&税率数字**(伤账主类,已修一部分,验回归 + 新型号/量词/税率)。
3. **E/T 收入退款找零**(方向错=伤账)。
4. **G 语言跟随 + H/AA 诚实 + J 注入**(Tier B 真大脑才测得出回复内容)。
5. 其余 + 全量扰动批跑回归。
