# 全平台业态套餐 · 规划(POS 之后做)

> Zihao 2026-06-07 拍板:POS+库存 上线后接此项目。本页先把概念/范围/架构/工作量/决策规划好,
> 到时按此出完整图纸(同 docs/pos 的 12 份套路)再施工。当前=规划,未开建。

## 一句话

**新用户注册时选一次"你做什么生意",系统自动把整个 Pearnly 该开的模块配好、不该出现的藏起来。** 把"选业态→一键配模块"从只管 POS(屏8)放大到管整个平台。对齐 `docs/PRODUCT_VISION_MODULAR.md` 模块化原则 #2(业态套餐代替逐个拨开关)+ #3(导航随模块显隐)。

## 为什么(非技术用户能用起来的关键)

- 服务型公司注册完界面干净,不被库存/POS 吓到;零售店一进来就有收银/库存,不用找。
- 没开的模块标"可开启 →",想用一点就开(发现性,不是永久藏死)。
- 否则现状=所有人看同一套全菜单,有用没用都堆着。

## 业态 → 模块预设(与愿景一致 · 可微调)

| 业态 | 默认开的模块 |
|---|---|
| 会计事务所(代记账) | 开票 · 进项 · 对账 · 知识库 · 多账套 |
| 零售 / 小卖部 | 开票 · 库存 · POS · 会员 |
| 餐厅 | 开票 · 库存(原料) · POS(桌台/厨房单) |
| 服务型公司(咨询/律师/设计) | 开票 · 进项 |
| 批发贸易 | 开票 · 库存 · 应收 · 进项 |
| 药房 | 开票 · 库存(批号效期/拆零) · POS · 会员 |

> **底座永远开**(不在套餐开关里):账号/权限 · 商品(主数据) · 客户 · AI 录入 · 多语言 · 工作台。
> **可开关模块**:开票(销项) · 进项/费用 · 库存 · POS · 对账 · 应收 · 知识库。

## 现状 / 缺口(决定工作量)

- ✅ 开关表 `tenant_modules` 已建(POS 项目地基)+ `module-nav`(前端按开关显隐)已有雏形。
- ✅ POS / 库存 已认这个开关(开了才显)。
- ❌ **最大工作量**:开票/进项/对账/应收/知识库等**现有主程序模块还没接 `tenant_modules`**——目前对所有人"默认全开",导航是硬编码。要让它们也按开关显隐(后端 gating + 前端导航数据驱动)。
- ❌ **注册引导**:注册/首次进入时的"你做什么生意"选择页没有(POS 的屏8 只配收银那摊)。

## 架构思路(复用 POS 已建的)

1. **复用 `tenant_modules`**:业态预设 = 一组 module_key 的开/关写入。
2. **导航数据驱动**:现有侧边栏硬编码 → 改成按 `tenant_modules` + role 渲染(每个一级项/分组标 module_key,开了才显;没开标"可开启 →")。复用 POS 的 `module-nav` 范式。
3. **复用 POS 开通页交互**:屏8"选业态→右侧自动开模块预览"放大成全平台注册引导页。
4. **每个现有模块加门控**:后端路由 `assert_module_enabled(key)`(POS 信封层已有);前端路由/页面按开关 gate。
5. **设置页"业务/模块"区**:随时重配/开关(关=隐藏不删数据,原则 #5)。

## 关键决策(规划阶段先定方向,出图纸时细化)

1. **老租户不破坏**:已注册租户**默认保持当前全开**(不强制选业态、不突然藏功能);只有**新注册**走业态选择。老用户可在设置里主动选业态/精简。← 高风险点,必须这样兜底。
2. **粒度**:底座常开不可关(账号/商品/客户/AI/工作台);其余 7 个模块可开关。
3. **发现性**:没开的模块标"可开启 →"(owner 可见),不是彻底消失(原则 #3)。
4. **业态可改**:选错/业务变了能改;关模块=隐藏不删数据,随时重开。
5. **权限叠加**:module 开关 ∧ role 权限——员工/收银员在已开模块内再按 role 裁;两层独立。
6. **计费按模块**:愿景原则 #6,**本期不接**(现按额度计费),留 config 余量。
7. **与导航 IA 铁律协调**:现有导航跟 `pearnly_nav_prototype_final.html`;数据驱动化要保持其结构,只做"显隐",不重排既有 IA。

## 工作量拆解(POs · 出图纸时细化)

- **P1 模块门控全覆盖(最重)**:7 个可开关模块全部接 `tenant_modules`(后端 assert + 前端导航/路由数据驱动)+ 业态预设定义。
- **P2 注册业态引导页**:注册/首次进入选业态 → 写预设(UI 先出桌面预览稿 → Zihao 拍板 → 再做)。
- **P3 设置页"业务/模块"管理 + "可开启 →" 发现入口**。
- **P4 老租户迁移策略 + 业态切换**(默认全开兜底 + 主动精简路径)。

## 风险

- **动现有导航/模块显隐 = 高风险**:藏错模块会让(尤其付费)用户找不到功能 → 老租户默认不变 + 充分真账号 E2E(每业态登录看导航对不对)+ 灰度。
- 现有模块没有统一门控点 → 要逐个接,散落改动多 → 出图纸时先做"现有模块清单 + 各自门控位置"勘测。
