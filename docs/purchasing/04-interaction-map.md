## 三、按钮 → 去向(每个动作终点钉死)

**屏1 主屏**
| 控件 | 去向 |
|---|---|
| 「拍进项票」 | 调相机/选图 → `POST /intake` → 落**屏10**(预填) |
| 「LINE 记费用」 | 打开**屏3**(说明 + 跳 LINE / 或内嵌一句话输入) |
| 列表行(draft) | 落**屏10**(编辑草稿) |
| 列表行(posted) | 落**屏6**(详情) |
| chip 全部/进货/费用/未付 | 前端筛(`GET /docs?kind=&unpaid=`) |
| 搜索框 | `GET /docs?q=`(供应商/单号) |

**屏10 录入/编辑**
| 控件 | 去向 |
|---|---|
| 「切换公司」 | 套账切换器(整 App 换上下文) |
| 供应商「切换 ▾」 | 唤起**屏9**(供应商选择器) |
| 行「未配 SKU·匹配」 | 唤起**屏8**(商品匹配) |
| 「+生成替代收据」 | `POST /substitute-receipt` → 附件区出 PDF 卡 |
| 「+附付款凭证/扣缴凭证」 | `POST /attachments` |
| 行 商品/服务 切换 | 服务→自动带出 WHT 率(取 settings.default_wht_service_rate)→ 重算合计 |
| 行 VAT/数量/单价/折扣 改 | 即时重算合计(税前→VAT→含税→WHT→实付) |
| 「+加一行」 | 追加空明细行 |
| 「存草稿」 | `POST /docs`(status=draft)→ 回主屏 |
| 「确认入账」 | `POST /docs`(若新)→ `POST /post` → 回主屏(列表刷新·新行 posted) |

**屏6 详情**
| 控件 | 去向 |
|---|---|
| 「记付款」 | 唤起**屏7** |
| 「作废」 | `POST /void`(已入库回冲)→ 详情转灰"已作废" |
| 「编辑」(仅 draft) | 落**屏10** |
| 票图「放大看」 | 看大图 |

**屏7 记付款** → `POST /pay` → 关弹窗 + 详情/主屏付款态刷新 + 未付 KPI 减。
**屏8 商品匹配** → `POST /lines/{id}/match-product` → 行变"已配 SKU"。
**屏9 供应商选择器** → 选定/建新 → 回填屏10 供应商段。
**屏4 供应商管理** → CRUD(`/suppliers`)· 「加供应商」弹窗 · 行「编辑」弹窗。
**屏5 采购设置** → `GET/PUT /settings` · `GET/POST/PATCH /categories`(两级)· 「保存」。

## 五、状态流(数据怎么在屏间流)

```
拍照/手录 ──POST /intake|空──► 屏10(draft)
   存草稿 ──POST /docs(draft)──► 主屏出"草稿"行(灰)
   确认入账 ──POST /docs+/post──► 主屏出 posted 行 + KPI(进货/费用/可抵税/未付)更新
                                      │
   posted 行点开 ──► 屏6 详情
       记付款 ──POST /pay──► payment_status 变 + 未付 KPI 减(部分付款=partial)
       作废 ──POST /void──► status=void(灰) + 已入库回冲库存
LINE 一句话 ──POST /expense──► 直接 posted(费用) + 回卡 + 生成替代收据;可从主屏点开补全(屏10)
```
> 单一事实源:列表/详情/KPI 都从 `purchase_docs` 派生,不另存状态镜像(同 erp_push_logs 铁律精神)。

## 六、设置 ↔ 行为(联动钉死 · 否则"设置了没反应")

| purchase_settings | 影响 |
|---|---|
| auto_stock_in | 屏10「同时入库」开关默认值 + `/post` 是否写库存 |
| dedupe_block | `/intake`/`/docs` 命中 dedupe_key 时拦(409)还是仅提示 |
| default_due_days | 屏10 账期/到期默认 |
| default_vat_rate | 行 VAT 率默认 |
| default_wht_service_rate | 行 item_type=service 时 WHT 默认率 |
| pay_needs_approval | 屏7 付款前是否要审批 |
| base_currency | 多币种折算本位币 |

## 七、跨模块 seam(在哪一下触发 · 别只画屏漏联动)

## 九、双端适配(铁律 · 桌面+手机都做,非只手机)

- 主屏:桌面=表格 / 手机=卡片(已定稿)。
- 录入屏10:桌面=票图凭据(左)+ 信息/明细/合计(右)两栏 / 手机=同一套竖向堆叠。
- 弹窗(7/8/9):桌面=居中 .modal / 手机=底部抽屉。
- 详情6:桌面两栏 / 手机单列。
- **同一份设计两端各自最优,不是手机版缩水,也不是只做手机。**
