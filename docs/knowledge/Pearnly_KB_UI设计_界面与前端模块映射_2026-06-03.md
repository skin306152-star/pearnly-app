# Pearnly 知识库 · UI 设计 · 界面与前端模块映射

生成日期：2026-06-03
作者：Opus 4.8（设计，非执行）
配套：《…多窗口执行计划…》第 6 节（P1/P4/P5 前端）+ 第 2 节目录
对象：做前端的窗口照此实现；与既有 Pearnly `src/home/*` 模块化、四语 i18n、账套主体上下文一致。

---

## 0. 三条设计基调（不许破）

1. **不是聊天孤岛，是长在现有工作流里的几块**。用户感受是"Pearnly 变聪明了"，不是"我在用一个 AI 网站"。最高频入口是 **OCR 结果页的风险检查 panel**，不是问答页。
2. **一切按账套主体（workspace client）隔离**。任何知识库界面顶部都明确"现在在给哪家公司做账"；未选账套不允许上传客户私有文档、不允许问答。
3. **AI 的话必须可核对**。每个 RAG 结论带可点开的出处；无出处则显式"资料不足，无法判断"，不编。死规则结论标出"命中第几条规则"。

四语（th/en/zh/ja）、空/加载/错误/无权限态、移动端可用——全部一等公民，不是补丁。

---

## 0.5 必须贴合现有 Pearnly 设计语言（不许另起一套）

知识库 UI **不引入新色板/新字体/新组件风格**，一律复用主项目既有设计语言。实现窗口动手前，去主项目取真实 token 与组件，别凭空造。

- **设计 token**：颜色/圆角/间距/阴影全用 `home-*.css` 既有 CSS 变量（`var(--brand)`、`var(--ink)` 等），**禁硬编码 hex**（参见主项目"按钮发黑"教训：高特异性 + 变量收口）。
- **磨砂玻璃质感**：悬浮件、抽屉、卡片沿用着陆页那套玻璃/磨砂风（`backdrop-filter: blur` + 半透明 + 柔阴影 + 圆角），与玻璃 logo 标牌/登录卡同语言。
- **品牌呼应**：悬浮问答按钮可用品牌猫吉祥物图标（`static/brand/`），延续着陆页猫主题，让"问 AI"有 Pearnly 性格。
- **复用既有组件**：按钮用 `.btn-primary` 等既有类；弹窗/抽屉复用 `home.html` 既有 modal/drawer root，不新造一套 modal 体系。
- **四语 + RTL 无关但泰文优先**：泰文文案偏长，留足排版空间。
- 迁移进主项目时：CSS 进既有 `home-NN.css` 模块体系，改前端必 bump `?v=`（铁律）。

## 1. 五块主界面 + 模块映射总览

| 界面 | 出现在哪 | 主模块 | 阶段 |
|---|---|---|---|
| ① 风险检查 panel | OCR 结果页内嵌 | `knowledge-risk.js` + `knowledge-sources.js` | P5 |
| ②a 悬浮问答（全局快捷入口·premium） | 任意页面右下角悬浮件 | `knowledge-ask.js` + `knowledge-sources.js` | P4 |
| ②b 客户知识中心（文档库/问答历史/规则三 tab） | 新页面「客户知识」 | `page-knowledge.js` 装配 + `knowledge-documents.js` / `knowledge-ask.js` / `knowledge-rules.js` / `knowledge-sources.js` | P1/P4/P3 |
| ③ ERP 推送前检查弹窗 | 推送动作前 guard | 复用 `knowledge-risk.js` 取检查结果 | P5 |
| ④ 管理视角（拦错数/采纳率/用量成本） | admin 后台 | admin SPA 内新卡片（独立，不在 home bundle） | P5+ |

所有数据请求统一走 `knowledge-api.js`（fetch 封装：带鉴权、错误归一、tenant/workspace 自动附带）。

## 2. 界面① OCR 结果页 · 风险检查 panel（`knowledge-risk.js`）

最核心、最高频。OCR 识别完成后，在结果区右侧（移动端则在字段下方）渲染。

```
┌─────────────────────────────────────────────────────────┐
│  发票识别结果                          🟡 风险检查:中    │
│  ┌──────────────────┐  ┌────────────────────────────────┐│
│  │ 卖方  ABC 有限公司 │  │ ⚠️ 中  VAT 对不上               ││
│  │ 税号  0105558...   │  │    应为 70.00,票上 80.00       ││
│  │ 发票号 INV-2024..  │  │    规则 R-VAT-01               ││
│  │ 金额  1,000.00     │  │ ─────────────────────────────  ││
│  │ VAT   80.00        │  │ 🔴 高  这家供应商需人工复核      ││
│  │ 合计  1,080.00     │  │    原因:客户要求大额必审        ││
│  │ ...                │  │    规则 R-SUP-02               ││
│  │                    │  │ ─────────────────────────────  ││
│  │                    │  │ 🟡 中  超出合同金额上限          ││
│  │                    │  │    上限 30,000,本票 35,000     ││
│  │                    │  │    📄 服务合同2024.pdf 第3页 ›  ││ ← 点开来源抽屉
│  │                    │  │                                 ││
│  │                    │  │  每条:  [✓ 采纳]   [✕ 忽略]    ││
│  └──────────────────┘  └────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

- **总评徽章**：红/黄/绿 = `invoice_risk_checks.risk_level`。绿色时折叠成一行「✓ 无异常」，不占视线。
- **每条 finding**：图标(严重度) + 一句话 + 证据数字 + 来源（死规则→规则号；RAG→可点的文档来源，点击调 `knowledge-sources.js` 开抽屉）。
- **采纳/忽略**：调 `knowledge-api.js` 回写 `accepted_count`/`dismissed_count`（采纳率数据）。
- **高风险**：显眼置顶 + 触发界面③的推送拦截。
- 状态态：检查进行中显「检查中…」骨架；失败显「检查未完成，可重试」；无规则/无知识库则只跑全局死规则，正常出结果。

## 2.5 界面②a 悬浮问答组件（`knowledge-ask.js`，P4 · 主入口）

**这是问答的主入口**（专门的问答 tab 退为"历史回看"）。它比独立页面更贴"随处可用、不做孤岛"的基调：会计在看发票、对账、做账任意页面，右下角点一下就能问，不跳页。

**收起态**：右下角一颗悬浮按钮（FAB），品牌猫图标，磨砂玻璃 + 柔阴影，低调不挡操作。
```
                                              ┌──────┐
                                              │ 🐱 问 │ ← FAB·磨砂玻璃
                                              └──────┘
```

**展开态**：点开向上展开成一张磨砂玻璃悬浮卡（非全屏遮罩，桌面端约 380×520，移动端贴底全宽），与着陆页玻璃卡同语言：
```
        ┌────────────────────────────────┐
        │ 客户知识助手   [账套:XX 公司 ▾] ✕│ ← 顶部固定当前账套
        ├────────────────────────────────┤
        │ 你:差旅费报销上限多少?          │
        │                                 │
        │ AI:单笔上限 5,000 泰铢,超出需  │
        │    总经理审批。                 │
        │    📄 采购政策.docx 第2页 ›     │ ← 点开来源抽屉
        │                                 │
        │ (无来源时)                      │
        │ AI:资料不足,无法判断。          │
        ├────────────────────────────────┤
        │ [输入问题…………………]   [↑发送]  │ ← 底部输入框
        └────────────────────────────────┘
```

- **上下文感知**：自动带当前账套主体；若当前在 OCR 结果页，可一键"问关于这张票"，预填上下文。
- **必带 citation**：答案出处可点，经 `knowledge-sources.js` 开来源抽屉；无来源固定话术拒答。
- **交互**：`Esc` 收起，`Enter` 发送，收起态保留会话直到切账套。
- **设计语言**：磨砂玻璃 + `var(--brand)` 主色 + 既有圆角/阴影 token；FAB 用品牌猫，呼应着陆页。**不新造 modal 体系**，复用 home 既有 drawer/overlay。
- **未选账套 / 无权限**：FAB 仍在，点开提示"请先选择账套主体"或"无权限访问该客户知识"。
- 与界面②b 共用同一套答案/引用渲染逻辑（同模块 `knowledge-ask.js`），悬浮件与 tab 只是两个挂载点。

## 3. 界面②b 客户知识中心（`page-knowledge.js` 装配）

侧栏新增入口「客户知识 / Client Knowledge」。顶部账套主体选择器（复用既有 workspace switcher）。三个 tab。

### 2a. 文档库 tab（`knowledge-documents.js`，P1）
```
客户知识  ▸  [账套:XX 贸易有限公司 ▾]
 ┌ 文档库 ┊ 问答 ┊ 规则 ┐
 │  📄 文档库                              [+ 上传文档] │
 │  ┌──────────────────────────────────────────────┐  │
 │  │ 服务合同2024.pdf      ● 已就绪    [删除]       │  │
 │  │ 采购政策.docx         ◌ 解析中 45%             │  │
 │  │ 税务登记.pdf          ● 已就绪    [删除]       │  │
 │  │ 旧报价.xlsx           ✕ 失败 [重试] [查看原因]  │  │
 │  └──────────────────────────────────────────────┘  │
 └──────────────────────────────────────────────────────┘
```
- 上传：拖拽/选择，支持 PDF/docx/xlsx/csv/图片；上传即建 `knowledge_documents` + `ingest_jobs`。
- 状态徽章对齐后端状态机：uploaded→extracting→chunking→embedding→ready / failed。轮询或 SSE 刷新进度。
- 删除：标 deleted + 停止检索（按保留策略决定是否物删）。
- 权限：员工权限不足时隐藏上传/删除，仅查看已授权客户。
- 未选账套：整页置灰 + 提示「请先选择账套主体」。

### 2b. 问答 tab（`knowledge-ask.js` + `knowledge-sources.js`，P4）
```
 │  你:这家客户的差旅费报销上限是多少?                  │
 │                                                        │
 │  AI:根据《采购政策.docx》,差旅费单笔上限 5,000 泰铢,│
 │      超出需总经理审批。                                │
 │      📄 采购政策.docx 第2页  ›   ← 点开看原文          │
 │  ────────────────────────────────────────────────     │
 │  （无来源时）                                          │
 │  AI:资料不足,无法判断。建议上传相关合同或政策文档。   │
 │                                                        │
 │  [输入问题…………………………………]  [发送]              │
```
- 答案**必带 citation 卡片**，点击经 `knowledge-sources.js` 开右侧来源抽屉，高亮命中片段。
- 无足够来源 → 固定话术「资料不足，无法判断」，禁止给确定结论。
- 历史问答可回看（`ai_answers`）。
- 不做花哨的"思考动画"，要快、要朴素、要像工具。

### 2c. 规则 tab（`knowledge-rules.js`，P3）
`client_rules` 表的可视化管理。分组展示：
```
 │  供应商白名单           [开关:开]   [+ 添加供应商]    │
 │   • ABC 有限公司 0105558...           [移除]          │
 │   • XYZ 商行     0107561...           [移除]          │
 │  金额上限                              [+ 添加规则]    │
 │   • 服务类  单票 ≤ 30,000 THB         [编辑][停用]    │
 │  强制人工复核的供应商                  [+ 添加]       │
 │   • DEF 公司 — 原因:大额必审          [移除]          │
 │  会计期间限制           [开关:关]                     │
```
- 每条规则可见来源标记（人工/从合同提取/学来）；learned 规则醒目标"待确认"，需点"启用"。
- 开关 = `feature_toggle`；增删改即时生效，影响下一次风险检查。

## 4. 界面③ ERP 推送前检查弹窗（复用 `knowledge-risk.js`）
```
 ┌─ 推送前检查 ──────────────────────┐
 │  这张票有 1 个高风险未处理:        │
 │  🔴 供应商需人工复核(R-SUP-02)    │
 │  [仍要推送]        [先去处理]       │
 └────────────────────────────────────┘
```
- 仅当存在未处理 `high` finding 时弹出；无高风险则不打断，正常推送。
- "仍要推送"需记审计（谁绕过了哪条）。**RAG/规则不写 `erp_push_logs`**，真推送仍走既有 ERP 流程。

## 5. 界面④ 管理视角（admin SPA，P5+）
后台加卡片：本月 AI 拦错数、整体采纳率、各账套文档量与就绪率、AI 用量与成本。给老板看"值不值"。独立于 home bundle，按主项目 admin 约定做。

## 6. 前端模块清单（落 `src/home/`，各 <500 行）

| 模块 | 职责 | 依赖 |
|---|---|---|
| `page-knowledge.js` | 知识中心页装配、tab 路由、账套上下文守卫 | 下面各模块 + api |
| `knowledge-api.js` | 所有 fetch 封装（鉴权头、错误归一、自动附 tenant/workspace） | — |
| `knowledge-documents.js` | 文档列表/上传/删除/状态轮询 | api |
| `knowledge-ask.js` | 问答输入/答案渲染/citation 卡/历史；**同时驱动悬浮问答件(FAB+展开卡)与知识中心问答 tab,两个挂载点共用答案逻辑** | api, sources |
| `knowledge-sources.js` | 来源抽屉/原文片段高亮预览（问答 + 风险共用） | api |
| `knowledge-rules.js` | 客户规则增删改 + 开关 | api |
| `knowledge-risk.js` | OCR 结果页风险 panel + 推送前弹窗取数 | api, sources |

- **不塞大入口**：`main.js` 仅 side-effect import 装配；新模块各管一摊，禁写成一个巨文件。
- **window.* 桥克制**：模块间通过 import/事件通信，别再加全局桥（主项目教训）。
- i18n：文案全进 `static/i18n-data.js` 四语，模块只引 key。
- 缓存：迁移进主项目时改了前端必 bump `?v=`（铁律）。

## 7. 通用态（每个界面都要做，别只做 happy path）

- **加载**：骨架屏，别白屏。
- **空**：文档库空→引导上传；问答空→给 2-3 个示例问题；规则空→说明默认行为。
- **错误**：可重试 + 人话原因，别抛栈。
- **无权限**：隐藏写操作，只读；越权访问后端兜底拒绝。
- **未选账套**：置灰 + 明确提示，禁止客户私有操作。
- **移动端**：风险 panel 改字段下方堆叠；抽屉改全屏；保证关键路径可用。
