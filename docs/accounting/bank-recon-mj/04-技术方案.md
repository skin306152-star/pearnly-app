# 银行对账 + 手工凭证 · 第四阶段:技术方案(2026-06-11)

> 原则:复用最大化(01 审计的复用清单是硬约束),新增面收敛为 3 张表 + 1 个 router + 2 个前端屏。
> 禁止:重写解析/评分/过账/学习任何一套既有逻辑;禁止动旧 bank_recon session 表。

## 1. 状态流转

```
银行流水行: [导入] → unmatched ──确认/组合/新建──→ matched(matched_voucher_id)
                        │ ←──────── 撤销(同时 unpost 新建型凭证) ──┘
                        └─排除→ excluded(可还原→unmatched)
凭证(复用既有机): pending_review → posted/auto_posted → void
  手工: method=manual · 草稿=pending_review · 过账=posted
  对账新建: source_type=bank_line · 确认即 posted(用户确认=审)
```

## 2. 数据库变更(ensure_* 启动自愈,照 accounting 先例,无 alembic)

新增 3 表(`services/accounting/schema.py` 扩展,全带 tenant_id + workspace_client_id 双隔离 + 索引):

```
acct_bank_accounts: id uuid PK / tenant_id / workspace_client_id / bank_code(KBANK..) /
  account_label / account_last4 / coa_account_id(银行存款科目 FK) / is_active / created_at
  uq(tenant, ws, bank_code, last4)

acct_bank_lines: id uuid PK / tenant_id / workspace_client_id / bank_account_id FK /
  line_date / amount numeric(14,2) / direction(in|out) / description / bank_ref /
  import_batch_id / source_file_sha256 / status(unmatched|matched|excluded) /
  matched_voucher_id NULL / matched_at / matched_by / created_at
  uq(bank_account_id, line_date, amount, description, bank_ref)  -- 防同批重复行
  ix(tenant, ws, status), ix(tenant, ws, line_date)

acct_voucher_templates: id uuid PK / tenant_id / workspace_client_id / name /
  lines jsonb([{account_id, dr_cr, memo}]) / use_count / created_by / created_at
  uq(tenant, ws, name)
```

不动表:journal_vouchers(source_type 加合法值 'bank_line' 仅是应用层常量)、review_learned、旧 bank_recon 三表(零接触)。

## 3. API(新 `routes/accounting_bank_routes.py`,独立 router 铁律#17;手工凭证扩展进既有 accounting_routes)

| 端点 | 方法/权限 | 说明 |
|---|---|---|
| /api/accounting/bank/accounts | GET view · POST settings.manage | 银行账户登记(选 COA 银行科目) |
| /api/accounting/bank/import | POST review | multipart;sha256 查重(409 duplicate_file);调 services/recon 解析管道 → 落 lines;返 {parsed, inserted, skipped, opening/closing_balance} |
| /api/accounting/bank/summary | GET view | 三余额(银行=最近导入 closing_balance;账面=books 试算表银行科目;差额=unmatched 净额)+ 进度 |
| /api/accounting/bank/lines | GET view | ?account&period&status 分页 |
| /api/accounting/bank/lines/{id}/candidates | GET review | 建议:复用 bank_recon_scoring 容差评分,候选源=①银行科目凭证行 ②未收款 sales_documents ③未付款 purchase_docs ④POS 按日聚合 ⑤review_learned 命中 |
| /api/accounting/bank/lines/{id}/match | POST review | body 三选一:{voucher_id} / {doc_ids[]}(组合,后端复核合计=金额,否则 422)/ {new_tx:{kind:income|expense|transfer, account_id, contact?, memo}};生成/关联凭证(source_type=bank_line)→ 行置 matched;**单事务** |
| /api/accounting/bank/lines/{id}/unmatch | POST approve | 行回 unmatched;新建型凭证走 unpost;operation_logs 记 bank.unmatch |
| /api/accounting/bank/lines/{id}/exclude · /restore | POST review | 排除/还原 |
| /api/accounting/bank/harvest | POST review | body {line_ids[]};逐行 SAVEPOINT(照 POS sync 先例,单行失败不污染整批);返 per-line 结果;operation_logs 记 bank.harvest |
| /api/accounting/vouchers/manual(已有) | POST review | **扩展**:body 加 draft:bool(true→status=pending_review)、template_id? |
| /api/accounting/voucher-templates | GET view · POST review · DELETE coa.manage | 模板 CRUD(POST 可 from_voucher_id 一键存) |

错误码全走 acct.* 信封(acctErrMsg 已有管道):acct.bank.duplicate_file / acct.bank.line_already_matched / acct.bank.combo_mismatch / acct.period_closed(复用)/ acct.unbalanced(复用)。

## 5. 权限校验 / 并发幂等 / 日志

- 每端点 require_perm(逐路由码,authz 第 8 道闸自动覆盖);套账参数 fail-closed(withWs)。
- 幂等:import 靠 sha256+行级 uq;match 靠行状态 CAS(`UPDATE ... WHERE status='unmatched'` 影响行数=0 → 409)+ uq_jv_source 防凭证重复;harvest 逐行 SAVEPOINT。
- 并发:两人同对一行 → 后到者 409 人话;凭证生成在行锁事务内。
- 日志:operation_logs 记 bank.import/harvest/unmatch(操作人/套账/数量);凭证自带 created_by/reviewed_by。

## 6. 性能 / 安全

- 流水池查询走 ix(tenant,ws,status);candidates 限近 90 天单据+LIMIT;不做全表扫。
- 解析在 asyncio.to_thread(铁律#10,Playwright/重 CPU 同理);50MB 上限沿用 nginx client_max_body_size。
- 安全:文件解析复用既有管道(坏文件→422 不 500,Pillow/pdf 先例);SQL 全参数化;附件存储走 pdf_storage 鉴权读。
- 对首屏零影响(全部懒加载路由)。
