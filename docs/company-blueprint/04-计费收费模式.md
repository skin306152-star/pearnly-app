# 平台总纲 · 04 计费收费模式

> 骨架#4:订阅优先,ARR/NDR 当仪表盘。金蝶用 11 年把云收入做到 82.5%、订阅 35.6 亿、ARR 40.9 亿 —— 经常性收入是 SaaS 估值与经营的根。⚠️裁剪:不学"忍五年亏损",初创要尽快正现金流。

## 现状(摸底确认)

纯 credits 按量后付单档:注册即用 → OCR/问答按次扣 `tenant_credits.balance_thb` → 转账截图充值(SlipOK 自动验证/人工审核)。无订阅、无套餐分层、JWT plan claim 只剩 'credits'/'admin' 两值。

诚实评价:管道是 🟢 资产(泰国转账习惯+自动验证已跑通),模式是 🔴 缺口 —— 纯按量没有经常性收入,没有 ARR/NDR 可言,客户用得少≠留存好,投资叙事和经营仪表盘都立不起来。

## 目标模式:订阅底座 + 按量增值(混合制)

```
订阅档(按 01 章三层) = 功能边界 + 内含用量包(OCR n 张/月)
      +
credits(现管道原样保留) = 超量 OCR / 智能问答 / 增值动作,按量扣
```

为什么混合:纯订阅会浪费已跑通的 credits 管道,且 OCR/AI 成本是真实变动成本,纯包断送毛利;纯按量没有 ARR。混合 = FlowAccount 的"档位含 AutoKey 额度,高档无限"同款思路 + 我们已有的扣费体系,改造量最小。

## 档位草案(锚定本地价格带 · 数字待拍板)

| 档 | 层 | 月价锚(฿) | 边界(示例) |
|---|---|---|---|
| Free | S | 0 | POS 单机 + 简单账 + 少量 OCR 额度(获客口,学 FlowAccount/PEAK 都有免费档) |
| Starter | S | 199–299 | + 自动 VAT + LINE 采集 + 标准 OCR 包 |
| Business | M | 549–899 | + 进项/库存/应收 + **做账报税** + **多用户权限**(viewer 不占席?拍板)+ 审批 |
| Business+ | M | 999–1,499 | + **e-Tax 直报** + **API/电商对接** + 大 OCR 包(对标 PEAK PRO Plus 把 e-Tax/API 收在高档) |
| Firm | F | 按客户数阶梯 | 事务所工作台 + 批量 + 伙伴分成资格(02 章);"按代账客户数"计价 = 账无忧/Canopy 路数 |

- **权限体系是 M 层付费墙的核心商品**:FlowAccount 把"unlimited users with user roles"放 Pro 档 —— `docs/permissions/` 那套不只是安全件,是直接卖钱的功能件。
- 年付折扣对齐本地习惯(FlowAccount 83 折;PEAK 干脆只卖年付)。

## 存量平迁(零破坏)

现 credits 用户:余额原样保留(继续当增值钱包);给 90 天双轨窗口选档,默认落到与其用量匹配的档;老用户给创始价/折扣锁定,不搞硬涨价。平迁方案施工前单独出细稿(高敏·涉钱)。

## 经营仪表盘

超管后台(收入面板已有地基)加:MRR/ARR、按层 NDR、订阅↔credits 收入占比、免费→付费转化率。学金蝶按层披露的纪律:**每层单独看,别看平均数**。
