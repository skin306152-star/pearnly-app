# Pearnly 2.0 · 产品定位重定义 + 闭环升级路线图(B 路线)

> Zihao 2026-06-07 拍板方向。本页是新北极星,**升级**(不是推翻)`PRODUCT_VISION_MODULAR.md`:
> 三层架构(底座/模块/出口)+ 业态套餐仍成立;本页新增「自成闭环的进销存-做账-报税」能力 + 把定位从
> "事务所助手"放大到"全规模企业的 AI 财税自动驾驶"。任何窗口加功能前先对这页。

## 一、新定位(一句话)

**Pearnly = 泰国企业的「AI 财税自动驾驶」。你只管做生意(进货 / 卖货 / 收款),系统自动把它变成账、税、报表。** 路边小店、中小企业、集团、会计事务所,**同一套**,够用、好用、**不用录**。

## 二、灵魂(护城河 · 跟石板 ERP 的根本区别)

**经营动作本身就是输入** —— 拍发票(OCR)、POS 卖货、扫码进货、LINE 一句话 → AI 自动生成凭证 / 账 / 申报。
**"录入即经营,记账自动发生"。**

- 传统 ERP(Odoo/SAP/FlowAccount…)= **人迁就系统**:录单据、配科目、走流程、要培训,僵硬繁琐。
- Pearnly = **系统迁就人**:AI 把经营动作自动变成账和税,人只确认例外。

这一条是 Pearnly 打破市场上"石板 ERP"的核心,所有功能都为它服务。

## 三、打破石板 ERP 的六条原则(做每个功能都守)

1. **录入即经营,不另设录入**:拍照 / POS / 扫码 / LINE 就是输入,不让用户"再录一遍单"。
2. **AI 自动记账**:凭证、借贷科目 AI 自动生成 / 推荐,小白不用懂会计,只确认异常。
3. **业态套餐一键配**:选"我是什么店"自动开好,不像 ERP 开几百个开关、配三个月。
4. **零培训**:四态 / 向导 / 手机端 / LINE,看一眼会用。
5. **自成闭环但不锁死**:能自己记账报税,也能继续喂外部 ERP —— **客户自由选,不绑架**。
6. **全规模同一套,按需伸缩**:小店到集团到事务所,开不同模块,不换系统。

## 四、全规模适配(谁用、看到什么)

| 用户 | 默认套餐 |
|---|---|
| 路边小店 / 个体 | POS + 简单账 + 自动 VAT,一部手机够 |
| 中小企业 | + 进项采购 + 应收应付 + 报税申报 + 报表 |
| 大企业 / 集团 | + 多门店 + 审批流 + 部门/项目核算 + 细科目 + 多用户权限 |
| 会计事务所 | 代多家(多账套)+ 对账/VAT 复核 + 帮客户一键出申报 |

靠**业态套餐 + 规模按需开模块**实现(复用 `tenant_modules`),不是给每类做一套。

## 五、三层架构仍成立(闭环在其中的位置)

```
① 底座(共享)  商品 · 客户 · 供应商 · 账号权限 · AI录入引擎 · 多语言 · 多账套
② 模块(开关)  进项采购 · 销售开票 · POS · 库存 · 应收应付 · 对账 · 知识库 · 【做账】· 【报税】
③ 出口        发票/小票/税票 · 财务报表 · VAT/WHT 申报 · e-Tax · 喂外部ERP(可选)
```
闭环 = 把 ② 里的"做账""报税"两个新模块补上,并把进项/销售/POS 的数据自动串成 ③ 的账与税。

## 六、B 路线闭环升级路线图(阶段 · 不估工作量)

**已有地基(直接复用,别重造)**:OCR 识别引擎 · 销项开票 · POS 收银 · 库存(批次/FEFO)· 对账/VAT 复核 · ERP 中间件 · LINE · 多账套 · 多语言 · 业态套餐开关。

**Phase 1 · 进项采购闭环**(把"进"补上)
- 供应商主数据 · 进项票**拍照 OCR**(扩 OCR 认进项票)· 进项单据/费用 · 进项 VAT 记录 · 进货→库存入库联动。
- 智能点:拍照即录、AI 认供应商 / 费用科目、与现有进项规则(白名单)联动。

**Phase 2 · 自动做账引擎(核心 · 打破 ERP 的关键)**
- 泰国标准科目表(预设,开箱即用)· **AI 自动生成凭证**:销售/进项/POS/收付款 → 借贷分录 → 总账+明细账。
- 智能点:**不让人配科目、不让人录凭证**——按业务类型 AI 自动生成,人只审例外。这是"自动驾驶"的发动机。

**Phase 3 · 报税闭环**(出口)
- 销项 + 进项 VAT 自动汇总 → **ภ.พ.30 一键申报表** · WHT(PND 50/53)· 月底一键出报税包。
- 智能点:自动汇总 + 差异/异常提醒(复用现有对账能力),不手工编表。

**Phase 4 · 财报 + e-Tax + 看板**
- 利润表 / 资产负债表自动出 · e-Tax 直报税局 · 老板经营看板。

> 贯穿四阶段:每环节 AI 自动化 + 四态/向导/手机/LINE + 进销存账税一条龙。

## 七、不冲突声明(为什么和现有功能/ERP 都不打架)

- **现有对账 / VAT 复核**:那是给**会计事务所核别人账**的工具,继续存在,归"事务所套餐"。B 路线是给**企业自己**做账报税,两者并行、互不删改。
- **外部 ERP**:想用 Xero / MR.ERP 的客户**照样喂数据**(ERP 中间件保留);想要闭环的客户用 Pearnly 自记账。**两条都支持,客户选** —— Pearnly 既是"喂 ERP 的数据源",也是"自成账的系统"。
- **技术上**:B 路线全是"**新增模块**"进现有三层架构,靠 `tenant_modules` 开关,关了不报错(模块化原则#4),不破坏任何现有租户。

## 八、设计完整性原则(2026-06-07 教训 · 防"只做大概,中途返工")

今天踩的坑:UI 设计只画了"操作面"(收银员卖货),反复漏掉"老板配置后台 / 登录接入 / 边界",边做边补(收银员管理、桌台管理、收款设置、店铺码…都是中途新增)。

**铁律:一个模块"设计完成"必须覆盖六类,缺一类 = 没设计完(必返工):**
1. **操作面**(日常用的人:收银员/会计 怎么用)
2. **老板配置后台**(主数据 + 设置:加桌台/加收银员/收款方式/科目…)
3. **登录 / 接入流**(谁怎么进:角色落地 / 设备绑定 / 首次开通)
4. **四态 + 边界**(空/加载/错 + 异常输入)
5. **角色权限**(谁能看能改)
6. **跟其它模块的联动**(数据从哪来、汇到哪去)

出设计稿前先用这六类**过一遍清单**,把缺的屏补齐,再叫"设计完成"。别先画操作面就开工。

## 五-bis、模块按"谁用"归类(商户栈 vs 事务所栈 · 2026-06-08 Zihao 纠偏)

现有的 进项识别 / 对账(GL-VAT、银行)/ VAT 复核,当初是给**会计事务所代账核对**做的工具,被误当"通用功能"放给所有人 → 商户看到一堆"对账/复核"懵。按"谁用"重分:

- **商户栈**(零售/餐厅/药房/批发/服务 · 自己做生意):采购进货 · 销售/POS · 库存 · 收款 · 应收应付 · **自己的**做账 · **自己的**报税。求简单、自动、不用懂会计。
- **事务所栈**(替别人记账):识别中心(批量 OCR)· 对账(GL-VAT/银行)· VAT 复核 · 代多家(多套账)· 知识库 · 喂外部 ERP。专业工具。
- 落地:选「会计事务所」业态开事务所栈;选商户业态开商户栈,**不给商户显示那套事务所对账工具**。

**两个"进项"必须分清(治混淆)**:
- **商户的采购(进项)** = 我这家店进货/付供应商/记费用/抵我的进项税。入口 = 拍照 OCR 进项票 + LINE 一句话记费用 + 供应商。简单 →(= B 路线 Phase 1)。
- **事务所的"进项"** = 批量识别客户公司票据 → 对账 → 喂 ERP。是另一回事,归事务所栈。

## 六-bis、头号地基:套账隔离(2026-06-08 Zihao 确认为真问题)

闭环(做账/报税)必须建在"套账隔离正确"之上,否则多公司数据混一起。**定为 2.0 闭环的头号地基前置(排在进项/做账之前)。** 详见 `docs/workspace-isolation/00-04`。
- 套账(`workspace_client`)= 一个法人公司/经营主体(一套账本)= **隔离硬边界**;界面词按业态可变(事务所→「公司/客户」· 商户→「我的公司」)。
- **门店 = 套账内部的 location**(连锁:一套账多门店),**不是多套账**;集团多套账合并报表留后续。
- 三层分清:**租户**(账号/计费/谁能进哪些套账)/ **套账**(公司·隔离边界)/ **买方 client**(套账内的筛选维度)。
- 现状做了一半(库存/POS 对,商品/销项/进项/对账没按套账滤)→ 按 workspace-isolation 项目 PO-0→8 补齐(风险低→高 · 涉钱先报方案)。

## 九、与旧愿景的衔接

- `PRODUCT_VISION_MODULAR.md` 的「不是又一个 ERP」一句**修订为**:不做"石板录入式 ERP";Pearnly 走"AI 自动驾驶式"的进销存-做账-报税闭环 —— 既能自成账,也能喂外部 ERP。三层架构 + 业态套餐 + 模块化六原则**全部保留**,本页是其上的定位升级与闭环补全。
