# T5 施工单 · 优化小助手「首次设置」配对窗(砍黑话 · 自动探测 · 可最小化)

> 来源:Owner 真机截图反馈(2026-06-22·昨日讨论遗漏项·补登)。companion 本地配对窗 `gui_pairing.py`。
> 原则(Owner 五字):**简单 · 方便 · 快捷 · 精准 · 兜底**。让不懂技术的会计**最少输入**就能接通,黑话由小助手自己解决。
> 施工窗口:companion repo `D:\pearnly-companion`(纯本地·不碰云端·改 UI 每新逻辑 ≥1 测试)。

---

## 1. 问题(Owner 截图)

当前「首次设置」窗逼客户填一堆技术黑话:① Express 账套目录(网络路径)② 账套在选择列表第几行(神秘行号)③ PACK 工作日(日期)。客户看不懂、也不该懂。且窗口**无最小化按钮、强制挡在主屏前**。

## 2. 现成可复用(PM 已读真码 · 直接采信)

- `account_probe.probe_account_sets(express_root)` **已实现**:扫 `…\70EXP` 子目录,凡含 `ISINFO.DBF` 即一账套,读 `THINAM`(公司名·人话)/`ENGNAM`/`TAXID`,标 `writable`。配对时本就调它上报候选(`PITFALLS_AND_FLOW.md` §5)。
- 即:**"自动读账套 + 列公司名给客户选"所需的数据,代码已经有了** —— 现在是有数据却不用,反让客户瞎填。本单 = 把已探测的东西**呈现出来给客户选**,而非要他手填。

## 3. 改造(四点)

### 3.1 砍「Express 账套目录」手填 → 自动探测(对应红框1)
- 不再让客户敲账套目录路径。流程改为:小助手**先定位 Express 根目录**(`express_root`)→ `probe_account_sets` 扫出全部账套。
- **定位 `express_root` 的兜底链**(精准+兜底):① 先试自动探测(常见安装位置 / 已装 ExpressI 的注册表或默认数据路径);② 探不到 → 只在此时给**一个「找到我的 Express」浏览**(指到 `70EXP` 目录即可·一次)→ 之后照常扫账套。**正常客户零输入,异常才浏览一次。**

### 3.2 「账套第几行」神秘行号 → 探测结果下拉选(对应红框2 第一行)
- 删「账套在选择列表第几行」数字输入。改为**下拉/列表展示探测到的账套(显示真公司名 + 可写标记)**,客户**点自己那家**。
- 从这一选,小助手**自动推出**:`account_dir`(账套目录)、`account_company`(公司名·PACK 硬闸期望)、`account_set_row`(PACK 公司 grid 行号)、账套名。客户不碰任何一个。
  - ⚠️ `account_set_row`(PACK 时公司 grid 第几行)推导:若 grid 顺序 ≠ probe 顺序,按**公司名/账套名匹配**定位行号,别假设同序。这是探测→行号唯一需要小心的映射。
- 与 T2(账套审批)对齐:客户选的是"我家账套",最终能不能写仍由云端 Owner 审批 + 三重闸把关(本单只管 UI 选得对、值填得准)。

### 3.3 「PACK 工作日」→ 全自动,客户不可见(对应红框2 第二行/最后一个)
- 删「PACK 工作日」字段。真客户**默认 = 今天**(测试账套 DATAT 那个 31/12/2558 是测试期专用·挪到隐藏/高级,不进客户视野)。
- "每次录完自动跑一遍":录入后**机会式触发 PACK**(检测 Express 空闲即跑·占用则顺延·复用现有 `pack_scheduler` 占用检测 + 夜间窗兜底),客户**完全看不到 PACK 这个词**,不配置、不选择。

### 3.4 窗口可最小化 · 不强制置顶
- 加**最小化按钮**(标准窗口控件);去掉强制置顶/模态独占,允许缩到托盘后台,不硬挡主屏。

## 4. 改后客户只剩三样要碰

**① 粘贴配对码 · ② 填 Express 登录账号 + 密码(T1·本地加密) · ③ 从列表点自己的账套。** 其余(目录/行号/PACK 日期)全自动。「开机自启」勾选保留。

## 5. 测试

- `probe_account_sets` → 列表渲染:多账套时按公司名列出、`writable` 标记正确、可选。
- 选某账套 → 正确推出 `account_dir`/`account_company`/`account_set_row`/账套名(含 grid 顺序≠probe 顺序的映射用例)。
- `express_root` 自动探测命中 → 零浏览;探不到 → 走一次浏览兜底。
- PACK 工作日缺省 = 今天;客户视野无该字段。
- 窗口最小化/还原行为;非置顶。
- 既有 companion 单测全绿;新 UI 逻辑(探测→映射、行号推导)≥1 测试。

## 6. 验收(真机清单 · DATAT)

1. 全新配对:窗口只需配对码 + 登录账密 + **从下拉选 DATAT(显示真公司名 มานะชัยบริการ)** → 配对成功(目录/行号/日期均自动)。
2. 配对结果落 config 的 `account_dir`/`account_company`/`account_set_row` 与手填版**一致**(回归:自动推导 == 旧手填值)。
3. 录一张测试单 → 机会式 PACK 自动跑 → 报表可见(对齐 T0/T1 链)。
4. 窗口能最小化、不强制挡屏。
5. `express_root` 探不到时浏览兜底走通。
