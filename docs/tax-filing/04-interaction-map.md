## 一、导航 IA
做账组(或独立「报税」组·按业态/模块 key=`tax`)新增:
```
报税 ▼ (门控 key=tax · receivable 类业态开)
  ├ 报税中心    data-route="tax"            → 屏1(主屏·本月要报清单)
  ├ 报税设置    data-route="tax-settings"   → 屏4(配置后台·平铺页)
```
PP30/PND 复核 = 从报税中心点行进(非独立导航项)。门控 `assert_module_enabled(tax)`。

## 三、状态流
```
做账 close-period ──generate_filings──► 本期 PP30/PND prepared(报税中心出现·状态"待复核/已备好")
  复核 → recompute(数据变时)→ 报前体检(anomalies)
  e-Tax 提交(二次确认·体检过)──► filed + 回执号;做账对应期可锁
  未连 e-Tax → 导出官方格式 → 手工上传 RD → 回来 mark-filed
```
> 单一事实源:税表从做账账本派生;已报(filed)不可改,错了走更正申报。

## 四、跨模块 seam
| seam | 触发 |
|---|---|
| 做账 → 报税 | close-period → generate_filings(销项税/进项税/WHT 汇总) |
| 进项 → PND | 进项 service 付款 wht + 供应商类型 → PND 明细;扣缴凭证关联 |
| 报税 → e-Tax | /file(etax) → RD e-filing 提交回执 |
| 报税 → 看板 | 本月应交税 → Phase4 经营看板待办 |

## 五、四态 + 边界
| 屏 | 空 | 加载 | 错 | 边界 |
|---|---|---|---|---|
| 报税中心 | 「本期还没到报税时间 / 还没账」 | 骨架 | [重试] | 本期未结账→提示先去做账结账 |
| PP30 复核 | (有数据)| 骨架 | [重试] | 未结账数据不全提示;留抵(进>销)显"留抵下月" |
| PND 复核 | 「本月没代扣预扣税」 | 骨架 | [重试] | 缺税号硬拦不让报 |
| 报税设置 | (默认已 seed) | 骨架 | 保存失败提示 | 未登记 VAT→不显 PP30 |

## 六、角色权限
owner/会计:看+复核+提交+设置;员工:只看。写/提交 `require_account_owner` 或会计;闸 `test_tax_permission_gate`。

## 七、交互完整性(过 DESIGN_LANGUAGE 四-bis · 每动作三态+不可逆确认+落点)
| 动作 | 成功 | 失败 | 进行中/边界 |
|---|---|---|---|
| e-Tax 提交 | **二次确认**「提交后不可改·更正需走更正申报」→ 成功「已报·回执 #」+ status=已报 | 「e-filing 失败·重试/改手报」;体检硬异常→拦+列异常 | 提交中 loading;未结账→拦提示先结 |
| 导出 PP30/PND | 下载官方格式 | 失败重试 | — |
| 重算本月 | toast「已按最新账本重算」 | — | filed 的不重算 |
| 发扣缴凭证 | toast「已发给 N 个收款方」 | 缺邮箱/LINE 提示 | 缺税号的跳过+提示 |
| 标记已报(手报) | 填回执→toast | — | — |
| 设置保存 / 接入 e-filing | toast | 接入失败说明 | — |
> 字段空态:回执号未报显「未提交」;留抵显「留抵 ฿X 下月」非空白;缺税号显式标红非静默。**0 税额照常生成表,不隐藏。**
