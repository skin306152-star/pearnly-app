## 「UI 到底包含什么」——设计系统的完整解剖(9 层)

UI 不只是"长什么样",它是 9 层叠加。**有一层没定标准 = 那一层各屏各搞 = 廉价感。** 本标准逐层钉死:

| 层 | 是什么 | 本标准状态 |
|---|---|---|
| 0 设计原则 | 为什么这么排(收拢/分层/克制…) | ✅ 第 1 节 |
| 1 基础令牌 | 色/字/**间距**/**布局**/尺寸/圆角/阴影/层级/动效/断点/图标 | ✅ 第 2 节 |
| 2 组件 | 按钮/输入/弹窗/表格… | ✅ kit-final |
| 3 页面模板 | 6 种页骨架 | ✅ templates |
| 4 **位置/对齐** | 主操作放哪、数字右对齐… | ✅ 第 4 节 |
| 5 状态/反馈 | 四态/动作三态/校验 | ✅ 第 5 节 |
| 6 内容/格式 | 货币/日期/佛历/语气/i18n | ✅ 第 6 节 |
| 7 主题 | 浅色/暗夜 | ✅ 令牌第二套 |
| 8 无障碍 | 焦点/对比/触控/aria | ⚠️ 基线(第 7 节) |

## 1. 设计原则(每屏都守)
收拢不悬空 · 分层定主次(主信息带→控件带→内容)· 克制渐进披露(不甩 KPI 卡墙)· 内容当主角 · 行动卡非堆被动统计 · 动作有归属(按钮收容器内右对齐)。

## 2. 基础令牌(改一处全站换 · 值见 kit-final.html `:root`)
- **颜色**:主 `--accent` 绿 #0E7C66 / 语义 green·amber·red·blue / 中性 ink·ink2·ink3·line·line2 / bg·card。暗夜 = `:root.dark` 第二套。
- **字阶**:基线 14px;标题 21–22/700;北极星数 28–34/700;标签 10.5–12/600;小字 11.5。字体 Inter + Noto Sans Thai。
- **间距标度(全站只许用这几个值)**:`--s1 4 · --s2 8 · --s3 12 · --s4 16 · --s5 20 · --s6 24 · --s8 32`(4 的倍数)。禁止裸写 13px/17px 这种。
- **布局栅格**:侧栏 240 · 内容区 **流式铺满**(`width:100%` · **无 max-width · 不居中聚拢** · Zihao 2026-06-09 拍板「铺满不聚拢」)· 面板宽 = **填满内容区 100%** · 面板间距 16(--s4)· 面板内边 20–24(--s5/s6)· 行内边 12–14。两栏详情 = 主 1fr / 侧 360。
  - **禁小固定 max-width + 禁居中聚拢**:历史/识别记录(history)是标杆(流式铺满到边);进项待归类(760)、供应商(920)等小宽度、以及 `margin:0 auto` 居中留白都致"宽屏右边大片空白 + 缩放跑偏"= bug,一律改真铺满。kit `.ui .wrap` = `width:100%;max-width:none`。
  - **任意浏览器缩放(50%–200%)都贴满自适应**,不留大片死白、不跑偏。验收每屏在 67% / 100% / 150% 缩放各看一遍。
- **尺寸标度**:按钮高 36–40 · 输入高 38 · 图标 16(行内)/18(导航)/24(空态)· 头像 32/34 · 圆角 面板/控件 7、chip 5–6。
- **阴影/高度**:卡 `--sh`(1px 柔)· 浮层 `--sh2`(弹窗/下拉/Toast)。
- **层级 z-index 标度**:base 0 · sticky 100 · dropdown 1000 · modal 2000 · toast 3000 · tooltip 4000。**禁随手写 z-index:9999。**
- **动效**:时长 快 120ms(hover)/ 常 180ms(开合)· 缓动 ease-out。仅透明度+位移,禁花哨。
- **断点**:桌面 ≥1024 / 平板 768–1023 / 手机 <768。手机:两栏→单列、弹窗→底抽屉、侧栏→汉堡。
- **图标**:Lucide(线性·stroke 2·currentColor)。**禁 emoji 当图标。**

## 4. 位置 / 对齐(钉死 · 消"放哪"争议)
- 页主操作 → **头部带右上**(北极星左 · 按钮右),标题外不留悬浮元素。
- 筛选/页签 → 控件带**左**;搜索/月份/导出 → 控件带**右**。
- 表单:**标签在输入上方**;必填星号在标签后;错误在输入下方。
- 数字/金额 → **右对齐 + tabular-nums**;文本 → 左对齐;状态 pill → 行尾。
- 行内动作(⋯/编辑/删)→ 行**最右**。弹窗按钮 → 底部**右**(次左主右)。
- 「新东西放哪」决策树:全新业务域→新模块组 / 现有数据的新动作→该页头部带按钮 / 新子视图→页内 tab / 配置→设置页一节 / 新字段→表单分区。
- **全局外壳**:左上角 = **logo + 品牌名「Pearnly」**(固定品牌,**不渲染用户名**)。用户名/账号/退出 → 收进**右上角头像菜单**。顶栏 = 账套切换(左)· 全局搜索(中)· 通知铃 + 头像(右)。

## 4-bis. 动作/按钮分类标准(功能多也不乱 · 每个屏都按此摆按钮)
> 不给每个按钮单独定标准 —— 按 3 轴**归类**,每类有固定长相+位置。再独特的功能(作废/红冲/生成替代收据…)也落进这套。
- **A. 重要度(决定长相)**:主 = 实心(`.btn.pri`,**一个容器只 1 个**)· 次 = 描边(`.btn.sec`)· 三级/不常用 = 文字/幽灵(`.btn.gh`)· 危险 = 红(`.btn.dng`)+ **必二次确认**。
- **B. 作用域(决定位置)**:页级主操作 → **页头带右上** · 列表筛选/导出 → **工具条** · 单行的 → **行尾**(常用露 1–2,其余进 `⋯` 菜单)· 多选的 → **批量条** · 某卡片内的 → **该卡底部右**。**按钮永不悬空。**
- **C. 交互模式(决定套路)**:任何功能都是这几种之一 —— 增/删/改/查/审批/导入导出/分配/筛选/复制/状态流转。每种套对应标准件(如"删"=Alert Dialog,"筛选"=工具条 chips)。
- **D. 渐进披露(铁律)**:每个位置**只露 1–2 个最重要的**,其余收进 **`⋯ 更多`**;**一排 ≥4 个按钮 = 违规**,必须收。全局 `Ctrl+K` 命令面板可达任何功能,不靠堆按钮。
- **E. 命名**:一律 **动词 + 名词**(记付款/生成凭证/导出 Excel);术语全站统一(别"发票/单据"混用)。
- 例:单据详情页头 8 个按钮 → 记付款(主)+ 作废(危险)+ 编辑(次)+ 生成替代收据/匹配商品(收进对应卡片)+ 其余(`⋯`)= "1 主 + 1 危险 + 1 次 + ⋯",不再挤成一排。

## 5. 状态 / 反馈(四态 + 三态 · 缺一不算完)
- 每个数据视图必有四态:**加载(骨架屏)/ 空 / 错误(带重试)/ 已载入**。
- 每个动作必有三态:**成功(toast/inline)/ 失败(具体错+下一步)/ 进行中(禁连点+loading)**。绝不"点了没反应/静默改"。
- 字段缺值:合理占位"—" 或 缺则不显 或 标"待补",不留谜之空白。
- 删除/作废:二次确认 + 成功反馈 + 连带影响说明。

## 6. 内容 / 格式
- 货币 `฿` 前缀 + 千分位 + 两位小数。
- **日期/历法(标准已封板 · 三层收口)**:① 内部**永远存公历 ISO**(DB 不存佛历)② **入口归一**:OCR/导入判佛历(年>2400 减 543)/公历统一转公历再存(**已修 ✓**)③ **出口一个共享格式化器** `formatDate()`:显示/表格/CSV/PDF 全走它,读历法偏好渲染。**默认佛历 พ.ศ.(2026-06-09 拍板)**,全局开关(佛历/公历)。**待接通(前端)**:建统一 `formatDate` + 全局开关 + 各处显示/导出改走它(现状:设置页"日期格式"仅样式、佛历只在开票向导逐张选)。
- 文案语气:简洁、不自曝短处(禁"OCR 抽不准"这类)、**不露「AI」字样**(用智能/自动/系统)。
- i18n:th/en/zh/ja 四语;新组件测长文本不撑破。

## 7. 无障碍基线(先守这几条)
焦点可见(focus ring,见 `.fin.foc`)· 文字对比 ≥ WCAG AA · 触控热区 ≥ 40px · 弹窗有标题/可 Esc 关。深做(aria/读屏)延后。

## 8. 治理(让标准不退化)
- 新屏**必须声明是 6 模板里哪一种**,不合规打回。
- 视觉保真闸:生产页 getComputedStyle 逐项 vs 实物源快照,差异 = 红 = 拦推。
- 禁新增裸 hex(扩 `TOKEN_CSS` 白名单闸);令牌值只在 `:root` 改。
