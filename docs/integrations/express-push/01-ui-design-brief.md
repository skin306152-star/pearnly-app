# Express 自动推送 · UI 设计要求(给设计师/设计 AI)

> 用途:把"Pearnly 识别完发票 → 自动推送进 Express 记账 → 异常留人工审核"这套能力,设计成可用的界面。
> 交付:每页要桌面 + 手机两套、要可点交互原型(状态流转、弹窗、实时更新),不是静态图。
> 语言:产品面向泰国会计/中小企业,**UI 主文案用泰语**(下方给了泰语建议文案),界面需支持泰/中/英切换。
> 风格:克制、专业、大厂级 SaaS 质感(参考 Stripe / Linear / QuickBooks 后台)。不要花哨渐变、不要 emoji 堆砌、不要"AI 味"营销文案。要支持暗夜模式。

---

## 0. 一句话讲清这个产品

会计事务所每天收一堆供应商发票。Pearnly 用 AI 把发票拍照识别成结构化数据。**这个功能负责把识别好的发票,自动записать(记账)进客户正在用的 Express 财务软件里**,会计不用再手敲一遍。识别不靠谱的单子不自动推,排队等人工确认。

设计的本质是三块界面:**①接通 Express(配置)→ ②看推送进展(运营)→ ③处理推不进去的(异常审核)**。

---

## 1. 必须先让设计师理解的 5 个概念(界面里会反复出现)

| 概念 | 大白话 | 在 UI 里表现为 |
|---|---|---|
| **ERP 连接(Connection)** | 一条"通往某个财务软件的管道",一家公司可有多条 | 连接列表里的一张卡 |
| **本地 Agent(本地小助手)** | 装在客户办公室某台电脑上的小程序,真正动手把单子敲进 Express。Express 是本地软件、没网络接口,只能靠它 | 连接卡上的"在线/离线"状态灯 + 安装引导页 |
| **发件箱队列(待推送队列)** | 已批准、排队等着被推进 Express 的发票 | 运营中心的"待推送"列表 |
| **置信闸门(自动/人工分流)** | AI 有把握的自动推;没把握的拦下来等人确认 | 一条阈值滑杆 + 异常审核队列 |
| **录入方式(两种)** | 把单子写进 Express 的两种手法,用户二选一:**模拟录入(稳,推荐)** / **直写数据(快,高级,有风险)** | 配置里的单选 + 风险确认弹窗 |

## 2. 入口与信息架构

- 放在 Pearnly 主导航的 **「集成 / การเชื่อมต่อ」** 模块下,新增子项 **「ERP 推送 / ส่งเข้าระบบบัญชี」**。
- 该模块顶部一条横向 Tab:**连接(การเชื่อมต่อ)· 推送中心(ศูนย์ส่งข้อมูล)· 异常审核(รอตรวจสอบ)· 设置(ตั้งค่า)**。
- 「异常审核」Tab 上要带**数字角标**(有多少单等人处理),这是会计每天最先看的地方。

## 3. 逐页设计要求

### P1 · ERP 连接列表(Tab:连接)

**目的**:一眼看清接了哪些财务软件、通不通。

- 列表/卡片网格,每张卡 = 一条连接:
  - ERP 图标 + 名称(如 `Express`)、连接的**账套/公司名**(如 `9.ข้อมูลทดสอบ DATAT`)
  - **状态徽章**(四种,颜色语义化):
    - 🟢 已连接 เชื่อมต่อแล้ว(Agent 在线 + 最近推送正常)
    - 🟡 Agent 离线 ออฟไลน์(配好了但那台电脑没开/没联网)→ 卡上提示"待推送 N 单会等它上线"
    - 🔴 异常 ขัดข้อง(连接报错,需处理)
    - ⚪ 未完成 ยังไม่เสร็จ(向导没走完)
  - 一行**迷你统计**:今日已推 N · 待推 N · 失败 N
  - 右上角 `⋯` 菜单:编辑、暂停推送、测试连接、删除
- 右上角主按钮 **「+ 新建连接 / เพิ่มการเชื่อมต่อ」**。
- **四态**:空态(还没接任何 ERP,给一句引导 + 大按钮)、加载骨架屏、错误态、有数据态。
- **交互**:点卡片进入该连接详情(= P3 Agent 状态 + 该连接的推送统计);状态灯**实时更新**(Agent 心跳,设计上做个轻微的"呼吸/脉冲"动效表示在线)。

### P2 · 新建连接向导(分步,核心交互页)

分步向导(顶部步骤条),**每步一个决策,不要堆在一屏**:

**Step 1 选 ERP 类型**
- 网格卡:Express(本期主推)、FlowAccount、PEAK、Xero…(后几个置灰标"即将支持")。
- 选 Express 后,出一句说明:"Express 是本地软件,需要在能打开 Express 的那台电脑上装一个小助手"。

**Step 2 装本地 Agent**
- 大区块分两栏:左 = **下载按钮**(Windows 安装包)+ 三步图示(下载→安装→粘贴密钥);右 = **连接密钥 token**(一串码 + 一键复制按钮,带"请勿外传"提示)。
- 下方一个**实时等待区**:"正在等待小助手回连…"(转圈)→ 一旦 Agent 上线,自动变成 ✅ "已检测到小助手(电脑名:ACC-PC01)",并自动跳下一步。
- **交互重点**:这里要做"等待→检测到"的实时状态切换动画,让用户知道装完会自动接上,不用手动刷新。

**Step 3 选账套/公司**
- Agent 回连后,自动读出 Express 里的**公司/账套列表**(下拉或单选列表),例如:
  - `58ASIA-SPORT (PDATAT)`、`9.ข้อมูลทดสอบ (DATAT)`、`Z.ข้อมูลเปล่า (DATAZ)`
- 用户选一个作为**推送目标**。被选中的高亮;**危险提示**:若选的是非测试账套,顶部出一条黄条"这是真实账套,推送会写入真实账目,建议先用测试账套验证"。

**Step 4 选录入方式(二选一单选卡)**
- 两张并排大卡:
  - **A · 模拟录入 RPA**(默认选中)— 标签"แนะนำ 推荐 / 稳"。副文案:"像人一样在 Express 里录入,由 Express 自己完成记账,最安全。需要那台电脑保持开启。"
  - **B · 直写数据 DBF**— 标签"ขั้นสูง 高级 / 快"。副文案:"直接写入数据文件,更快,但有数据风险。"
  - 选 B 时**弹风险确认弹窗**(见 P7),用户必须读完勾选"我已了解风险"才能继续。
- 视觉上 A 卡更"安心"(绿系/盾牌图标),B 卡更"性能但谨慎"(琥珀/闪电图标 + 小警示)。

**Step 5 字段映射(可折叠,默认自动)**
- 默认显示一句"已自动匹配字段 ✅",下面一个**可展开**的高级区:
  - 一张映射表:左列 Pearnly 字段(卖家、税号、发票号、日期、税前金额、VAT、含税合计、品项…),右列对应 Express 字段(下拉可改),中间箭头。
  - **默认科目规则**:一句"采购科目按供应商自动选;新供应商首次推送会在 Express 自动建档"。可设一个"兜底采购科目"。
  - 赊购/现购:单选"按 Pearnly 的付款状态自动判断(已付→现购 HP / 未付→赊购 RR)",默认开。
- 多数用户**不展开**直接下一步;高级用户可调。

**Step 6 测试 / 试推一张**
- 一个"测试连接"按钮 → 跑通显示绿勾(检查项清单:Agent 在线 ✓、账套可写 ✓、字段映射完整 ✓)。
- 可选"试推一张样例发票到测试账套"→ 推完展示**将记入的分录预览**(见 P6 的分录卡),让用户亲眼确认记得对。

**Step 7 完成**
- 成功页 + 一个开关 **「开启自动推送 / เปิดส่งอัตโนมัติ」**(默认关,让用户主动开,体现"状态诚实")。
- 引导去看「推送中心」。

> 向导通用交互:可随时**上一步/保存草稿退出**;每步底部"下一步"在未满足条件时禁用并提示原因。

### P3 · 连接详情 / 本地 Agent 状态

- 顶部 Agent 状态卡:🟢/🟡/🔴 + 电脑名 + Agent 版本 + **最后心跳时间**(如"30 秒前")+ 当前队列积压数。
- Agent 离线时:醒目黄条"小助手离线,N 单在等待。请确认那台电脑已开机并联网"+「重新检测」按钮。
- 下方:该连接的推送统计(今日/本周折线或数字卡)、最近推送记录列表(可点进 P6)。
- 操作:暂停/恢复推送、更换账套、重新生成 token、卸载指引。

### P4 · 推送中心(Tab:推送中心 —— 日常运营主页)

**目的**:会计每天盯这页,看单子流过去了没。

- **状态分栏**(顶部分段控件或筛选):**待推送 · 推送中 · 已推送 · 失败**;每段带计数。
- 列表每行一张发票:
  - 缩略图、卖家名、含税金额、**置信度徽章**(高/中/低)、目标账套、状态、时间
  - 状态用语义色 + 文案,**诚实**:只有 Express 真写成功才显示"已推送 ส่งแล้ว"(配 Express 单号);写入中显示"推送中";不要提前显示成功。
  - 失败行:红色 + 一句错误原因摘要 + 行内操作「重试 / 转人工 / 查看」。
- **批量操作**:多选 → 批量推送 / 批量重试 / 批量转人工审核。
- 顶部:搜索(卖家/金额/单号)、按日期/账套/状态筛选、按月折叠。
- 实时:推送进行时,行状态**自动流转**(待推送→推送中→已推送/失败),配轻量过渡动效,不需手动刷新。
- 四态齐全(空/加载/错误/有数据)。

### P5 · 异常人工审核(Tab:异常审核 —— 带角标)

**目的**:AI 没把握的、推失败的,集中在这让人快速决断。

- 进入这里的两类:① **低置信/识别异常**(从没进过推送队列,闸门拦下的)② **推送失败**(进了但 Express 写不进去,如供应商不存在、金额异常、账套被锁)。
- 每条卡片要给**人快速判断所需的一切**:发票原图(可放大)、AI 识别出的字段(可疑字段**标黄**)、为什么被拦的原因标签(如"税号无效""金额低置信""供应商 Express 中不存在")。
- 三个明确动作:**确认并推送 ยืนยัน**(改完字段后送进队列)/ **编辑 แก้ไข**(就地改字段)/ **忽略 ข้าม**(不推,留 Pearnly)。
- 设计成"一张一张快速过"的体验(类似审批流/收件箱),处理完一张自动滑到下一张,顶部角标实时减一。

### P6 · 单据推送详情(抽屉 / Drawer)

从任意列表点一张发票滑出,**核心是"将记入 Express 的预览",让用户信任**:

- 顶部:发票原图 + Pearnly 识别字段(两栏对照)。
- **中部:分录预览卡(最重要)** —— 把这张发票即将在 Express 里记成的复式分录画出来:
  ```
  借 (Dr)  采购/存货        375,347.20
  借 (Dr)  进项税 ภาษีซื้อ    26,274.30
  贷 (Cr)  应付账款 เจ้าหนี้           401,621.50
  ─────────────────────────────────
  发票号 RR581231-002 · 税期 12/2558 · 供应商 ก005
  ```
  借贷两栏 + 合计,**借贷不平要红色警示**。已推送的额外显示 Express 实际单号。
- 下部:**状态时间线**(识别 → 已批准 → 待推送 → 推送中 → 已推送 / 失败,带时间戳),失败显示完整错误 + 重试按钮。
- 抽屉底:操作按钮(重推/转人工/打开 Express 对应单据的指引)。

### P7 · 录入方式风险确认弹窗(选 DBF 直写时)

- 模态弹窗,克制但严肃(非吓人红屏):标题"直写数据模式 · 请确认风险"。
- 要点(图标列表):直接写入财务数据文件;若中断/异常可能导致账目不一致;**系统会在每次写入前自动备份该账套并校验借贷平衡,失败自动回滚**;强烈建议先在测试账套验证。
- 底部:勾选框"我已了解并接受风险" → 勾上才激活"确认使用直写"按钮;另有"改用模拟录入(推荐)"返回。

### P8 · 设置(Tab:设置)

- **自动推送阈值**:一条滑杆 / 三档(保守·均衡·激进)—— 解释"置信度高于此值的自动推,低于的进异常审核"。配一个实时小预估"按当前设置,约 X% 发票会自动推"。
- 默认兜底科目、赊购/现购规则、推送时间(实时 / 定时批量)。
- 通知:推送失败 / Agent 离线时,通过哪些渠道提醒(站内 / LINE / 邮件)。

## 4. 全局交互与状态规范(所有页通用)

- **四态 UI 必做**:空态(给引导别给空白)、加载(骨架屏,别转圈黑屏)、错误(说人话 + 重试)、有数据。
- **状态诚实**:凡涉及"已推送/已记账"的字样,只在后端真确认后才显示;过程态如实显示"推送中"。绝不为了好看提前显示成功。这是产品底线,设计上要给"推送中/待确认"留位置。
- **实时性**:Agent 心跳、队列状态、异常角标都应**自动更新**(WebSocket/轮询),设计上体现"活的"而非"要手动刷新"。
- **危险操作二次确认**:删除连接、切真实账套、DBF 直写、批量推送真账套 → 都要确认弹窗。
- **Toast/反馈**:每个动作有即时反馈(成功/失败 toast),失败给原因 + 下一步。
- **可访问性**:键盘可达、对比度达标、状态不只靠颜色(配图标/文字)。

## 5. 视觉与语气

- 整体:专业财务 SaaS,信息密度适中,留白克制,层级清楚。参考 Stripe Dashboard / QuickBooks / Linear。
- 颜色语义化:绿=正常/已推送,琥珀=待处理/离线/谨慎,红=失败/危险,中性灰=未开始。
- **暗夜模式**必做,与亮色等质量。
- **文案去 AI 味**:简洁、说人话、动词开头("装小助手""选账套""试推一张")。不要"轻松实现智能化对接"这种营销腔。
- i18n:泰语为主(已给建议文案),中/英可切;泰语注意用词统一、避免中英夹杂。
- 图标统一一套线性图标库;ERP 用各自品牌色块占位即可。

## 6. 一条龙主流程(给设计师串场景用)

**Happy path(接通到自动跑起来)**
1. 进「集成 → ERP 推送」→ 空态 → 点「新建连接」
2. 选 Express → 下载装 Agent → 粘 token → 页面自动检测到 Agent 上线 ✅
3. 选测试账套 DATAT → 选「模拟录入(推荐)」→ 字段自动匹配 → 测试连接全绿 → 试推一张看分录对 → 完成 → 开启自动推送
4. 回「推送中心」,看到新识别的发票自动从"待推送→推送中→已推送"流过去

**异常 path(会计的日常)**
1. 「异常审核」角标显示 3
2. 进去:第 1 张"税号无效"标黄 → 会计就地改对 → 确认并推送
3. 第 2 张"供应商 Express 中不存在" → 选"确认推送"(系统会自动在 Express 建档)
4. 第 3 张图太糊识别不全 → 忽略,留着手工处理
5. 角标归零

**Agent 离线 path**
- 推送中心顶部黄条"小助手离线,5 单等待中"→ 会计去那台电脑开机 → 几秒后状态转绿 → 5 单自动补推
