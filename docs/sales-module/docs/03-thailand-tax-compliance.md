# 03 · 泰国税务合规(销项)

> ⚠️ 客户本身是**会计事务所**,以下法条/格式是工程侧的合规底线梳理,**最终细节以客户确认为准**。
> 目的:让数据模型和单据格式从一开始就不踩硬合规雷(尤其连号、不可改、红冲)。

## 销项单据类型

| 单据 | 泰文 | 何时用 | 关键要求 |
|---|---|---|---|
| 完整税务发票 | ใบกำกับภาษี (เต็มรูป) | B2B、需进项抵扣的买方 | 买卖双方名称/地址/税号齐全 + VAT 分列 |
| 简式税务发票 | ใบกำกับภาษีอย่างย่อ | 零售/POS、买方为终端消费者 | 可不含买方信息,VAT 可含在价内并注明(适配 POS!) |
| 收据 | ใบเสร็จรับเงิน | 收款凭证 | 证明已收款 |
| 报价单 | ใบเสนอราคา | 开票前 | 非税务凭证 |

> Q1 若为"零售 POS",简式税票(อย่างย่อ)是合法且更轻的选择;若 B2B 开票,需完整税票。模型两者都要支持。

## 完整税务发票必备字段(Revenue Code §86/4)

1. 明显标注 **"ใบกำกับภาษี"** 字样
2. 卖方:名称、地址、**税号(TIN)**
3. 买方:名称、地址(完整税票需买方税号)
4. **发票号 เลขที่**(+ 必要时册号 เล่มที่)—— **连续编号**
5. 商品/服务:名称、数量、金额
6. **VAT 税额单独分列**(7%)
7. **开具日期**
8. 总公司/分公司标识(สำนักงานใหญ่ / สาขาที่ …)

## 三条硬约束(直接定数据模型)

1. **连号不跳(gapless)**:发票号必须连续。→ `document_number_sequences` 事务内 `FOR UPDATE` 取号;
   草稿不占号,只有正式 issue 才取号。作废也不回收号(留 void 记录)。
2. **开出即不可改(immutable)**:正式开出后不能改金额/明细。→ `status='issued'` 后业务层禁改;
   改动只能通过红冲/补开。
3. **更正走红冲/补开**:
   - 调减(退货、折让、开错调低)→ **ใบลดหนี้(credit note,§86/10)**
   - 调增(补收、开错调高)→ **ใบเพิ่มหนี้(debit note,§86/9)**
   - 二者都引用原发票号,自身也要连号。→ `credit_debit_notes` 表(Phase per Q3)。

## VAT 与申报

- **VAT 率 7%**(留 `vat_rate` 列以防政策变动)。
- **VAT 注册门槛**:年营收 > 1.8M THB 才强制注册 VAT;未注册不得开税票,只能开普通收据。
  → 模型需支持"卖方未注册 VAT"场景(`vat_rate=0` / 不出税票)。
- **ภ.พ.30(PP30)**:月度 VAT 申报,次月 15 日前报;= 销项税额 − 进项税额。
  → 销项数据自动汇入,复用主仓库 `vat_excel`(现做进项侧),扩成进销一并汇总。
- **WHT 预扣税**:某些服务交易买方代扣(ภ.ง.ด.3/53)。模型留 `wht_amount`。

## e-Tax Invoice & e-Receipt(Q3 最高级,Phase 3+)

- RD 的电子发票上报系统,两条路:
  - **e-Tax Invoice & e-Receipt**:XML + **数字证书(CA 签发)** 签名,上报 RD。适合量大。
  - **e-Tax Invoice by Email**:小企业,经 ETDA 时间戳盖章,门槛低。
- 都需对接外部,**不进 MVP**;数据模型预留 `etax_submissions`(状态/RD 回执)。

## 工程侧落点小结

| 合规点 | 落到哪 |
|---|---|
| 连号不跳 | `document_number_sequences` + 事务取号 |
| 开出不可改 | `sales_documents.status` + 业务层写保护 |
| 红冲/补开 | `credit_debit_notes`(Q3 gated) |
| VAT 分列/7% | `sales_documents.vat_rate/vat_amount` + 行级 `vat_applicable` |
| 简式税票(POS) | `doc_type` 支持 อย่างย่อ + 买方信息可空 |
| ภ.พ.30 汇入 | 复用扩展 `vat_excel` |
| e-Tax 上报 | `etax_submissions`(Phase 3+) |
