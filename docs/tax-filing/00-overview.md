# 自动报税(Phase 3)· 00 概念与边界

> Pearnly 2.0 闭环第三块 = 把账本变成税表、按时报掉。动前先读 + `docs/product-vision/00`(2.0)+ `docs/product-vision/02`(竞品对位)+ `docs/ui/DESIGN_LANGUAGE.md`。
> **状态:UI 设计封板(2026-06-08)**。4 屏定稿(`桌面/Pearnly_报税_UI预览/` 01-04·收拢版)+ 图纸 00-05。前置:做账(Phase2)产出账本/VAT 结转/WHT 明细。

## 一句话
**自动报税 = 拿做账的账本数据,自动算好每月该报的税表(PP30/PND),核对就报 —— 不用编表、报前抓错、能直连税局提交。** 老板不懂税表,系统备好,点确认。

## 闭环位置
`业务 → 做账(账本) → 报税(本模块:PP30/PND/财报) → e-Tax 直报税局`。报税**吃做账的产出**(销项税/进项税/WHT 明细),不重算业务。

## 真差异化(竞品对位 · 见 `docs/product-vision/02`)
FlowAccount/PEAK 都能"生成 PP30/PND 表"= 泰国标配,不算本事。我们要更进一步:
1. **不用编表** — 数字全从做账账本自动来,老板不碰科目。
2. **报前异常提醒** — 进项超 6 月失效/缺税号 等,报之前就抓,不被税局退。
3. **e-Tax 直报** — 连 RD e-filing 直接提交、拿回执,不用打印跑税局(没连就导出官方格式手报)。
4. **一站所有税种 + 截止提醒** — PP30/PND53/PND3 一屏看齐,到期前 LINE+站内提醒。

## 报什么(泰国月度 · 调研依据 product-vision/02)
| 表 | 是什么 | 截止(e-filing) |
|---|---|---|
| **PP30**(ภ.พ.30)| 增值税:销项税 − 进项税 = 应交/留抵 · **0 也要报** | 次月 15 日(+延展) |
| **PND53**(ภ.ง.ด.53)| 预扣税:付给**公司**的服务费代扣 | 次月 15 日 |
| **PND3**(ภ.ง.ด.3)| 预扣税:付给**个人**的代扣 | 次月 15 日 |
| (年度)PND50/51 + 财报 | 企业所得税 + 报 DBD | 留 Phase4 |

## 四屏(UI 已定稿 · `Pearnly_报税_UI预览/`)
1. **报税中心** — 本月要报的税清单(北极星本月要交税 + 最近截止行动卡 + PP30/PND53/PND3·自动备好/待复核/已报)。
2. **PP30 复核** — 销项−进项=应交·数字逐张可追溯·异常提醒(超期/缺税号)·导出/e-Tax 提交。
3. **PND 复核** — PND53(公司)/PND3(个人) tab·逐笔代扣可追溯·扣缴凭证自动生成·缺税号前置提醒。
4. **报税设置** — 税号/VAT 登记/总分公司·e-Tax 直报接入·截止提醒·0 税额也报。

## 六类完整性(全覆盖)
| 类 | 覆盖 |
|---|---|
| ① 操作面 | 报税中心 · PP30/PND 复核 |
| ② 配置后台 | 报税设置 |
| ③ 接入/角色 | 复用登录;**报/提交 = 会计或 owner;员工只看** |
| ④ 四态+边界 | 空(没到期/没数据)/加载/错 + **未结账→提示先去做账结账**/有不平凭证/0 额也报/缺税号拦 |
| ⑤ 角色权限 | 同③;e-Tax 提交/设置写 = require_account_owner 或会计 |
| ⑥ 模块联动 | 做账(账本/VAT结转/WHT明细)→ 报税数据;报税 → e-Tax 提交;扣缴凭证(进项已生成)关联 |

## 套账隔离
税表/明细按 `workspace_client_id` 隔离;每个主体报各自的税(税号按主体)。
