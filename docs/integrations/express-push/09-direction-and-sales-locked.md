# 方向(进项/销项)第一类公民 + 销项扩展(已锁 · 2026-06-20 Owner)

> 与其它文档冲突时以本文为准(方向相关)。核心诉求:**进项/销项都自动推、两种录入法都支持、用户先选(确认)方向、绝不录错地方。**

## 0. 铁律
**方向(进项采购 / 销项销售)在 Pearnly 这边定死,小助手绝不自己判。** 标签随票一路传到录入端,小助手只照标签办。

## 1. 方向路由(两种录入法各配好)
| 方向 | RPA 走的门 | DBF 写的表族 | 单别 |
|---|---|---|---|
| **进项采购** | `1.ซื้อ → 4.ซื้อเงินเชื่อ` | AP(APTRN/APBAL)+ ISVAT('P')+ GL | RR/HP |
| **销项销售** | `2.ขาย → 4.ขายเงินเชื่อ` | AR(ARTRN/ARBAL)+ ISVAT('S')+ GL | IV/… |

## 2. 防"录错地方"三道闸
1. **源头定死**:方向由 Pearnly 识别判 + 用户确认,载荷带 `direction`;小助手不猜。
2. **录前门校验(RPA)**:Express 窗口标题含单别(`ซื้อเงินเชื่อ`/`ขายเงินเชื่อ`)。录入前**核对"标题 == 标签方向",不一致立刻停手不录 + 报异常**。
3. **写表校验(DBF)**:写错表族会被 gold 标准答案比对 + 借贷校验当场抓出。

## 3. "用户先选择"落点
- **每张票**:Pearnly 自动判方向,用户在确认/异常审核可改(拿不准的票必经人确认)。
- **连接设置**:本连接处理 进项 / 销项 / 两者(默认两者)。

## 4. 系统要加的(方向贯穿全链)
- 识别/确认:带 `direction`(进项/销项)。
- P1 载荷:加 `direction`;现 `mapper`/`purchase_adapter` 只采购 → **加销售映射**(客户/IV/应收 AR/销项税)。
- 小助手:方向路由 + 录前门校验(RPA)、写对表族(DBF)。
- 前端:审核显方向可改、连接设进项/销项/两者。

## 5. 销项是采购的镜像(非重做)
- 架构同一套;销项 = 把 supplier→customer、AP→AR、ISVAT('P')→('S')、`1.ซื้อ→4`→`2.ขาย→4`、RR→IV 镜像一遍。
- **节奏**:进项先真机验证录入模式跑通(DBF/RPA)→ 销项照镜像加,省时省险。
- 顺带:`pearnly-companion` 原本就是冲销项做的(字段是客户/ขายเงินเชื่อ)→ 销项 RPA 模板/字段有现成底子。

## 6. 不变
方向贯穿不改 P1 既有口径(钱/税/状态确定性引擎算);只加 `direction` 维度 + 销售映射 + 路由 + 门校验。
