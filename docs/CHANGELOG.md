# Pearnly · CHANGELOG 自动生成方案(REFACTOR-WC-G6)

> 整顿期 G6 · 给项目接 git-cliff 自动生成 CHANGELOG 的**方案** + **配置存档** + **手动跑法**。
> 暂不上 CI(2026-05-28 窗口 C 决策 · 见下"为啥不上 CI")。
>
> 本文 = 方案说明 + cliff.toml 配置模板(拷贝到项目根即可启用)。

---

## 0. 为啥要做(WHY)

整顿期(2026-05-22 起)commit 严格 conventional commits 风格(铁律 #20):
```
<type>(<scope>): <subject> · REFACTOR-<task-id>
```
机器友好 + 任务 ID 可追溯 → 适合用 [git-cliff](https://github.com/orhun/git-cliff) 自动产 CHANGELOG。

收益:
- 出 release notes 不用手抄
- 月度 / 季度回顾自动出整顿进度
- 给会计师 / 集成方 / 法务一份"整顿期都做了什么"的对外报告

---

## 1. 为啥暂不上 CI(WHY NOT)

窗口 C 拍板:**方案先到位 · CI 集成等收官**:
1. **整顿期 commit 频次高**:B / C 阶段每天 5-15 commit · CI 每次自动重生整 CHANGELOG · 噪音大 · git diff 也大
2. **STATE 类 commit 频繁**:`docs(state):` 每轮一次 · 必须过滤(已写在 commit_parsers `skip = true`)· 但 CI 集成更复杂
3. **真正需要 CHANGELOG 的时机是 release**:整顿期没 release · 等出 v1.0 / 整顿收官 · 那时再"每个 release 自动 cliff 一次"
4. **本地手跑成本低**:`git cliff -o docs/CHANGELOG.md.generated`(< 5s)· 任何窗口想看就生成

**未来切 CI 模式的触发条件**(写在本文 · 后续窗口照办):
- 整顿期收官(`scripts/refactor_progress.py` 综合 ≥ 90%)+ 出 v1.0 tag
- 或:Zihao 拍板"开始按月发 release"
- 触发:加 `.github/workflows/changelog.yml` · 在每个 tag push 时跑 `git cliff --tag v$NEW -o RELEASE.md` · attach 到 GitHub Release

## 3. 拷贝即用 · cliff.toml(放项目根)

```toml
# Pearnly · git-cliff 配置 · REFACTOR-WC-G6 · 2026-05-28
#
# 跑法:
#   git cliff -o docs/CHANGELOG.md.generated         # 全历史
#   git cliff --tag v0.20 -o RELEASE.md              # 给特定 tag 出 release notes
#   git cliff --unreleased --tag v0.21               # 本次 unreleased
#
# 文档:本文件 docs/CHANGELOG.md

[changelog]
header = """
# Pearnly · CHANGELOG

> 由 git-cliff 自动生成 · 不要手改这里(改 cliff.toml + 重跑)。
> 整顿期(2026-05-22 起约 5-8 个月)只显示 refactor/chore/test/docs 类型 ·
> feat 类型出现即 audit(整顿期 0 新功能 · 铁律 #18)。

"""

body = """
{% if version %}\
## [{{ version | trim_start_matches(pat="v") }}] - {{ timestamp | date(format="%Y-%m-%d") }}
{% else %}\
## [unreleased]
{% endif %}\
{% for group, commits in commits | group_by(attribute="group") %}
### {{ group | striptags | trim | upper_first }}
{% for commit in commits %}
- {% if commit.scope %}*({{ commit.scope }})* {% endif %}\
    {% if commit.breaking %}[**breaking**] {% endif %}\
    {{ commit.message | upper_first | split(pat="\\n") | first }} \
    ([{{ commit.id | truncate(length=7, end="") }}](https://github.com/skin306152-star/pearnly-app/commit/{{ commit.id }}))\
{% endfor %}
{% endfor %}\n
"""

trim = true
footer = """
<!-- generated by git-cliff -->
"""

[git]
conventional_commits = true
filter_unconventional = false
split_commits = false

commit_preprocessors = []

commit_parsers = [
    # 整顿期标志类型(按重要度排序)
    { message = "^feat", group = "🚀 新功能(整顿期审视!)" },
    { message = "^fix", group = "🐛 BUG 修复" },
    { message = "^hotfix", group = "🔥 紧急修复(影响付费用户)" },
    { message = "^refactor", group = "♻️  重构(整顿主线)" },
    { message = "^perf", group = "⚡ 性能优化" },
    { message = "^test", group = "🧪 测试" },
    { message = "^chore.*\\(state\\)", skip = true },  # STATE 更新不进 CHANGELOG
    { message = "^docs\\(state\\)", skip = true },
    { message = "^chore", group = "🔧 杂项 / 构建 / CI" },
    { message = "^docs", group = "📚 文档" },
    { message = "^style", group = "🎨 代码风格" },
    { message = "^ci", group = "🤖 CI / 发布" },
    { message = "^build", group = "📦 构建" },
    { message = "^revert", group = "⏪ 回滚" },
    { message = ".*", group = "其他" },
]

filter_commits = false
tag_pattern = "v[0-9].*"
skip_tags = ""
ignore_tags = ""

topo_order = false
sort_commits = "oldest"
```

## 4. 跑法 · 三种典型用例

### A. 任何时候看"上次 release 以来做了什么"

```bash
git cliff --unreleased -o /tmp/unreleased.md
cat /tmp/unreleased.md
```

### B. 出整顿期月报(给 Zihao 看 / 给读者看)

```bash
git cliff --date-order \
    --range 'HEAD@{1 month ago}..HEAD' \
    -o docs/refactor/monthly-report-$(date +%Y-%m).md
```

### C. 整顿收官 · 出 v1.0 release notes

```bash
git tag v1.0.0
git cliff --tag v1.0.0 -o RELEASE.md
# attach RELEASE.md 到 GitHub Release
```
