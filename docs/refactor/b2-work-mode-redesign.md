# B2 重设计 · 工作模式(Work Mode)· 替代"登录后全站硬拦截"

> 2026-05-26 · Zihao 新思路:**小改动大效果**。不做登录硬拦截,做"工作模式选择" +
> 业务功能**惰性提示**。本文件是重设计 + 对原 13 点的提升/冲突评审。实现待确认。

## 1. 核心模型(比原"强制弹窗"更轻)

会话有一个 **work mode**:
- **客户业务模式**:选定一个 workspace 客户 → 本会话业务功能默认归属它。
- **个人事务模式**:不选客户,正常进系统;进业务功能时才惰性提示。

**不在登录时封锁全站**。登录后默认进入(见 §4 默认态),全站可逛;只有踏入"需要客户"的业务功能时,个人模式才弹一个**轻提示**:
> 「这个功能需要先选择客户」 [选择客户] [取消]

## 2. 唯一入口 = 右上角"当前客户"(合并 work mode + 选客户)

把"工作模式 + 选/切客户"**收进右上角同一个控件**(原点 9:唯一切换入口),状态二选一显示:
- `👤 个人事务`(未选 workspace)
- `🏢 当前客户:BAKELAB`(已选 workspace)

点击 → 轻量选择面板:`[处理个人事务]` / `[为客户处理业务 → 选择/新建客户]`。
**不在上传抽屉、ERP 推送抽屉、对账流程里再选 workspace**(原点 8)。业务功能一律继承右上角当前 workspace。

## 3. 功能分区(决定惰性提示对谁触发)

| 区 | 功能 | 个人模式可用? |
|---|---|---|
| 个人/全局 | 账号设置 / 充值 / 帮助 / 通知 / 超管后台 / 成本监控 / 客户管理 | ✅ 直接用 |
| 业务(需 workspace) | 上传/OCR · 银行对账 · 收入/销项对账 · ERP 推送 · 批次中心 · 报表 | ⚠️ 惰性提示选客户 |

实现:一个可复用守卫 `requireWorkspace()` 挂在业务功能入口(路由/按钮)· 个人模式且无 workspace → 弹轻提示;有 workspace → 直接进。

## 4. 0/1/多 workspace + 默认态(原 11/12/13 + 我的修正)

- **默认态(登录后)**:不弹任何阻塞框。恢复上次会话的 work mode + workspace(localStorage);没有记录 → 进**个人事务模式**(最轻 · 不打扰)。
- **进业务功能时**按 workspace 数量分流:
  - **0 个**:**老板/超管** → 引导"新建客户"(不死循环 · 可取消回个人模式);**员工** → "你还没有被分配客户,请联系老板"(员工无建客户权 · 见 §6 冲突)。
  - **1 个**:自动选中 · 不弹 · 直接进业务。
  - **多个**:弹轻选择面板让选(就是右上角那个 · 不是登录硬弹窗)。

## 5. 对原 13 点的"提升"建议

1. **work mode 持久化**:存 `localStorage`(`pearnly_work_mode` + `pearnly_active_workspace_client_id`)· 跨页不重复问 · 切客户即时刷新当前页数据。
2. **合并入口**:work mode 选择 + 选客户 + 惰性提示**复用右上角同一面板**,不做三套 UI(更少代码、更一致)。
3. **默认个人模式**而非默认弹选客户 → 真正"不封锁、不打扰"。
4. **惰性提示统一组件**:`requireWorkspace(onOk)` 一处实现,所有业务入口调用 · 文案 4 语。

## 6. 冲突点(必须先厘清,否则会踩坑)

1. **0 workspace 行为按角色分**:原点 11"引导新建客户"只适用**老板/超管**;**员工无权建 workspace 账套**(B3/B4 后端 `_require_owner_or_super` 已锁)→ 员工 0 客户只能显示"联系老板"。**不能让员工卡在'引导新建'死循环。**
2. **旧 ClientSwitcher 怎么办**:现右上角已有一个 ClientSwitcher(v118.28.0 · localStorage `pearnly_current_client_id`),它按 **clients=买方** 维度。新 work-mode 切换器是 **workspace 账套**维度。原点 9 要"唯一切换入口" → **新切换器应取代旧的**(旧买方切换器退役/移除)· 否则右上角两个"客户"控件并存会更乱。**这步要碰 home.js 旧控件 → 需确认。**
3. **发票买方 ≠ workspace**(原点 10):上传/识别出的买方仍按每张发票走,写 `history.client_id`;workspace 写 `history.workspace_client_id`(B1 相 1 已就绪)。两者绝不混。
4. **超管(Earn)**:只进 /admin,从不碰业务 UI → work mode 对其无意义(等同个人模式)· §3 已含。
5. **惰性提示 vs 硬校验**:本期只做**前端惰性提示**(B2)· 后端 B1 相 2 强校验是另一层(缺 workspace 拒绝)· 二者独立 · B2 不依赖 B1 相 2。

## 7. 与 B4 的关系(实现顺序)

B2 的"右上角控件"就是 **B4 切换器**的最终形态。所以:
1. 先接 B4 切换器到右上角(发 `X-Workspace-Client-Id`)+ work-mode 两态显示。
2. 再加惰性提示 `requireWorkspace()` 到业务入口。
3. 最后(确认后)退役旧 ClientSwitcher。

## 8. 红线(不变)
- 不在登录时封锁全站(本设计的根本)。
- 个人模式能正常用个人/设置/充值/帮助/通知/超管/成本。
- 不在上传/推送/对账流程里重复选 workspace。
- 发票买方按每张识别,不等于 workspace。
- 员工 0 客户不死循环(显示联系老板)。
- 退役旧 ClientSwitcher / 任何 home.js 改动前先确认 + 浏览器验 + 4 语 release_notes。
