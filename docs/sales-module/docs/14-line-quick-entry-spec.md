## 1. Paypers 的 LINE 流程(据客户 11 张截图逐步还原)

用户在 LINE 对 Paypers 发一句 `ค่าน้ำ 50`,Paypers:

1. 立刻回 `✅ บันทึกค่าใช้จ่ายเรียบร้อย!`(记账成功)
2. 回一张**结构化卡片**`✅ บันทึกค่าใช้จ่ายสำเร็จ`,字段:
   - จำนวนเงิน(金额)= **50 THB**
   - ประเภทเอกสาร(单据类型)= -
   - ประเภทค่าใช้จ่าย(支出类型)= 🛍 สินค้า(商品)
   - วันที่(日期)= 当天
   - หมวดหมู่(分类)= ค่าใช้จ่ายสำนักงาน(办公费用)
   - หมวดหมู่ย่อย(子分类)= ค่าน้ำ / ค่าไฟ(水电)
   - ธุรกิจ(业务主体)= บริษัท สบาย จำกัด
   - ผู้ขออนุญาตเบิก(报销申请人)= Patrakorn …
   - รายละเอียด(明细)= ค่าค่าน้ำ
   - ผู้ขาย/ร้านค้า(卖家)= -
   - 📄 ดูใบแทนใบเสร็จ(查看"收据替代凭证")
3. 点卡片 → web(app.paypers.ai)`แก้ไขรายจ่าย` 编辑页,三 tab:
   - **เอกสารรายจ่าย**:补传凭证(JPEG/PNG/WebP/HEIC/PDF ≤10MB)+ 自动生成的 ใบแทนใบเสร็จ PDF
   - **ข้อมูลรายจ่าย**:付款状态(已付/未付挂账)、开票日期、报销人、是否有 ใบกำกับภาษี、税号、卖家信息、总公司/分公司
   - **รายการค่าใช้จ่าย**:币种、行项目(明细/商品|服务/数量/分类/子分类/税前额/VAT)、汇总(税前/折扣/VAT/WHT/四舍五入/应付/备注)

**本质**:一句话 → 解析金额+品类 → 自动归类 → 落一条支出 → 生成可补全的凭证。单机闭环,导出了事。

## 2. 我们的目标体验(同口径 + 更智能)

用户对 Pearnly 发 `ค่าน้ำ 50`(或 `โค้ก 30 บาท`、`ข้าวกล่อง 2 80`、中文 `打车 120`):

1. 机器人回 Flex 卡片(对齐 Paypers 字段)+ 一句"已记账"。
2. **不确定就多轮反问**(Paypers 没有):缺卖家/缺税号/分类拿不准 → 机器人追问一句,用户答 → 补全。
3. 卡片带"在网页打开/编辑"深链 → 落进**凭证中心**草稿,可补传真凭证、改科目、推 ERP。
4. 录完自动进入**对账/查重/报税**链路(Paypers 完全没有)。

入口三态都要兼容(已在主仓库 `line_webhook_routes.py` 留好分发):
- 发**图/PDF** → OCR(已上线)
- 发**一句话** → 本篇引擎(待建)
- 发**语音** → STT → 转一句话走同引擎(P2)

## 3. 解析引擎(核心 · "一句话 → 结构化支出")

输入:自由文本(泰/中/英),口语、缩写、无标点。输出:`ExpenseDraft`。

**两层解析(快+准)**:
- **L1 规则层(秒回、零成本)**:正则/分词抽**金额**(`50`、`50 บาท`、`฿50`、`1,250.50`)+ **数量**(`2 จาน`)+ **关键词→分类**词典(`ค่าน้ำ/ค่าไฟ→水电`、`โค้ก/ข้าว→商品`、`แท็กซี่/打车→差旅`)。命中高置信 → 直接出卡。
- **L2 Gemini 层(兜底、补全)**:L1 不足或歧义 → 喂 Gemini(复用 OCR 管线的同一 client/计费)按 schema 出 `{amount, qty, category, subcategory, expense_type(goods/service), vendor?, date?, vat?, note}`。
- **多轮澄清**:仍缺关键字段(金额必填;卖家/税号可选)→ 机器人按缺啥问啥,会话态存 `line_pending_entry`(短 TTL)。

`ExpenseDraft` 字段(对齐 Paypers + 我们扩展):
```
amount_pretax, qty, currency=THB,
expense_type(goods|service), category, subcategory,
vendor_name?, vendor_tax_id?, doc_date(default today),
vat_mode(none|7%|included), vat_amount?, wht_amount?,
note, source='line_text', raw_text, confidence
```

## 4. 科目 / 分类体系

- 一套**进项支出分类树**(大类→子类,中泰双语),对齐泰国实务:办公费用(水电/文具/网络)、差旅、采购商品、服务费、餐饮等。
- 关键词→分类的**可学习词典**:用户改过一次的归类记下来(复用现有"字段记忆/纠错"思路),下次同词直接对。
- 落库时映射到**凭证中心**的借贷科目(凭证中心官方定义:AI 推荐借贷科目分录;本引擎产出分类 → 凭证中心出分录)。

---

## 5. 数据流 & 落库

```
LINE text → parse(L1/L2) → ExpenseDraft
  → (缺字段?) 多轮澄清
  → 落「凭证中心」草稿(status=draft, source=line_text)
  → push Flex 卡片(字段 + 编辑深链)
  → web 补全/审核 → 生成 ใบแทนใบเสร็จ PDF → 推 ERP
```

- **复用** `insert_ocr_history` 同款写入范式,但目标是凭证草稿表(凭证中心 schema,待建,见 `02-data-model.md`)。
- **多租户/账套隔离**:沿用 `workspace_client_id` 与 RLS(铁律红线 1/5)。
- **计费**:L2 Gemini 调用按现有 OCR credits 口径扣(L1 命中零成本)。

## 6. 回复卡片(LINE Flex Message)

对齐 Paypers 卡片字段(金额/类型/日期/分类/子分类/业务主体/明细/卖家)+ 两个按钮:
- `📄 ดูใบแทนใบเสร็จ`(凭证 PDF · 生成后可看)
- `✏️ แก้ไขบนเว็บ`(深链到凭证中心草稿)

4 语(th/zh/en/ja)。文案接 `services/line_binding/line_i18n.py` 现有体系(已留 `already_bound_hint` 等"发一句话记账"铺垫文案)。

## 7. 凭证 PDF(ใบแทนใบเสร็จ / 收据替代凭证)

- 模板化生成(复用 `services/excel`/报表引擎同思路,PDF 渲染)。

## 9. 超越 Paypers 的点(壁垒)

1. **多模态同口**:图/PDF/一句话/语音 → 同一支出引擎(Paypers 只手填 + 打字)。
2. **多轮澄清**:不清楚会问,而不是默默填错。
3. **税号直连税局验真伪 + 带 17 字段**(Paypers 手搜)。
4. **录完自动查重 + 与银行流水核销**(Paypers 无对账)。
5. **一键推 Mr.ERP / Xero**(Paypers 独立导出)。
6. **报税前拦错**(知识库规则引擎)。
7. **中/泰/英三语**(华人在泰做账客群;Paypers 泰语为主)。

> 一句话:Paypers = 录得快;Pearnly = 录得快 + 验真伪 + 自动算 + 自动校错 + 自动对账 + 自动入 ERP。
