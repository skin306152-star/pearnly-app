## 一、闭环逻辑:两种人,两条路

```
【A 新老板注册】注册 → onboarding(可跳过):
  ① 选业态 → 自动配模块(复用 onboarding-business.ts)
  ② 建首个账套主体(三分支):
     · 企业(有税号)→ 输 13 位 → 税局自动带出名/址 → 选 VAT 状态
     · 未注册小店(无税号)→ 手动填店名+地址 · VAT=否 · 仅开收据
     · 个人/freelancer → 个人主体 · 零税号 · 仅开收据
  ③ 账务设置(可跳过)→ 完成 · 开局清单(checklist)

【B 受邀成员】收到邀请 → 接受设密码(已做)→ 进来【不走建主体】:
  → 仅见"分配给我的客户"(老板在客户管理里分派)
  → 未分配 → 友好空态"等管理员分配客户"

【局内闭环】
  ④ 顶栏主体切换器 → 切当前做账主体
  ⑤ 公司资料页(新)→ 看/改完整信息 · 行内编辑不弹窗
  ⑥ 客户管理 → 列全部主体 + 新增 + 分派会计(=设 member_scopes)→ 闭环回 B
```

## 二、PRD

**目标用户**:① 新注册老板/个体(建主体) ② 事务所(管多客户) ③ 受邀会计(看分配)。
**痛点**:现注册后是"自动弹业态选择器"就结束,没有正经建主体/公司资料/客户总览的闭环;个人模式是脏兜底;没注册小店被税号挡。
**核心价值**:新用户前 5 分钟顺滑落地("aha"=建好主体能开票/记账);事务所一屏管多客户;成员各看各的客户。
**页面清单 / 状态**:
| 页面 | 入口 | 四态 |
|---|---|---|
| Onboarding 引导(业态→主体→账务→完成) | 注册后自动 | 载/可跳过/完成清单 |
| 主体登记(税号带出/手动填/个人 三分支) | onboarding ② | 带出成功绿卡/未带出/手动 |
| 主体选择(0/1/N 分流) | 多主体登录 | 0引导建/1直进/N选择页 |
| 公司资料页(行内编辑) | 局内导航 | 看/编辑/保存 |
| 客户管理(多客户+分派) | 事务所导航 | 列表/空/搜索无果 |
| 受邀成员视角 | 成员登录 | 已分配/未分配空态 |
| 手机端(主体选择/业态) | 手机访问 | 响应式 |
**权限矩阵**:建主体=owner(settings.workspace.manage);分派=owner/admin(team.member.edit_role + member_scopes);成员只读分配主体(作用域 fail-closed)。
**本期范围**:上述全部 + 个人模式迁移 + 税号带出 + 公司资料页。
**不做(P2)**:个人主体→企业的完整升级向导(本期只留入口);多级审批;真人 onboarding session。
**数据指标**:注册→建成首个主体转化率;onboarding 完成率(含跳过后回填);个人主体占比;事务所多客户渗透。

## 三、技术方案

- **库变更**(ensure 自愈,照 accounting 先例):`workspace_clients` 加 `subject_type`(company|personal,默认 company)+ `tax_id` 允空(个人/未注册)。个人主体 = subject_type=personal·tax_id 空·vat_registered=false。
- **税号带出**:新端点 `GET /api/workspace/tax-lookup?tax_id=` → 调 RD/税局(报税已对接 RD 方向,复用其凭证)返公司名+地址;查不到则降级手动填。
- **API**:onboarding 复用 `PUT /api/me/onboarding`(apply_preset)+ `POST /api/workspace/clients`(建主体,加 subject_type);新增 tax-lookup;公司资料读写复用 workspace_routes(补 PATCH)。
- **前端组件**:`src/home/onboarding-flow.ts`(向导壳·复用 onboarding-business 业态卡)+ `subject-create.ts`(主体登记三分支)+ `company-profile.ts`(公司资料页·行内编辑)+ 客户管理增分派(复用 workspace + member_scopes)+ 受邀成员分流(core-boot 判 role=member→不走建主体)。
- **🔴 老代码迁移(不推倒重做)**:
  - 个人模式 `work_mode=personal` → **数据迁移脚本**:存量该状态用户自动建一个 subject_type=personal 主体,历史数据(无 workspace_client_id 的)归入,**不丢**。
  - 删 `workspace-switcher.ts` 的 personal 分支(getWorkMode/enterPersonalMode/LS_MODE);全站"个人模式/未选→提示"(acct/inventory/pos/purchase-common 等)改"未选→引导选/建主体"。
  - 旧账套主体选择弹窗 → 替换为 0/1/N 分流。
  - 旧 onboarding 业态选择器保留,职责降级=只配模块,后接建主体。
- **⚠️ logo 暗夜(Zihao 提醒)**:logo 透明底,暗夜模式下必须垫白圆角底板——照 `home S2-bis` / `console html.dark .brand-icon` 先例(`html.dark .brand-logo{background:#fff;border-radius:8px;padding:2px}`),onboarding/侧栏/手机端所有 logo 出现处都要。
- **并发幂等**:建主体 uq;迁移脚本幂等(已迁过跳过)。**操作日志**:建主体/分派进 operation_logs。**性能**:tax-lookup 缓存+超时降级,不拖 onboarding。
- **测试**:onboarding 全流程 E2E(三分支+可跳过+完成清单);个人模式→个人主体迁移脚本(造存量数据验不丢);税号带出(mock RD);受邀成员分流(member 不走建主体);0/1/N 分流。
