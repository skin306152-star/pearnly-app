## ข้อสรุปสำคัญ · 核心结论

**Express ไม่ใช่แค่ "รับใบที่ OCR แล้ว" — มันคือระบบบัญชี DBF รุ่นเก่าเต็มรูปแบบ**
การจะทำให้คุณ "แทบไม่ต้องแตะมือ" ผู้ช่วยต้องทำงานจริงทุกขั้นแทนคุณให้ครบ ไม่ใช่แค่บันทึกหัวเอกสาร

*中文:Express 不是"只接 OCR 推送"那么简单,它是一套完整的老 DBF ERP。要让你无感,小助手必须把每个真实动作替你做全,不能只写单头。*

## ข้อสังเกตที่สำคัญที่สุด · 最关键的一点

> **เอกสารจริงเกือบทั้งหมดมี "รายการสินค้า" (STCRD)**
> ในชุดทดสอบ: ซื้อ 3,247 ใบ มีรายการ **3,145** ใบ · ขาย 1,743 ใบ มีรายการ **1,735** ใบ
> ดังนั้นถ้าทำแค่หัวเอกสาร (APTRN/ARTRN) ถือว่า **ไม่ครบ ไม่ตรงกับการใช้งานจริง**
>
> *中文:真实单据绝大多数都有明细。采购 3247 张里 3145 张有明细、销售 1743 张里 1735 张有明细。只做表头=偷懒,不符合真实使用方式。我们做的是「完整单据 + 明细 + 主数据 + 索引 + UI 可查」的硬闭环。*

## เราวางตำแหน่งผู้ช่วยไว้อย่างไร · 我们的定位

**ผู้ช่วย = "เครื่องมือรันงาน Express บนเครื่องคุณเอง" (Local ERP Execution Engine)**

- **Pearnly (บนคลาวด์):** อ่านเอกสาร · ตัดสิน · ให้คุณยืนยัน *เฉพาะรายการผิดปกติ*
- **ผู้ช่วย (ในเครื่องคุณ):** ลงข้อมูลเข้า Express · จัดเรียงดัชนี · ตรวจสอบ · รายงานผลกลับ
- ❌ ไม่ลงปลั๊กอินใน Express ❌ ไม่ต้องใช้ API ทางการ ❌ ไม่ต้องกด PACK ทุกวัน

*中文:小助手 = 你电脑上的「本地 ERP 执行引擎」。Pearnly 负责识别/判断/让你只确认异常;小助手在本地执行写入、索引整理、校验、回报。不装插件、不需官方 API、不用天天点 PACK。*

**ผู้ใช้ทั่วไปทำแค่ 4 ขั้น · 普通用户只需 4 步:**

1. เชื่อมต่อผู้ช่วย · 连接小助手
2. เลือกชุดบัญชี · 选账套
3. อัปโหลด / OCR · 上传 / OCR
4. ยืนยันรายการผิดปกติ · 确认异常
