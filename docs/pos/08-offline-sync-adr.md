# POS 项目 · 08 离线同步架构(ADR)

> 最高风险块。POS 现场断网必须能继续卖,联网无损同步、不双扣、号不乱。本文件钉死规则;施工严格照此,别现场发挥。
> 关联:幂等键见 03/04;离线态文案见 06;离线流程按钮见 05。

## 背景与目标
零售 POS 的硬要求:网抖/断网,收银**无感继续**。Pearnly 现为纯在线 SaaS,这是架构级新增。
目标:① 断网可持续开单整个班次 ② 联网自动补传、零丢单、零双扣 ③ 小票号合规(简式税票需连号)④ 不引入消息队列等重基建。

## 决策总览(ADR 列表)

### ADR-1 · PWA 外壳 + 数据快照
- /pos 做成 PWA:`manifest.json` 可装桌面;`service worker` 缓存 app 外壳(HTML/JS/CSS)。
- 登录后 `GET /api/pos/bootstrap` 拉**数据快照**存 IndexedDB:商品+单位+价格+库存数、模块/能力块、终端、设置、当前班次。
- 断网时 UI、选品、算价全用本地快照 → 完全可用。

### ADR-2 · 本地存储 = IndexedDB(持久 outbox)
- `outbox`:待传的小票/库存流水(带 `client_uuid`)。**持久化**——关页面/崩溃/重启不丢(绝不放内存)。
- `cache`:商品/库存快照、班次。
- 写小票 = 先写 outbox + 本地扣缓存库存 + 立即出小票(不等网络)。

### ADR-3 · 小票号 = 终端分段连号(避免"联网重编号")
**问题**:简式税票要连号,但离线发不了全局号。
**决策**:**每个终端独立号段**,前缀含终端码 → 离线生成的号**就是正式号**,联网不重编。
- 格式:`<前缀>-<终端码>-<年>-<连续号>`,如 `RCP-T01-2026-000123`。
- 每终端本地维护自己的连续号(IndexedDB),离线照发;联网时服务端校验该终端号段无冲突即接收。
- 好处:小票打出去的号 = 最终号,顾客拿到的号不会变,合规连号按"终端内连续"成立。退货/查单用此号稳定。
- 备选(否决):全局号离线占位、联网重编 → 顾客手里号会变 + 重编复杂,弃。

### ADR-4 · 金额计算:本地预算 + 服务端权威复算
- 本地用与 `totals.py` **同一套规则**(价内/价外/VAT/折扣)算给收银看 + 出小票。
- 同步时服务端**用 totals.py 复算**;与本地不一致 → 以服务端为准 + 标记该单 `recompute_mismatch`(告警,极少发生)。
- 防分歧:本地算法以 03/`totals.py` 为唯一规格;加"本地 vs 服务端"等价测试(09 施工规约列为必测)。

### ADR-5 · 库存:乐观本地扣 + 服务端幂等应用 + 允许负库存告警
- 离线:从本地缓存库存乐观扣;卖光按设置(默认**允许继续卖**,零售丢单比超卖更糟)。
- 联网同步:服务端按 `client_uuid` 幂等写 `inventory_transactions`(重复补传不双扣)。
- **超卖处置(决策)**:多终端离线都卖最后一件 → 服务端库存可能转负;**允许负 + 给老板告警**(库存页红标"负库存,请核查/补货"),**不退单**。理由:已收钱的单不能因库存回滚而作废。
- 备选(否决):离线预留/锁库存 → 离线无法跨终端协调,弃。

### ADR-6 · 同步引擎(联网即冲)
- 触发:`navigator.onLine` 恢复 / 定时心跳 / 手动。
- 动作:按 `sold_at` 顺序读 outbox → `POST /api/pos/sales/sync`(批量)→ 按 `client_uuid` 匹配结果 → 成功移出 outbox、回填服务端确认;失败留 outbox 退避重试(2s,4s,8s…上限)。
- 部分失败不卡其余(04 sync 逐张结果)。UI 角标显示"待同步 N 单"。

### ADR-7 · 鉴权:离线 token TTL 覆盖班次
- 收银员 PIN 登录签发 token 带较长离线有效期(`offline_ttl_hours`,默认 12h ≥ 一个班次);离线期间无法重新鉴权,故 token 必须活过离线窗口。
- 离线不验 token 撤销(联网后心跳校验);班次结束/超时需联网重登。

### ADR-8 · 缓存新鲜度
- 离线期间商品/价格变更不可知 → 接受短时陈旧(日内改价罕见)。
- 联网时同步**先拉新快照再冲 outbox**;已离线售出的按售出时价(行上已冻结单价),不追溯改价。

### ADR-9 · 时间
- `sold_at` = 设备本地时间(离线成交时刻,出现在小票/报表);`synced_at` = 服务端入库时间。
- 时钟漂移风险:信设备 `sold_at`,服务端额外记接收时间;偏差过大(>班次)标记审计。

## 离线一笔的完整生命周期
```
断网 → 收银点"确认收款"
  → 本地:生成 client_uuid + 终端连号 + totals 预算
  → 写 outbox(持久)+ 本地扣缓存库存
  → 立即出热敏小票(用终端连号)
  → UI:成交,"待同步+1"
联网 →
  → 拉新快照
  → POST /sales/sync(outbox 批量,按序)
  → 服务端:client_uuid 去重 → totals 复算 → FEFO 扣库存(幂等) → 落库 → 回执
  → 本地移出 outbox;不一致标记告警
```

## 风险 → 缓解
| 风险 | 缓解 |
|---|---|
| 重复补传双扣 | `client_uuid` UNIQUE 幂等(03/04) |
| 关页面丢未传单 | outbox 持久化 IndexedDB,启动即恢复 |
| 多终端超卖 | 允许负库存 + 告警,不回滚已收款单 |
| 本地/服务端金额分歧 | 服务端权威复算 + 等价测试 + mismatch 告警 |
| 号冲突 | 终端分段连号,联网仅校验不重编 |
| 时钟漂移 | sold_at 设备 + synced_at 服务端双记 |
| token 离线过期 | offline_ttl ≥ 班次时长 |

## v1 范围 / 暂不做
- **做**:离线开单/收款/出小票/扣库存/联网同步/挂单(本地)。
- **暂不**:离线退**未同步**单(先要求联网退;或仅本地撤未传单)、跨终端离线库存协调、离线升级正式税票(税票需联网连号,离线只出小票,联网后再升级)。

## 施工与验收锚点
- 施工:service worker + IndexedDB outbox + 同步引擎 + 本地 totals(对齐服务端) 各带测试。
- 验收(10):飞行模式下连开多单→出小票→关页面重开 outbox 仍在→恢复网络→断言全部同步且库存只扣一次、号不冲突。
