## 素材分两类

### A. 客户明确开口要的(翻译 App 截图,泰↔中)

- "你了解销售点(POS)系统吗?" / "想看 POS 界面" —— 在试探我们能不能做销项。
- 商品录入要 **4 种方式**:① 手输商品代码 ② 扫 bar code ③ 扫 qr code ④ 从照片选。
  现状只支持手输。参考 **MK 等餐厅点单屏**的交互。

  > "我想做一个**类似 Mr.ERP 销售点系统的开收据/发票功能**,但能用 4 种方式选要卖的商品。"

### B. 竞品 Paypers(app.paypers.ai)截图(11 张,UX 参照)

注意:这 11 张全是 Paypers 的 **`แก้ไขรายจ่าย`(编辑「支出」= 进项)** 界面,不是销项。
客户拿它当**界面/体验样板**,不是要我们再做一遍进项(进项我们已有 OCR)。可提炼的形态:

- **三 Tab 表单**:单据(传文件)/ 信息 / 费用明细。
- **传凭证**:拖拽或选文件,≤10MB,JPEG/PNG/WebP/HEIC/PDF。
- **对方信息**:按名称或税号搜索、地址、总公司/分公司(สำนักงานใหญ่/สาขา/ไม่มีสาขา)。
- **税务字段**:税前金额、折扣、VAT(有/无 7%)、WHT 预扣税、四舍五入、应付额。
- **分类 + 子分类**(办公费用 / 水电费…)。
- **付款状态**:已付 / 未付先挂账(ตั้งหนี้)。
- 生成 **ใบแทนใบเสร็จ(收据替代单)PDF**。
- **★ LINE Bot 极速录入**:在 LINE 发 "ค่าน้ำ 50"(水费50)→ 机器人自动记一笔 + 回结构化卡片
  (金额/分类/日期/公司/审批人/明细 + "查看单据"按钮)。

## Zihao 的澄清(2026-06-04)

1. 客户要的是**上面 B 类那种能力我们也要有**(选项②):不止销项,还要 Paypers 式轻量录入/LINE 记账铺到全局。
2. **而且要比 Paypers 更智能**。我们的差异化牌:Gemini 提取 / RD 查税号真伪 / 知识库规则校错 /
   自动对账 / 一键推 ERP / 中泰英三语。详见 `docs/05-competitive-paypers.md`。
3. 关键认知:Paypers 录入靠**手填 + LINE 打字**;我们录入靠**拍照 OCR**。客户可能真正羡慕的是
   "LINE 一句话就记一笔"的轻便——这是我们现在**没有**的,值得专门确认是否要补(见 `docs/09` 最痛点题)。

## 范围边界(避免误解)

- 销项(收钱开票)与进项(付钱记账)是会计**相反两个方向**,不是同一功能;但共用同一套引擎/界面。
- 销项税票有**法律级**约束(连号不跳、开出不可改、红冲走 ใบลดหนี้/ใบเพิ่มหนี้),进项记录没有——
  这直接影响数据模型不可变性,见 `docs/03` 与 `docs/02`。
