# 15 · 销项发票 · 买方信息模块规格(加强版)

> Zihao 2026-06-06 拍板 · 下个窗口实现。当前状态:**未做**。现开票买方仅 `clients` 表
> (名称/税号/地址),无买方类型、无身份证/护照/匿名区分、无总公司/分店、无切换状态机。
> 本规格是净新增,落地到 PO-4 开票表单 + PO-10 前端 + 买方数据模型。

## 1. 核心设计原则

泰国发票买方信息**不是固定表单,而是随「买方类型」联动的动态表单**。一刀切同时犯两个错:
对公司漏掉「分店标识」(缺第 13 项强制字段 → 发票无效),对个人硬塞「分店」(个人无分店概念
→ 误导)。**第一步必须让用户选买方类型。**

## 2. 买方类型选择器(放买方区块最顶部)

```
买方类型 / ประเภทผู้ซื้อ:
  ( ) 公司 / นิติบุคคล
  ( ) 个人 / บุคคลธรรมดา
  ( ) 外国个人 / ชาวต่างชาติ
  ( ) 匿名散客 / ลูกค้าทั่วไป (ไม่ระบุ)
```

默认选中 **「公司」**(B2B 开票主场景)。选项变化 → 下方字段实时显隐 + 校验规则切换。

## 3. 字段联动表(实现按此表)

| 字段 | 公司 | 个人 | 外国个人 | 匿名散客 |
|---|---|---|---|---|
| 名称/姓名 | 必填 | 必填 | 必填 | 选填 |
| 地址 | 必填 | 必填 | 必填 | 选填 |
| TIN 标签 | 公司税号 13 位 | 身份证号 13 位 | 护照号 | — |
| TIN 校验 | 13 位数字 | 13 位数字 | 字母数字 4–20 位 | 留空 |
| 总公司/分店 | 必填(默认 HQ) | 隐藏 | 隐藏 | 隐藏 |

**关键易错点(务必传达给开发):**
1. 「总公司/分店」只对「公司」**渲染**;个人/外国/匿名一律**不渲染**(不是禁用)。
2. TIN 三类买方同一个 13 位框,但 label + 校验不同:公司=公司税号、个人=身份证(后端都 13 位数字);外国人放宽成护照格式。
3. 卖方(账套主体)的 HQ 标识照旧保留(见 PO-6),**别和买方逻辑混用**。

## 6. 校验与状态文案(四态 · 中泰双语)

- **TIN 位数错(个人/公司)**:中「身份证号/税号需为 13 位数字」· 泰「กรุณากรอกเลข 13 หลัก」
- **匿名散客留空姓名(warning · 非阻断)**:中「未填买方信息,此发票买方将无法用于抵扣进项/个税」· 泰「ไม่ระบุข้อมูลผู้ซื้อ จะไม่สามารถใช้ขอคืนภาษีได้」
- **个人买家成功填好(hint)**:中「个人买家可用身份证号作为税号」· 泰「บุคคลธรรมดาใช้เลขบัตรประชาชนเป็นเลขผู้เสียภาษีได้」
- **公司缺分店标识(error · 阻断)**:中「请选择总公司或填写分店号(泰国完整税务发票必填项)」· 泰「กรุณาระบุสำนักงานใหญ่หรือสาขา」

## 7. 合规依据(留档)

- 完整税务发票 13 项强制字段:Revenue Code §86/4,含买卖双方总公司/分店标识(仅法人适用)。

# 买方类型切换 · 字段状态机规格

## 1. 字段数据模型(后端/状态层统一字段名)

```js
buyer = {
  type:        "company" | "individual" | "foreigner" | "anonymous",
  name:        string,
  address:     string,
  taxId:       string,   // 公司税号 / 身份证号 / 护照号 共用此字段
  branchType:  "hq" | "branch" | null,   // 仅 company 有意义
  branchNo:    string | null,            // branchType==="branch" 时有效
}
```

**关键**:`taxId` 一个字段承载三种语义,由 `type` 决定 label 与校验。**不要为每种类型建独立字段**——否则切换时残留更难清。

## 2. 每种类型的字段配置(声明式 · 实现成一张配置表,别写散落 if-else)

```js
CONFIG = {
  company:    { name:req, address:req, taxId:{show:true, validate:"th13"},    branch:show },
  individual: { name:req, address:req, taxId:{show:true, validate:"th13"},    branch:hide },
  foreigner:  { name:req, address:req, taxId:{show:true, validate:"passport"},branch:hide },
  anonymous:  { name:opt, address:opt, taxId:{show:false},                    branch:hide },
}
```

UI 显隐、必填星号、校验器全部从这张表读。切换类型 = 换一行配置重渲染。新增类型只改表,不动逻辑。

## 3. 切换时的清空/保留规则(核心 · 防脏数据)

| 字段 | 规则 | 理由 |
|---|---|---|
| name | 保留 | 名字跨类型通用,切错了不该丢 |
| address | 保留 | 同上 |
| taxId | **清空 ⚠️** | 公司税号 ≠ 身份证 ≠ 护照,语义变了,残值会被误当合法 13 位 |
| branchType | 切到非 company → null;切回 company → 重置默认 "hq" | 个人无分店概念 |
| branchNo | 切到非 company 或 branchType≠branch → 清空 | 防隐藏字段携带脏值提交 |

**最容易漏的两条:**
1. **taxId 必须清空**。用户先填公司税号,改选「个人」,若不清空,13 位公司号会通过「身份证 13 位」校验,混进个人发票。
2. **隐藏字段必须清空底层值,不能只 `display:none`**。否则提交时把上一类型的 branchNo 一起发后端。

## 4. branchType 子状态机(仅 company 内部)

```
[hq] ──选"分店"──> [branch] : 启用 branchNo 输入框, branchNo 必填(5 位数字)
[branch] ──选"总公司"──> [hq] : 清空 branchNo, 取消其必填校验
进入 company 默认态 = [hq]
离开 company = branchType→null, branchNo→""
```

## 5. 校验触发时机

- 切换 type 后:**立即清掉旧的校验错误标记**,不要对刚清空的字段立刻报「必填」(用户还没开始填)。
- 字段 blur 时校验单字段;提交时全量校验。
- 校验器映射:
  - `th13` → `^\d{13}$`
  - `passport` → `^[A-Za-z0-9]{4,20}$`
  - `branchNo`(branch 态) → `^\d{5}$`

## 6. 提交前归一化(保险层 · 后端也应再做一次)

```
if type != "company":   branchType = null; branchNo = null
if type == "anonymous": 允许 name/address/taxId 为空
taxId = taxId.trim()
```

## 7. 一句话验收标准(给测试窗口)

> 在「公司」填满(含税号+分店号),切到「个人」,再切到「匿名」,再切回「公司」——
> taxId/branchNo 不得残留任何上一类型的值;姓名地址保留;分店号控件随类型正确显隐;
> 切换瞬间不冒红字必填错误。
