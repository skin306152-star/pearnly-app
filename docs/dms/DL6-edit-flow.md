# DL-6 · LINE 确认卡逐字段修正（แก้ไข）

OCR 误读的人防封口（实案：泰文姓氏尾梢 นารถ 被读成 นาก，错值确认后入库）。

机制一句话：**改源值，整体重跑**——新建卡/差异卡各有 [แก้ไข] → quick reply 选字段
（姓名/证号/生日/电话/门牌/หมู่/ซอย/ถนน）→ 逐字段校验（证号 mod-11、生日真日历、
电话 0 开头 9-10 位）→ 写回会话草稿 → 重跑查重（`flow._run_dedup`）出新卡新 nonce。
天然覆盖新建(A)/更新(C)两分支与「证号改了→查重换分支」。

- 代码：`services/line_dms/edit_flow.py`（独立模块）+ cards/flow 接线。
- nonce 语义：编辑入口只校验不消费；写档仍走 `store.consume_nonce`。
- 地理四级 ID 不开放 LINE 直改，地址文本改后走既有级联匹配重解。
- 三层防线中的第 2 层（L1=引擎 A 档任务级覆写，L3=iApp 异构双读·挂起待客户确认）。
