# 01 · LINE 模块菜单 + 交互设计(方案待 Zihao 确认 → 再施工)

> 用户怎么在 LINE 里"先选要做什么、再让机器人去做",以及"双轨 + 菜单兜底防跑偏 + 没开模块怎么办"。
> 状态:**方案稿**,Zihao 确认后才施工。先文档后码(LINE 主路径)。

---

## 0. 一句话目标

老板打开 LINE 就能干活:**想点就点(菜单按模块选)、想发就发(直接拍票/打字 AI 自动判)**,
两条路并存。机器人永远先说清"我能为你做什么、你可以发什么给我",做完只说真发生的事。
**不要求先去网页开通模块**——LINE 菜单本身就是模块入口。

---

## 1. 核心原则:双轨交互(又智能又防跑偏)

老板最大的焦虑 =「我功能多,用户想做 A,机器人却做成了 B」。解药 = **两条路并存**,不是二选一:

| 路 | 谁用 | 流程 | 防跑偏机制 |
|---|---|---|---|
| **A · 显式(菜单先选)** | 怕做错 / 功能多记不住 / 新手 | 点菜单选车道 → 机器人进入该车道上下文 → 引导"发什么" → 用户发 → 机器人只在该车道内理解 | **选了=锁定意图**,AI 只在这条车道判 3-5 个动作,几乎不跑偏 |
| **B · 隐式(直接发)** | 图省事 / 老手(拍票就走) | 直接发照片/一句话 → AI 自动判车道+意图 | 高把握直接做(可撤销);**没把握才问一句**澄清,绝不硬猜 |

**车道上下文(关键新机制)**:点菜单选了"记账"后,在短时窗口内(如 5 分钟 / 直到切换),
用户发的内容**默认按"记账"理解**——即使一句话模糊,也不会被判成"查客户"。切车道 = 再点菜单
或明确说"我要查客户"。技术上 = `conversation.pending` 存当前 lane(已有基础设施,扩一个字段)。

---

## 2. Rich Menu(L0)= 按模块排 + 每个带说明

现状菜单是通用的(使用说明 / 联系客服 / 官网 / 注册)——**改成按模块**。LINE Rich Menu 一屏约 6 格,
够放核心模块;更多走"⋯ 更多"二级。每格 = 图标 + 模块名 + 点开后机器人一句"能为你做什么"。

**首屏 6 格建议**(按 SMB 老板日常高频排):
1. 📸 **拍票记账** — "拍张票或打一句(咖啡60),我帮你记账+分类。"
2. 📊 **查账/报表** — "问我'本月花多少 / 赚多少 / 上一笔',我真查给你。"
3. 👥 **客户(DMS)** — "拍身份证或发客户名,我帮你建/查客户、开订车单。"
4. 🧾 **报税** — "问我'这月要交什么税',我看你的账帮你算 PP30/PND 草稿(报前你复核)。"
5. 🏦 **对账** — "发银行对账单,我帮你和账本勾对、标出差异。"
6. ⋯ **更多** — 开票/库存/收款/团队审批/集成导出/设置。

> 注:菜单**按租户已开通+有权限动态显示**(没权限的不显示)。核心"记账+查账"对所有人常驻。

---

## 3. "没开模块怎么办" —— 三条规则(取代旧的"掉老文字")

1. **核心车道(记账/查账)对所有账号常开**——这是 LINE 的立身价值,不设门槛。
2. **未开通的高级模块**(如报税/对账/DMS):菜单上**要么不显示,要么点了当场引导**——
   一句话 + 一个按钮「✅ 开通报税(免费/去网页)」,**绝不"发了没反应"或吐老文字**。
3. **直接发内容但该能力没开**:机器人**先识别**(看懂是什么),再说"这看着是张银行对账单,
   你还没开对账,要开吗?"——**先帮看懂 + 给出路**,不静默、不甩锅。

→ 这彻底删掉"必须先在产品开通模块、否则掉老文字"的旧偏差。

---

## 4. 每条车道的对话模板(机器人"能为你做什么 + 你发什么 + 我去做")

统一三段式:**① 点菜单 → 机器人开场白(能力说明)② 用户发 → ③ 机器人做 + 出卡/回执**。

**记账**(参考实现,已上线):
- 点「拍票记账」→ 机器人:"📸 拍一张票,或直接打一句('咖啡 60' / '昨天加油 1000'),我帮你记账并分类。多张票一次发也行,我一张张处理。"
- 用户发票/句子 → 出四态数据卡(已入账绿 / 请确认琥珀 / 需补全灰 / 可能重复红)+ 可撤销/改。

**查账**:
- 点「查账」→ "📊 想知道什么?试试:本月花多少 / 本月赚多少 / 餐饮花了多少 / 上一笔。"
- 用户问 → 真查 DB(绝不让 AI 编数字)+ 迷你分类 Top3。

**客户(DMS)**:
- 点「客户」→ "👥 拍客户身份证我帮你建档/查重;或发客户名我帮你查。要开订车单也可以。"
- 用户拍证 → 识别 + 查重 + 可编辑卡 → 推送建/改客户(已上线能力)。

**报税**(严守三态文案):
- 点「报税」→ "🧾 我看你的账帮你算这期 PP30(增值税)/ PND(代扣税)**草稿**,报前你复核。**我不会替你假装报税成功**。"
- 用户确认 → 生成草稿 + 规则校验 → 提示截止日 → 真提交且拿回执才显"已申报"。

## 5. 主动智能(老板没问、机器人主动帮 —— "帮你多想"的设想池)

被动等用户发 = 普通工具;**主动推送 = 真助理**。按价值排(都先做成可开关,默认保守):

- **🔔 报税截止提醒**:PP30/PND 截止前 N 天主动提醒 + 一键看草稿。(对标会计师价值)
- **⚠️ 异常/重复票提醒**:识别到重复票、金额异常、缺税号(影响抵扣)→ 主动标给老板。
- **📅 每日/每周经营摘要**:"今天:卖 ฿X(N单)· 花 ฿Y · 净 ฿Z。本周对比上周 ↑/↓。"——老板每天一眼看生意。
- **🧑‍🤝‍🧑 费用审批**:员工 LINE 拍票 → 老板 LINE 收到"小王报销 ฿500 餐饮,✅批/✏️改/❌驳"。
- **🏦 对账差异**:上传对账单后,差异项主动列出 + 引导处理。
- **📦 低库存/近效期**(零售/餐厅):快没货/快过期 → 提醒补货。
- **🎙️ 语音记账**:老板发语音"昨天加油一千" → 转文字 → 记账(开车/忙时不用打字)。
- **💬 经营问答**:"我这月能抵多少进项税 / 哪个客户欠款最多" → 真查账回答(带数据,不编)。
- **📤 月底一键打包**:"帮我把这月的票导出给会计" → Drive/Sheets 归档包 + 链接。

> 这些不一次性全做。先把**记账车道做透**,再按价值逐条加(每条 = 框架里一条新车道/一个通知,
> 注册一行、不动其它)。主动通知尤其要克制:宁可少推、推得准,别变骚扰。

## 6. 这套是"更复杂"还是"更简单"?(客观结论)

- **对用户:更简单 + 更安心**。想省事直接发(一步到位);怕做错点菜单选(锁定不跑偏)。比"全靠 AI 猜"安全,比"每次都先点菜单"方便。
- **对你:不是推倒重来**。方向(`00`)没变,本页只是把交互讲细 + 删掉"先开模块"的旧偏差 + 列出主动智能的设想池。
- **行业验证**:微信服务号 / LINE 泰国超级 App / 各大银行 LINE OA 都是"Rich Menu 菜单 + 自由对话"双轨。我们是收敛成熟做法,不是发明。

---

## 7. 落地顺序(确认后)

1. **删旧偏差**:去掉"未开 expense 模块 → 掉老文字"的尾巴;核心车道对所有人常开;未开高级模块 → 引导而非静默。
2. **Rich Menu 改按模块** + 每模块开场白(LINE OA 控制台 + 文案)。
3. **车道上下文**:菜单点选后短时锁定 lane(`conversation.pending` 扩字段),防跑偏。
4. **逐条加主动智能**(报税提醒 / 异常票 / 日报 优先),每条做成可开关、保守默认。
5. 全程守 `00` 的不可破边界(三态诚实、金额不信 LLM、风险分层)。
