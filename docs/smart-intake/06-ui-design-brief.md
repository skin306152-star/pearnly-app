# Pearnly 进项采购模块 · UI 设计需求(交付外部设计 AI · 自包含)

> 把这整篇贴给设计 AI 即可。它无需任何其他上下文。产出 = 可交互 .html。

---

## 0. 任务

为一个**泰国中小企业财税 SaaS(Pearnly)**的「**拍单据 → 录入采购/进项 → 复核 → 外流**」模块,设计**可交互的前端原型(纯 HTML/CSS/JS,自包含,可直接双击打开)**,覆盖 **桌面端 + 手机端**,**所有交互必须能点能动**(不是静态图)。

- 用途:商户/会计拍一张供应商票据(发票/收据)→ 系统识别 → 用户复核字段 → 入账 → 可导出归档。
- 风格目标:**专业、克制、留白充足**,对标 Stripe / Linear 那种干净通透的后台,**不要拥挤、不要花哨、不要"AI 生成感"**。
- **必须严格使用第 1 节的设计令牌与全局规则。** 布局可在规则内自由发挥(尤其双端排布),但视觉元素不可偏离。

### 1.2 全局规则(硬性)
1. **禁黑底**:主操作按钮 = `--accent` 紫色,**不许用黑色/深色块做按钮或背景**。
2. **图标必须线性(描边 outline)SVG**,`stroke:currentColor; fill:none; stroke-width:1.8`。**禁用 emoji 当图标。** 用内联 `<symbol>` sprite。
3. **留白充足**:间距走 4px 倍数尺(8/12/16/24…);宁松勿挤。
4. **圆角**:面板/卡片 16px,控件(输入/按钮)10–12px,胶囊/标签 999px。
5. **阴影**:克制柔和 `0 1px 2px rgba(40,30,80,.05), 0 2px 8px rgba(40,30,80,.04)`。
6. **弹层用居中 `.modal`**(桌面)/**底部抽屉式 sheet**(手机)——**禁桌面用侧滑抽屉(drawer)**。
7. **四态**:任何列表/数据区都要有 加载中 / 空态 / 错误+重试 / 正常 四种状态的样子。
8. **字号层级**:正文 13.5px;区块标题 16px/700;页面标题 19–22px/700;北极星数字 30–34px/800;标签 11–12px。
9. **状态色用法**:绿=可抵扣/成功,琥珀=需确认/代扣,红=必填错/删除。字段三态着色见下。

## 2. 双端规则(手机优先,桌面是其响应式展开)

- **整体框架**:
  - **桌面**:左侧固定导航栏(~230px,含 logo + 分组菜单)+ 右主区(顶栏 + 内容)。
  - **手机**:导航收进**汉堡抽屉**(点击左上汉堡滑出 + 半透明遮罩);主区单栏。
- **请用一套响应式实现**(同一内容随宽度重排),并在原型顶部放一个「📱手机 / 🖥️电脑」切换器,让人对比两端。手机视口约 390px 宽(可套手机外框),桌面约 1100–1200px。
- 关键双端差异(务必照做,见各屏):
  - **采集屏**:桌面**只有"上传文件 / 拖拽 + 手工录入"**(电脑没摄像头,**禁出现"拍照"**);手机才有「拍照 / 相册 / 文件 + 手工录入」。
  - **编辑/复核屏**:桌面 = **左侧票据图(固定/跟随滚动)+ 右侧字段表(信息卡 + 明细卡 竖排)**;手机 = **三段连续滚动 + 顶部 Tab 吸顶**,且**滚动时对应 Tab 自动高亮(scroll-spy)**,点 Tab 平滑滚到该段。

## 3. 屏清单 + 逐屏规格

共 7 屏,顶部可切换:① 列表 ② 编辑/复核(核心)③ 采集入口 ④ 待归类 ⑤ 外流面板 ⑥ 集成中心 ⑦ LINE 结果卡(渠道参考)。

### ① 列表页(进项采购 Dashboard)
- **用途**:一页看清本月花费 + 进入每张单据。
- **从上到下**:
  1. 页头:标题「采购 / 进项」+ 副标题。
  2. **北极星卡**:一个大数字「本月花费」+ 旁边小字 进货 / 费用 / 可抵进项税(绿);右侧动作:主按钮「记一笔采购」+ 一个「⋯」更多(里面是"导出/归档")。
  3. **筛选条**:分段(全部 / 进货 / 费用 / 未付)+ 一排筛选 chip(日期 / 文档类型 / 付款状态 / 分类),每个 chip 点开是**多选下拉(带勾选框)**;右侧「票面日期 ⟷ 上传日期」开关。
  4. **按月分组列表**:组头(月份 + 笔数 + 右侧合计金额 + **可折叠箭头** + **全选勾选框**);组内每行:**行首勾选框** + 日期 + 小图标 + 供应商名(+下面一行小字:来源/类别/单据类型)+ 右侧金额(下面小字 VAT)+ 付款状态徽章(已付绿/未付琥珀/部分紫)。
  5. 分页:上一页 / 显示 X–Y 共 N / 下一页。
- **手机**:北极星竖排、动作按钮整行、筛选 chip 换行、列表行折行堆叠。

### ② 编辑 / 复核屏(核心,最详尽)
- **用途**:一张票进来,左看原图、右逐字段核对修正,存草稿或入账。
- **桌面**:左 = 票据图卡 + 凭据卡;右 = 信息卡 + 明细卡 + 汇总卡(竖排常驻,不分 Tab)。
- **手机**:三 Tab「文档 / 信息 / 明细汇总」,**吸顶 + scroll-spy**。
- **顶部**:返回 + 标题「复核单据」+ 副标题(供应商·金额·方向);右侧(桌面)动作:存草稿 / 确认入账(紫);**需复核横幅(琥珀)**:"有 N 处需要您确认:〈字段〉",点字段名平滑滚动并高亮该输入框。
- **票据图卡(文档区)**:深底浅灰、白纸浮起;**支持鼠标拖拽平移 + 滚轮缩放**;工具按钮:放大/缩小/旋转/复位,实时百分比;**多文件 1/N**(顶部翻页 + 下方缩略图条 + 「+ 附文件」);角标「智能识别 N 项」。
- **凭据卡**:按钮「生成替代收据(无票时)」+「经手成员」输入。
- **信息卡字段**(逐个):
  - 文档类型(分段:进货发票 / 费用单 / 采购订单)
  - 供应商(可搜+新建)
  - 税号(+ 旁边「RD 核验」按钮)
  - 分店类型(下拉:总部 / 分店 / 无分店)——**选"分店"时,"分店代码"变必填**
  - 分店代码 *、分店名(并排)
  - 地址
  - 出票日期 *、单号(并排)
  - 有无税票(开关)——**开则展开"税票号"输入**
  - 付款状态(分段:已付 / 未付·先挂账)
  - 币种(下拉)——**选非 THB 时展开"汇率"输入**
- **明细卡**:
  - "拆分多条 ↔ 合并记一条"二选一(默认拆分)
  - 提示条「按分类分别记账,便于出报税表」
  - 明细行(可多行):品名 + 「商品/服务」分段 + 数量 + 单价 + 大类(下拉)+ 小类(下拉,随大类联动)+ VAT% + 每行「折扣」开关 + 删除图标;底部「+ 加一行」(真能加/删)。
- **汇总卡**:税前小计 / 折扣 / 进项VAT(绿"可抵")/ WHT 代扣(琥珀)/ 四舍五入 / 应付净额(大字)——只读;**右上「手动改额」开关**:打开后允许直接改 小计/折扣/VAT/合计 去对齐票面,并显示提示「以票面为准」。
- **字段三态着色**:绿底=已识别可信(带"已识别"绿标)/ 琥珀底=请确认(带"请确认"琥珀标)/ 白=普通;另有"学过你的习惯"紫标用于分类。
- **底部动作**:删除(红边)/ 存草稿 / 确认入账(紫,主)。
- **硬必填校验**:点「确认入账」时,若必填项(分店代码[选分店时]/出票日期/税票号[勾有税票时])为空 → 弹琥珀报错横幅列出 + 对应输入框红框红字 + 点横幅条目跳转聚焦。

### ③ 采集入口
- **用途**:把票丢进来。
- 顶部只读提示「记账给:〈当前主体〉·换主体用顶栏切换」。
- 大块拖拽上传区(文案:拖入或点击选择文件;支持 JPG/PNG/HEIC/PDF ≤10MB 可多张)。
- **桌面**:下方「上传文件」「手工录入」两个按钮(**无拍照**)。
- **手机**:「拍照 / 相册 / 文件」三个 + 「没有发票?手工录入」。

### ④ 待归类
- **用途**:系统拿不准的票,人一点就归位。
- 列表项:缩略图 + 卖家/金额/日期 + "智能猜测:进项票/拿不准"标 + 一排动作按钮(记进项[紫主]/记费用/这是销项/这是银行单/忽略[琥珀])。

### ⑤ 外流面板
- **用途**:导出/归档到 Google。
- 一个收拢面板,三件事:「导出 Excel(免授权)」「归档到 Drive」「同步 Sheet」;提示「Drive/Sheet 未连接 → 点击自动跳集成中心授权;凭据按主体隔离」。
- 下方展示(只读示意):归档目录树(主体/年/月/{证据原图、交会计PDF})+ Sheet 列清单(一行=一条明细,末尾几列标绿=借贷分录,系本产品独有)。
- 交互:点「归档/同步」若未授权 → 跳到 ⑥ 集成中心并高亮 Google 卡。

### ⑥ 集成中心
- **用途**:连接外部服务。
- 卡片网格:**Google Drive/Sheets**(未连接 + 「连接 Google」按钮,点击演示变"已连接")· **LINE 官方账号**(已连接·智能助手定位)· Xero(未连接)· 邮件抓票(未连接)。每卡:线性图标 + 名称 + 状态徽章 + 动作按钮。

### ⑦ LINE 结果卡(渠道参考·手机内)
- 仿聊天气泡里一张结果卡:缩略图 + 「识别完成·请确认」+ 金额大字 + 字段(发票号带"请核"琥珀标 / 日期 / 分类紫标 / 卖家 / 记账给)+ 底部按钮(确认·入采购[紫] / 修改 / 改方向)。

- 编辑:手机 Tab 吸顶 + 滚动联动高亮 + 点 Tab 滚动;需复核横幅点击跳转高亮;票据图拖拽平移 + 滚轮缩放 + 旋转/复位;多文件缩略图切换;分店类型→分店码必填联动;有税票→税票号展开;币种→汇率展开;手动改额开关→出现提示;商品/服务分段;加行/删行;确认入账空必填→报错横幅+红框。

## 6. 明确不要做(禁区)
- ❌ **不做"报销人 / 员工报销 / 内部付款凭证(PV)"**——本产品定位老板/会计自记账,无报销审批流;列表不出现"报销人/出纳员"列。
- ❌ App 内**不放"LINE 录入"按钮**(LINE 是独立渠道,用户自己去用;App 里只有上传/拍照/手填)。
- ❌ **不用黑底、不用 emoji 图标、不用侧滑抽屉(drawer)**。
- ❌ **不抄袭任何竞品的文案或视觉**——这是 Pearnly 自有产品,按本文档的设计语言原创。

## 7. 交付格式
- 一个自包含 `.html`(内联 CSS/JS、内联 SVG 图标 sprite,无外部依赖,双击即开)。
- 顶部:「📱手机 / 🖥️电脑」设备切换 + 七屏切换。
- 桌面套常规后台框架(左导航+主区),手机套手机外框(单栏+汉堡)。
- 所有第 4 节交互必须真实可点。低保真到中保真皆可,但**视觉元素严格遵循第 1 节令牌与规则**。
- 如有取舍或不确定处,在文件顶部用注释列出你的假设,便于我们对齐。
```
