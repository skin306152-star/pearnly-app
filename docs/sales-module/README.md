## 这是什么

Pearnly 现在只有"进项"半边(OCR 识别别人开给我的票 + 对账)。客户要的是补上**销项半边**,
而且 Zihao 已拍板把范围抬高为一个**「统一智能录入引擎」**:

- 覆盖**两个方向**:销项开票 + 进项轻量录入(共用一套引擎/界面)
- 多种**轻量录入**:手填 / 拍照 OCR / 扫条码 / 扫二维码 / LINE 一句话 / 菜单点选
- 要**比竞品 Paypers 更智能**(自动查税号真伪 + 自动算 + 规则校错 + 自动对账 + 一键入 ERP)

销项发票 / POS 开票 = 这套引擎落地的**第一个出口**。详见 `docs/00-overview.md`。

## 死规则 · 执行规矩(Zihao 2026-06-04/05 拍板 · 全文见 `docs/13-build-plan-po.md` §一)

- **全站禁 emoji 图标**:用**线性图标**(lucide/feather inline SVG · currentColor · stroke 1.8)。
- **窗口换手 = 上下文 50–70% 才换**:连做多个 PO,到 50–70% 自主更新交接文档 + 计划勾进度再换;没到别换、别为单 PO 换。

- **不成整顿对象**:新代码按整顿理想形态写(<500/测试/Alembic/RLS/decimal/UTC/去AI味…见 `docs/10`)。
