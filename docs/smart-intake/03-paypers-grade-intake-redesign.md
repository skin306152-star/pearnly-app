## 二、竞品真数据逆向(本稿事实基础 · 非推测)

### 2.1 识别准确度 —— prod 跑 4 张真票 A/B 实证(`scripts/_ocr_ab_probe.py`)

Pearnly 现管线三层:`Google Vision(出文字)→ Gemini 2.5 Flash-Lite(只读文字·prompt 明令不许看图)→ 字段`;能看图的 L3(`gemini-2.5-flash`)**只在低置信兜底触发**。

| 字段 | 真值 | A1 现网纯文字 | A2 现网(L2+L3) | B image-first 图直喂 |
|---|---|---|---|---|
| 咖啡·发票号 | NZ01000017838 | 🔴 M201000017838 | 🔴 M201000017838 | 🟡 HZ01000017838 |
| 咖啡·卖家名 | กาแฟพันธุ์ไทย | 🔴 กาแฟ น ไทย ชาต(乱码) | 🟢 (L3修对) | 🟢 |
| 7-11·总额 | 110(净) | 🔴 115 | 🟢 110 | 🟢 110 |
| 7-11·小计/VAT | 102.80/7.20 | 🟡 110/5 | 🔴 空(L3丢了) | 🟢 102.79/7.21 |
| 海鲜·金额三件 | 2722/2544/178.08 | 🟢 | 🟢 | 🟢 |

**结论(真因,非玄学)**:抽不准卡在"主抽取看不到原图 + 模型档位最低"。Vision 把泰文读残一字,纯文字 Flash-Lite 救不回。**image-first 图直喂 Gemini Flash 几乎全中,咖啡票抽得和 Paypers 一字不差**。贯穿所有票还有一个真 bug:两位数年份→公历(佛历69→2026/25→2025)`_fix_two_digit_year_date` 失灵。

### 2.2 Drive/Sheets 外流 —— 借账号读到它真建的文件/表(完整 schema)

归档树:
```
Paypers/<主体>/<年>/
   ├─ 「<主体> - <年>」.gsheet         报表(主体×年一张·双tab:年汇总+当月)
   └─ <月: 06_มิถุนายน>/
        ├─ รวมหลักฐาน(证据)/<日期>_<商户>_<id>/ → 原图.jpg   每票一独立子文件夹
        └─ สำหรับสำนักงานบัญชี(交会计)/<日期>_<商户>_<报销人>_<id>.pdf   原图转PDF
```
- **Sheet 一行=一条明细**(非一张票),28 列:日期/ID/税票号/单据类型/付款状态/进项税票有无/描述/数量/单价/税前/行折扣/总折扣均摊/折后/VAT/WHT/四舍五入/应付/费用性质/大类/小类/卖家/卖家税号/分店/分店码/卖家地址/报销人/**证据(超链回原图文件夹)**/备注。
- **`id`(cuid)四处串联**:文件夹名 / 文件名 / Sheet ID 列 / 它库 → 全程可追溯。PDF 不带数据页(数据只在 Sheet)。
- **同一票产两份**:`jpg 留证`(按票分文件夹) + `PDF 交会计`(扁平·带报销人名)。

### 2.3 LINE 三层架构 —— 真截图逆向(`桌面/LINE/`)

**关键发现:它不是"做了个 LINE UI",是三层:**
1. **对话式 AI 助手(真 LLM)**:NL 意图→澄清反问→给跳转按钮。`รายงาน`→**自然语言分类汇总卡**(本月总额/张数/按大类分笔);`เมนู`→能力清单;`help/สวัสดี`→自介。
2. **原生卡 + 命令 + 图文菜单(8 入口)**:费用汇总/费用清单/建销售单(beta)/建收据/GG Drive链接/升级/联系我们/了解更多;文字记账"品名 金额"→记账+生成替代收据PDF(无限量);`ขอ link google drive`/`ขอ google sheet`→返回链接;发图→OCR 结果卡。
3. **LIFF 网页复用(重操作)**:✏️编辑/设置/仪表盘/导出 **直接在 LINE 里开 `app.paypers.ai` 的 webview**,复用 Web,不重画。
- **计费**:免费自动扫票 8 张/月,文字记账+网页录入不限量。

## 三、核心洞察:一张单据 = 富对象(8 层功能骨架)

> 上一版 UI 预览被否的根因 = 只做了 2 层骨架(几个字段+一个 banner),不是丑,是**功能太薄**。一张单据该派生这 8 层:

| 层 | 一张单据要回答 / 能做 |
|---|---|
| ① 输入 | 多渠道(Web拖拽/手机拍/LINE/文字) × 两模式(让系统读 / 自己填) |
| ② 归属 | 记给哪个**主体**(套账) · 方向(进项/费用/销项/对账/待归类)· 性质(商品/服务) |
| ③ 解析 | 文档级 + **行级明细**(每行独立大类/小类/VAT)两层 |
| ④ 税务 | 税前 / 折扣 / VAT(可抵) / WHT / 四舍五入 / 应付净额 |
| ⑤ 复核 | 置信度→**需复核字段高亮** · 行内编辑 · 图缩放旋转 · RD 税号核验 |
| ⑥ 归档/外流 | 原图存 Drive · 数据写 Sheet · 导出 Excel · 替代收据/付款凭证 PDF |
| ⑦ 聚合 | 月/年 KPI · 列表筛选搜索 · NL 报表("本月共?分类?") |
| ⑧ 渠道一致 | LINE 卡 ≈ 移动视图;轻操作 LINE 原生,重操作 LIFF 复用网页 |

## 五、设计:四块方案

### 5.1 识别准确度 —— image-first 提主 + 修年份 + 喂置信度

- **把 L3 的"看图抽取"从兜底提为主抽取**:发图直接喂 `gemini-2.5-flash`(图为主,Vision 文字仅作 hint),代码现成(`layer3_fallback.refine_with_image` 已是 image+text+JSON)。消灭乱码卖家名 + 算错总额。**这条改 OCR 主路径,先报方案(本稿即方案)。**
- **修两位数年份→公历**(`_fix_two_digit_year_date`),独立小修,收益全票。
- **保留置信路由**(`confidence_band`:auto/yellow_confirm/needs_review)→ 直接喂 §5.4 的"需复核高亮"。
- **模型档位待 A/B**:咖啡发票号 tiny 字 Flash 有 run-to-run 抖动。施工前补一轮 `Flash vs Gemini 2.5 Pro vs Claude vision` 对照(专打 tiny 发票号),用数据定档,别猜。
- 副作用情报:image-first 连 Vision 服务账号都不需要(只要 Gemini key),运维更简。

### 5.2 Drive/Sheets 外流 —— 照逆向 schema,且比它多写分录

照 §2.2 的归档树 + 28 列 Sheet 做,**与做账数据映射见 §6**。Pearnly 优势:
- 它 Sheet 只是票面字段;**你可多加"借贷分录"列**(你做账模块已生成),给会计的不只是票,是**做好的账**。
- 报销人列我们不做 → 留空或换成"经手套账成员"。
- 实现走 OAuth(用户授权)+ 服务端用 token 写;`id` 用你的 doc_id 串联;证据列超链回 Drive 文件夹。**改计费/外部集成,先报方案(本稿即方案)。**

### 5.3 LINE 三层 —— 对话 AI + 命令 + LIFF 复用(不重做 UI)

- **轻层(LINE 原生)**:① 发图→OCR **Flex 结果卡**(替换现纯文字,字段+置信"请核"标+[确认入采购][✏️编辑][改方向])② 文字记账(已有 F10,补结果卡)③ 取链接命令 `ขอ link drive`/`ขอ sheet` ④ Rich Menu 多入口 ⑤ NL 报表(`รายงาน`→分类汇总卡)。
- **重层(LIFF)**:✏️编辑/设置/导出 **用 LIFF 在 LINE 里开你现有的网页**(`purchase-form`/复核屏),**不把整套重做成 Flex**。这是最大省力点。
- **对话 AI 助手 → 二期 · 且做成【全站助手】不限采购(Zihao 2026-06-13 定)**:用 Claude tool-use 把 Pearnly **所有模块**(进项/销项/做账/报税/银行对账/POS/库存/权限…)的后端函数挂成工具,LLM 负责"听懂意图 + 说人话 + 路由 + 调工具 + 附动作按钮"。LINE + Web 同一个助手。**比 Paypers 强**:它只采集+汇总;你后端深得多,助手能干的事更多。**这是平台级初创,二期单开设计文档**(本稿只记决策)。一期先把 §5.3 轻层卡+命令+LIFF 做了。

### 5.4 复核屏 —— 需复核高亮 + 图缩放 + 手机优先(在 `purchase-form.ts` 上加强)

补 Paypers 有你没有的三件,复用现有字段/算价/RD/learned 分类,只重排交互:
1. **顶部"需复核"banner**:消费 §5.1 的 `confidence_band`,列出低置信字段、点击跳转高亮。
2. **字段三态着色**:绿=已识别可信 / 琥珀=请确认 / 普通。
3. **图查看器**:缩放/旋转/全屏(替换静态缩略图)+ Tab 分"基本/明细"。

## 七、手机优先布局(主要手机端)

- **采集**:三采集口(拍照走原生后置 `capture=environment` · 复用 `upload-camera.ts`)+ "读/手填"二选一。
- **复核(核心)**:手机单栏纵向 = 顶部需复核 banner → 图查看器(可折叠/全屏)→ Tab(基本/明细)→ 底部 sticky 动作条(存草稿/过账)。桌面由它响应式展开成"左图右字段"两栏。
- **LINE**:轻操作原生卡;✏️→LIFF 打开**同一套手机复核屏**。
- ⚠️ 真机验:headless≠真机(见 [[headless-not-real-device-mobile]]),布局定稿要真 iOS/Android 验。
