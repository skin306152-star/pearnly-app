# 自动做账(Phase 2)· 00 概念与边界

> Pearnly 2.0 闭环第二块 = 把进货/卖货/收款**自动变成账本**。动前先读这页 + `docs/product-vision/00`(2.0 定位) + `docs/ui/DESIGN_LANGUAGE.md`(UI 一律照此)。
> **状态:UI 设计封板(2026-06-08)**。5 屏定稿(`桌面/Pearnly_做账_UI预览/` 01-05·收拢版设计语言)+ 图纸 00-05。前置:进项 Phase 1 已上线闭环、销项/POS 已有。

## 一句话

**自动做账 = 业务一发生(进项/销项/POS/收付款),系统按规则自动生成会计借贷凭证 → 总账/明细账,人只审拿不准的那几笔。** 老板不配科目、不录凭证、不懂借贷,只确认例外。这是打破"石板 ERP"的发动机。

## 跟现有的关系(别混)
- **进项/销项/POS** = 记业务(花了/收了什么)。已上线。
- **做账(本模块)** = 把这些业务**翻译成会计账本**(借贷分录/凭证)。新建。
- **报税(Phase 3)** = 拿账本数据填税表(PP30/PND/财报)。后做。
- 闭环:`业务 → 做账(凭证/账本) → 报税(税表)`。

## 灵魂(2.0 六原则的落地)
- **录入即经营,不另录凭证**:进项一 `post()`、销项一开票、POS 一收款 → 凭证自动生成。
- **AI 自动记账**:按业务类型套借贷模板(见 [[02-posting-rules]]),科目自动映射,人只审例外。
- **人话翻译**:每张凭证配一句大白话("这批货进库存,付了税,先欠供应商"),不懂会计也看得懂。
- **越用越省**:审过的例外(某供应商=服务)系统记住,下次自动套。

## 六屏(UI 已定稿 · `Pearnly_做账_UI预览/`)
1. **做账主屏** — 凭证列表(北极星「本月X笔已自动做账」+ 待审行动卡 + 列表·行内展开看借贷)。
2. **逐笔审** — 审例外(业务 + 拿不准的点 + 借贷预览 + 一选就改 + 系统记忆)。
3. **科目表** — 泰标预置开箱即用 · 按类型筛 · 加/改(老板配置后台)。
4. **出账本/报税包** — 月末出口:法定账本 + VAT/WHT 报税材料 + 年末财报。
5. **做账设置** — 自动过账门槛/准则/存货法/本位币/科目映射(配置后台)。
6. **银行对账**(竞品调研后补 · 见 `docs/product-vision/02`)— 银行流水↔账本凭证智能匹配(高置信自动·低置信确认·学规则)。**账与银行对上 = 账对不对的最终校验**(Xero/QBO/PEAK 核心·不补则老板不敢信)。**复用现成 `services/recon` + `services/importer`(流水导入/AI映射/模板学习),不从零造。**

> **可见规则(竞品调研后补)**:把 review_learned 学到的做成**用户看得见、可改的规则**(像 Xero bank rules:"这供应商/这描述以后都记 XX")——在做账设置里列出+可编辑。智能不是黑箱。

## 六类完整性(全覆盖)
| 类 | 覆盖 |
|---|---|
| ① 操作面 | 主屏(凭证列表)· 逐笔审 |
| ② 老板配置后台 | 科目表 · 做账设置 |
| ③ 登录/接入·角色 | 复用主程序登录;**审/改/过账 = 会计或 owner;员工只看** |
| ④ 四态+边界 | 空(本月还没业务)/加载/错 + **借贷不平拦截**/红冲方向/重复防生 |
| ⑤ 角色权限 | 同 ③ · 科目/设置/过账写 = `require_account_owner` 或会计 |
| ⑥ 模块联动 | 进项/销项/POS/收付款 → 凭证;凭证 → 总账 → 报税(Phase 3);**银行流水 ↔ 凭证对账**(复用 recon/importer) |

## 套账隔离(硬要求)
科目表 / 凭证 / 账本全部按 `workspace_client_id` 隔离(切公司只看本家账)。连号(凭证号)按主体按期。建在套账隔离地基上(已收官)。
