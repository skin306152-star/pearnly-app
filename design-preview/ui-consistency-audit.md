# Pearnly UI 一致性审计清单(拔出来 · v1)

> 目的:把"UI 不统一"拆成可执行的清单,做一项勾一项,确保不漏。
> 范围:home.html(160 按钮)+ login.html(27)+ home.js 动态(34)≈ **221 个按钮**。
> 现状:已有 `.btn` 底子,但混着 **30+ 种杂牌按钮 class** + **11 个内联样式按钮** → 这是"修一处漏一处"的根因。

---

## A. 样式统一(颜色/大小/形状/hover/禁用)—— 低风险,纯 CSS

**标准(已定):** 品牌蓝 `#2563EB` 主 + 琥珀橙 `#F59E0B` 强调;38px 高(小 32);圆角 9;扁平;
四层级 = 主(实心蓝)/ 次(白底描边)/ 轻(幽灵)/ 删除(红描边);hover 变色+回弹;禁用=置灰+🚫。

**要收编的杂牌 class(扫描所得,逐个并入标准):**
- `tc-tool-btn` / `tc-btn` / `tc-btn-primary` / `tc-btn-text` / `tc-btn-danger`(test-center)
- `bank-filter-btn` / `bank-upload-btn`(银行对账)
- `email-interval-btn` / `email-interval-btn-title/desc`
- `dash-quick-btn`(工作台快捷)
- `ob-btn-skip` / `ob-btn-next`(引导)
- `int-btn-view-logs` / `erp-map-show-advanced-btn` / `erp-map-adv-btn-label`
- `folder-alt-btn` / `hist-batch-icon-btn` / `history-pager-btns`
- **11 个内联 `style=` 写死的按钮**(home.html)→ 全部去内联,改用 class
- 已规范的(保留):`btn btn-primary` / `btn-ghost` / `btn-danger` / `btn-small` / `btn-tiny`

**守门(关键 · 永不再漏):** 加 CI 检查 —— 任何 `<button>` 不带标准 `.btn` class、或带内联 `style`、或新增 `*-btn` 杂牌类 → 测试报红拦下。

## D. 位置一致(布局规范)—— 中风险(改结构)

**建议规范(全站统一):**
- 页面级**主操作**(上传/新建/开始)→ 固定在**卡片右上角**;
- **批量工具栏** → 固定在**列表正上方**,且**选中才出现**;
- **行内操作**(查看/编辑/删除)→ 固定在**行尾右侧**,图标按钮;
- **保存/取消** → 弹窗**右下角**,主按钮在右。

**要核对的模块**(同功能按钮现在位置不一):
上传识别 / 发票记录 / 对账中心(3 子)/ 客户管理(2 子)/ 异常栏 / ERP 推送 / 设置。

---

## 落地顺序与分工

1. **第1步 清点**(本清单)— 只读,安全,进行中。
2. **第2步 立标准** — 把 A 的品牌色按钮写成唯一一套中央 CSS。安全。
3. **第3步 收编 A**(样式)— 批量改 class,低风险;配视觉测试。
4. **第4步 装守门** — A 类加 CI 检查,自此自动防漏。
5. **第5步 B/C/D** — 减法/交互/位置,逐项改,中风险,走前端窗口 + 路由守护 + 视觉测试,一项一验。

> A+守门 让"样式"永久统一;B/C/D 靠本清单逐条扫+复查,做一项勾一项。
