# POS 项目 · 01 功能清单与场景

> 从用户真实场景出发,列全 POS+库存要具备什么。技术契约在后续文档从这些功能反推。

## 三个角色

| 角色 | 在哪 | 看到什么 | 设备 |
|---|---|---|---|
| 收银员 cashier | `/pos` 独立前台 | 只有:卖货 · 查商品 · 自己的班次 | 平板/收银机/手机(触屏) |
| 老板 owner / 会计 | `/home` 主程序 | 多出:库存 · 销售报表;其余照旧 | 电脑 |
| 超管 Earn | `/admin` | 不变(不碰 POS) | 电脑 |

## A. POS 收银前台(/pos · 收银员用)

### A1 通用内核(所有业态都有)

| 功能 | 说明 | 复用 |
|---|---|---|
| 登录/班次 | PIN 登录 · 开班(备用金)· 交班(日结对账) | 角色框架 |
| 选品 | 扫码(枪/摄像头)· 搜索 · 商品图库点选 · 常卖置顶 | products |
| 购物车 | 改数量 · 改价(权限)· 行折扣 · 整单折扣 · 删行 | totals.py |
| 收款 | 现金(自动算找零)· PromptPay 扫码 · 刷卡 · 混合支付 | promptpay.py |
| 出小票 | 热敏小票自动打印 · 可重打 | pdf_thermal.py |
| 挂单/取单 | 顾客临时离开,挂起本单,叫号继续 | 新建 |
| 退货/换货 | 凭原小票退 · 退款 · 库存回补 | sales_documents 红冲 |
| 实时扣库存 | 收款成功即扣 · 卖光提示 | 库存模块 |
| 升级税票 | 顾客要 ใบกำกับภาษี → 填买方 → 小票升正式税票 | sales 合规 |
| 离线开单 | 断网照常卖,联网自动补传 | 离线层(新) |

### A2 跨业态能力块(开关式 · 多个业态共用 · 库存/商品地基要预留)

这些不属于某一个业态,是"按需开"的能力。**几个是数据模型地基(批次/效期/多单位),必须从第一版就在表里预留,否则后接 = 改表返工。**

| 能力块 | 说明 | 哪些业态要 | 地基影响 |
|---|---|---|---|
| **批号 + 有效期** | 入库记批号/效期 · 近效期预警 · 先效先出(FEFO)· 卖出按批扣 | 药房 · 食品 · 生鲜 · 化妆品 | ⚠️ 库存表必须支持批次维度(地基) |
| **多单位 / 拆零** | 1 箱=N 瓶、1 盒=10 板=100 粒;整售/拆售换算、按最小单位扣库存 | 药房 · 批发 · 零售 | ⚠️ 商品表需单位换算关系(地基) |
| 称重 | 按重量计价(生鲜/散装) | 零售 · 生鲜 | 商品标"称重品" |
| 会员/积分 | 会员价、积分、挂账 | 零售 · 药房 · 餐厅 | 复用 clients |
| 条码枪 | 扫码进购物车(浏览器即键盘输入) | 零售 · 药房 · 批发 | 复用 products.barcode |
| 处方/药师 | 处方药登记 · 药师记录(可选) | 药房 | 药房专属 |
| 账期/应收 | B2B 赊销、账期、对账 | 批发 | 复用 sales payment 字段 |

### A3 业态预设(选业态一键开好对应能力块 + 模块)

| 业态 | 默认开的能力块 |
|---|---|
| 零售/小卖部 | 条码枪 · 称重 · 会员积分 · 多单位 |
| **药房** | 条码枪 · **批号+有效期** · **多单位/拆零** · 处方/药师 · 会员 |
| 餐厅 | 桌台 · 拼桌/并单 · 厨房单(KOT)· 堂食/外带/外送 |
| 批发贸易 | 多单位 · 批号+有效期 · 账期/应收 · 客户 |
| 服务业 | 预约 · 计时计费 · 服务人员指定 |

> 内核 = 选品→收款→出票;业态预设 = 一键勾好对应能力块。**通用兼容靠的就是"能力块开关",不为每个业态写一套。** 施工先做内核 + 零售 + 药房(覆盖批次/效期/多单位地基),其余预设随后叠加。(先做哪几个待 Zihao 过)

## B. 库存模块(/home 主程序 · 老板/会计用)

| 功能 | 说明 |
|---|---|
| 商品库存 | 每个商品的实时数量(按账套主体/仓库) |
| 入库 | 进货加库存 · 关联进项(可选) · 成本记录 |
| 出库 | POS 卖货自动出 · 手动出库(领用/报损) |
| 调拨 | 多仓之间调货(多仓为后续) |
| 盘点 | 实际数对系统数 · 差异调整 |
| 低库存提醒 | 低于阈值提醒补货 |
| **批次/有效期** | 入库带批号+效期 · 库存按批 · **近效期/过期预警** · 卖出 FEFO 先效先出(能力块开才显) |
| **多单位** | 商品定义单位换算(箱/瓶、盒/板/粒)· 入库整、出库可拆 · 库存按最小单位记(能力块开才显) |
| 库存报表 | 结存 · 进销存 · 周转 · 近效期清单 |
| 成本核算 | 移动加权/FIFO(先简单后精) |

> 数据模型影响(详见 03-data-model):库存账簿支持「批次维度」、商品支持「单位换算」必须**第一版就建进表**,即使初期 UI 不全开——后补 = 改表 + 改全部出入库逻辑 = 返工。

## C. 后台(/home · 老板/会计用)

| 功能 | 说明 |
|---|---|
| 开通收银 | 选业态 → 自动开模块 → 建收银员账号 |
| 收银员管理 | 增删收银员 · 设 PIN · 停用 |
| 销售报表 | 当日/区间营业额 · 按收银员/商品/支付方式 · 现金 vs 扫码 |
| 班次记录 | 每个班次的开班/交班/差异 |
| 收银设置 | 小票样式 · 是否允许改价/折扣 · 默认账套主体 · 业态功能开关 |
| 接会计 | POS 销售=销项数据,接现有对账/报税(ภ.พ.30) |

## D. 客户场景(验收锚点)

1. **零售小卖部一天**:开班 → 扫码卖货收现金找零 → 扫码收款 → 断网继续卖 → 网恢复自动补 → 顾客要税票一键开 → 交班日结。
2. **老板看店**:手机看今日营业额 · 库存不足提醒 · 月底销项账已就绪不用补录。
3. **退货**:顾客拿小票来退 → 扫小票 → 选退的商品 → 退款 → 库存回补。

## E. 非功能要求(硬指标)

- **快**:扫码到进购物车 < 200ms;收款到出小票 < 1s。
- **离线**:断网可持续开单 ≥ 整个班次;联网后自动无损同步。
- **触屏**:按钮 ≥ 44px;主操作单手可达;字号清晰。
- **多语言**:界面 4 语(th/en/zh/ja),收银员语言可设。
- **钱**:全程 Decimal,不用 float;时间 UTC 存、本地显示。
- **隔离**:RLS 强制租户隔离(POS 涉钱,后端硬隔离,不只前端藏)。
- **多端**:同一店多台收银机并发卖,库存不超卖、单号不撞。
