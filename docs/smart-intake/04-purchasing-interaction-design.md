## 〇、读法 + 三条铁原则

- **读法**:每屏按「桌面端 / 手机端 / 逐字段 / 与现有关系(冲突+兜底)」四看。
- **原则1 不破坏**:在现有 `purchase-*` 模块上长,不塞外来代码;现有进项/做账/报税/对账/套账联动一字不动。
- **原则2 有冲突就兜底**:每个新功能旁标清和谁撞、怎么兜(见各节 E 表)。
- **原则3 不做报销人**:Paypers 的"报销人/审批"全线省略或换"经手成员"(Pearnly 定位老板/会计自记账)。

## 五、编辑屏(录入/复核)

**A. 用途** — 一张进项票从(识别草稿/手动空白/详情编辑)进来,左看原图、右逐字段核对修正,存草稿或入账。**全模块唯一录入屏**,所有来源汇到这里。

**B. 桌面端** — 沿用现有**左右两栏常驻、不分 Tab**(不照搬 Paypers 右两Tab):左=票图卡+凭据卡;右=单据信息卡+明细卡+合计卡(三卡竖排,保 Pearnly 一屏纵览)。
- 左票图卡:看图/原生缩放旋转拖动/重拍换图;**新增多文件 1/N 相册**(一单挂多图/PDF:翻页+缩略图+「+附文件」,删二次确认;`purchase_attachments` 已支持多行,不建图片编辑器)。识别角标"智能识别 N 项"(不出现"AI")。
- 凭据卡:生成替代收据(单据已存才可点)+ 上传附件 + 「经手成员」(兜底替代报销人,沿用 `requester`)。
- 单据信息卡:文档类型(进货发票/费用单/采购订单 分段)、供应商选择器(搜名/税号+新建)、税号、分店类型、**新增分店码/分店名**、地址、出票日期、单号、有无税票(是→展开**税票号**)、付款状态(已付/未付挂账)、**币种改下拉**(非THB展开**汇率**,`fx_rate` 已存)。
- 明细卡:**新增"合并记一条↔拆分多条"**(默认拆分=现状)+提示"按分类记账便于报税";明细行(商品/服务、详情、数量、单价、大类、**新增小类联动**、VAT、**新增行折扣开关**、行额只读、删);「+加一行」;商品行"匹配商品/入库"(现有)。
- 合计卡:税前/折扣/进项VAT(可抵角标)/含税合计/WHT(代扣角标)/四舍五入/实付——只读即时重算。**新增"手动改额"开关(高风险兜底)**:默认关,打开**仅放开 四舍五入+整单折扣**两项+二次确认+"已手动调整"标记,**不放开直接改 VAT/实付**。
- **新增硬必填校验**(对应 Paypers 橙报错条):仅"确认入账"触发,缺项(出票日期/供应商/类型;选分店则分店码必填;勾有税票则税票号必填;非THB则汇率必填)→ 顶部橙条列出可跳转+字段红框;**存草稿不强校验**。底部:存草稿/确认入账+**新增删除**(改既有草稿时,二次确认)。

**C. 手机端** — 复刻 Paypers **三 Tab**(把桌面左栏拆成第1 Tab):Tab1 文档(票图+多文件+附件+凭据)、Tab2 信息(信息卡字段纵向单列)、Tab3 明细与汇总。动作条吸底常驻;报错条置顶,点缺项自动切 Tab+滚动定位;多文件缩略图横滑。

**D. 逐字段** — 见上述 B + 子代理详表(完整字段/控件/必填/来源/校验已逐行列出,施工时取用)。

**E. 与现有关系(关键·按风险分级)**
- 🟢 **低风险(schema 已有字段,只放开 UI,直接做)**:币种下拉(`currency`/`fx_rate`)、分店码(`branch_no`)、小类(`subcategory_id`)、行折扣(行 `discount`)、经手成员改名(`requester`)。
- 🟠 **需设计确认**:多文件相册(附件表已支持多行)、合并/拆分(前端态,保存仍按行存)、硬必填校验(现仅 dedupe 软提示)。
- 🔴 **高风险必走兜底**:手动改额(只放 rounding+整单折扣+二次确认,后端仍以 `purchase-calc` 为权威核对);税票号/分店名(schema 无独立列 → 复用 `doc_no`/`note` 承载或落 `ocr_raw`,**本期不新建迁移**)。
- 文档类型现有是分段非下拉、供应商选择器、分店类型、图片缩放、付款状态 —— 已有,沿用不动。

## 六、采集入口 + 待归类 + 智能分流

**A. 用途** — 用最省事方式(拖文件/拍照/一句话/LINE)把票丢进来,系统自动识别+判方向+记给哪个主体;能记的预填进录入,拿不准的安全落待归类,**绝不静默丢错、绝不生成空单**。

**B. 桌面端** — 主屏「拍进项票」(桌面=开文件选择器/拖拽,**无拍照**)+「⋯」里「手动记一笔」(直接手填费用)「LINE一句话」。识别后 `/api/purchase/intake` + `judge_direction` 按税号分流:买方=本主体有VAT→进项进录入;无VAT→费用;卖方=本主体→提示去开票;银行单→去对账;糊图/残缺/低置信→落待归类并带过去。

**C. 手机端** — 「拍进项票」给三入口:**拍照(原生后置)/相册(多选)/文件**。**和桌面唯一实质差异=桌面无拍照、手机有**。拍照前可弹"拍摄贴士"(可永久跳过);多选/连拍后底部浮条选"分别识别/合并PDF"。分流逻辑与桌面同一后端,不另搞一套。

**D. 逐区块** — 见子代理详表(主按钮/⋯菜单/上传区/拍摄贴士/缓冲浮条/识别中/分流结果/待归类/记账主体)。

**E. 与现有关系(冲突+兜底)**
- 🟢 上传拖拽区(`upload-files.ts`)、多格式、智能分流(`intake.py judge_direction`=比 Paypers 多判方向)、待归类5动作(`purchase-inbox.ts`)——已有,复用,是优势项。
- 🟠 **"读 vs 手填"二选一**:现有是两个分开入口(拍票=识别 / 记一笔=手填),语义已覆盖。**兜底:保留两入口只补文案**("没有发票?手动记一笔→"),不强行合并成单选卡(避免多一次点击)。
- 🔴 **桌面误放拍照**:现 `upload-camera.ts` 桌面把 camera input"退化成文件选择器"语义含糊 → **明确:桌面只显拖拽/选文件不显拍照;手机才显拍照/相册/文件**,按设备能力显隐。
- **采集时选主体**:沿用顶栏**全局套账切换器**,采集屏不放第二个选择器(防 ws 不同步坑),仅顶部只读提示"记账给:〈套账名〉";未选套账→提示先选。
- 待归类"记进项/记费用"二选一**已等于让用户改方向**,无需新增独立"改方向"按钮;仅把"智能猜测"显示更明确。

## 七、外流(Drive/Sheets/Excel)+ 凭证(替代收据/WHT)

**A. 用途** — 把某套账主体的进项票一键归档进老板自己的 Google Drive(原图+转PDF双产物)并同步成可追溯 Sheet;无票时开替代收据/扣缴凭证;**Pearnly 比竞品多写"已自动入账的借贷分录"列**。

**B. 桌面端**
- 入口**不平铺三按钮**(收拢原则):列表页一个"导出/归档"动作 → 收拢面板分三件:导出Excel(本地下载零授权)/归档到Drive/同步Sheet。**Google 授权放"集成"页**(与 LINE/Xero/邮件同列)新增"Google Drive/Sheets"卡;未授权时面板引导去授权再回来。
- 导出Excel:点即下载当前筛选范围明细 .xlsx(列同 Sheet),零授权、最稳兜底。
- 归档:已授权→选范围(主体/年月/仅已入账+已开凭证,草稿不归)→异步进度→完成给"已归档N张/跳过N张"+Drive/Sheet直达链接;失败给明确原因可重试且**幂等只补未成功**。
- **Drive 树(照逆向)**:`Pearnly/<套账主体>/<年>/<月>/` 下 ①证据原图(每票一 id 命名子文件夹放原图)②交会计PDF(原图转PDF扁平,名 `年-月-日_供应商_单据id.pdf`,报销人位换"经手成员"无则空)。单据id 在 文件夹名/PDF名/Sheet ID列 三处对得上。
- **Sheet**:每"主体×年"一张,双标签(全年/当月),一行=一条明细。列=Paypers 会计语义列 + **末尾追加 Pearnly 独有:借方科目/贷方科目/凭证号/入账状态**(来自 post 后过账钩子 `accounting/hooks.py` 生成的分录;未开做账模块该4列留空标"未记账")。证据列超链回 Drive 原图文件夹。
- 凭证(已有,不重造):详情页凭证区 无正式票→"生成替代收据"、WHT>0→"生成扣缴凭证",挂系统附件可下载;归档时这些PDF随该单进"交会计PDF"。

**C. 手机端** — 不做完整归档操作面(授权跳转+进度小屏体验差);手机以"取链接/查看"为主:详情可看原图、生成/看替代收据&WHT PDF。LINE 取链接命令(发"本月进项表"→当年 Sheet 链接;发单据id→Drive 原图文件夹链接;未授权→引导去桌面集成页)。

**D. 逐区块** — 见子代理详表(导出Excel/Google授权/归档Drive/同步Sheet/借贷分录列/替代收据/WHT/LINE取链接)。

**E. 与现有关系(冲突+兜底)**
- 🟢 明细数据源(`purchase_routes` docs 列表/详情)、替代收据&WHT(`documents.py` 已完整:生成/document.pdf/bill-image)、原图附件(`purchase_attachments`)——复用,不重造;归档读这些字节上传。
- 🟠 **借贷分录列**:过账钩子是"业务→做账"单向 enqueue,**无反查接口** → 新增只读"按 source_id 取分录+凭证号"喂导出列;未开 accounting 留空标"未记账"不报错。明细汇总:`reports.summary` 只给按期合计 → 新增"逐行明细"读取(沿用现有查询+套账隔离),不动 summary(继续供报税/KPI)。
- 🔴 **Google Drive/Sheets/OAuth 全新**(全仓 0 集成;登录的 Google OAuth 仅 userinfo 范围,拿不到 Drive/Sheets 写权,不能复用)→ 全新独立 OAuth 流+独立凭据表,**凭据按 (tenant, workspace_client) 存取,归档全程带套账过滤,绝不跨套账串目录/串表**(参照 Xero `oauth_store` 范式);走异步任务+失败可重试幂等;**Excel 永远作零依赖兜底**;Google 故障不污染进项主事务。
