# 16 · 竞品 Paypers 分类法 + 识别字段(对标参考)

> 2026-06-15 · 从 Paypers LINE 卡 + 网页筛选器逆向(用户截图)。**当参考**:① 我们默认费用科目树
> 可照此 seed(现状:测试账号无科目树 → 一直无法自动分类 → 全进确认)② 识别字段集对标补齐。
> 注:**报销人(ผู้ขออนุญาตเบิก)我们不做**——群聊发送人只记「经手成员」(铁律/历史决策)。

## 1. 费用科目(大类)· หมวดหมู่ —— 候选默认科目树

| Paypers(泰) | 中文 | 会计含义 |
|---|---|---|
| ค่าใช้จ่ายสำนักงาน | 办公费用 | 文具/水电/租金等日常 |
| ไอที & ซอฟต์แวร์ | IT & 软件 | 订阅/云/设备 |
| การตลาด & โฆษณา | 营销 & 广告 | 投放/物料 |
| บุคลากร & ค่าจ้าง | 人事 & 薪酬 | 工资/外包/福利 |
| เดินทาง | 差旅 | 交通/住宿 |
| อาหาร & รับรอง | 餐饮 & 招待 | 会议餐/客户招待 |
| ต้นทุนสินค้า | 商品成本(COGS) | 原料/进货 |
| ภาษี & ค่าธรรมเนียม | 税费 & 手续费 | VAT/WHT/手续费 |
| การเงิน | 财务(金融费用) | 利息/银行费 |
| อื่นๆ | 其他 | 兜底 |

> 子分类示例(它卡里出现的):อาหาร & รับรอง → **ค่าอาหารประชุม(会议餐费)**;ค่าใช้จ่ายสำนักงาน →
> 水电(ค่าน้ำ/ค่าไฟ)。我们的 `_EXPENSE_KEYWORDS`(services/purchase/expense_keywords.py)可照此扩。

## 2. 单据类型 · ประเภทเอกสาร —— 候选 doc_type 枚举 + UI 下拉

收款/付款类:ใบเสร็จรับเงิน(收据)· สลิปโอนเงิน(转账凭条)· บิลเงินสด(现金账单)·
ใบกำกับภาษี(完整税票)· **ใบกำกับภาษี(อย่างย่อ)(简式税票)** · ใบแจ้งหนี้(发票/账单)·
ใบแจ้งหนี้ที่ชำระแล้ว(已付账单)· ใบวางบิล(请款单)· ใบสำคัญรับ(收款凭证)· ใบสำคัญจ่าย(付款凭证)·
**ใบรับรองแทนใบเสร็จ(替代收据/无票凭证)**。
采购流转类:ใบเสนอราคา(报价单)· ใบขอซื้อ(请购 PR)· ใบสั่งซื้อ(采购订单 PO)· ใบยืนยันคำสั่งซื้อ(订单确认)·
ใบสั่งงาน(工单)· ใบส่งของ(送货单)· ใบรับสินค้า(收货 GR)· ใบโอนสินค้า(调拨单)。
银行/其他:Bank statement(银行对账单)· Credit card statement(信用卡对账单)· สลิปเงินเดือน(工资单)·
บัตรประชาชน(身份证)· หลักฐานอื่น ๆ(其他凭证)· ไม่ใช่หลักฐานการจ่ายเงิน(非付款凭证)· ไม่มีเอกสาร(无单据)。

> 我们现有 doc_type 只到 tax_invoice/simplified/receipt/credit_note/other(services/ocr schema)。
> 银行/电商截图应判「付款证据/订单证据」不冒充税票(大任务书 §6);此表可当 UI 下拉 + 分流目标。

## 3. 识别返回字段(它的 LINE 卡)vs 我们

| Paypers 字段(泰) | 含义 | 我们有吗 |
|---|---|---|
| จำนวนเงิน | 金额(总额) | ✅ amount |
| ประเภทเอกสาร | 单据类型 | ✅ document_type |
| เลขที่ใบกำกับภาษี | 税票号 | ✅ invoice_number |
| ประเภทค่าใช้จ่าย | 支出类型(商品/服务) | ✅ expense_type |
| วันที่ | 日期 | ✅ date |
| หมวดหมู่ / หมวดหมู่ย่อย | 分类 / 子分类 | ✅ category/subcategory |
| ธุรกิจ | 业务主体 | ✅ = 我们「套账」workspace |
| ผู้ขาย/ร้านค้า | 卖家/商户 | ✅ vendor |
| **ที่อยู่ผู้ขาย** | **卖家地址** | ⚠️ build_draft 已抽 seller_addr,但**卡未显** |
| **รายการค่าใช้จ่าย** | **逐条明细**(1.xxx 179 / 2.xxx 189 / และอีก N รายการ) | ⚠️ 有 purchase_lines,但**卡只折成一条「明细」** |
| รายละเอียด | 详情/摘要 | ✅ detail |
| ผู้ขออนุญาตเบิก | 报销人 | ❌ **不做**(只记经手成员) |
| (完整税票)卖家 13 位税号 | seller_tax | ⚠️ 已抽,卡未显 |

## 4. 可落地的三件(后期接 UI 时做)
1. **seed 默认费用科目树** = §1 十大类(+ 常用子类)→ 直接解决「无科目树→全进确认/不自动分类」。最高价值。
2. **doc_type 枚举扩到 §2** + 银行/电商截图分流为「付款/订单证据」。
3. **数据卡补两字段**:逐条明细(顶 2–3 条 + 「还有 N 项」)+ 卖家地址(税票时);完整税票显 seller_tax。
