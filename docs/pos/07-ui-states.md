# POS 项目 · 07 每屏四态

> 防"空白屏 / 转圈不消失 / 出错没反馈 / 边界 500"(销项坏图 500 那类)。每屏定义四态:正常 / 空 / 加载中 / 出错。施工时四态都要做,验收逐态查。
> 通用规则:加载=骨架屏或转圈(>300ms 才显,避免闪);出错=友好文案(06)+ 重试按钮;空=插画+引导;**任何 fetch 必有 try/catch → 出错态,不许整屏崩**。

## 屏 1 · 收银主屏
| 态 | 表现 |
|---|---|
| 正常 | 商品网格 + 购物车 |
| 空 | 搜索无结果:"找不到商品,换关键词";新店无商品:"先去后台加商品/导入" |
| 加载 | 首次 bootstrap 骨架(灰块);搜索 inline 转圈 |
| 出错 | bootstrap 失败:整屏"加载失败 [重试]";收款失败:弹窗内红字(06 文案)不关弹窗,可改可重试 |
| 边界 | 在库 0 且禁负库存→商品卡灰显"缺货"不可加;改价超权限→禁用 |

## 屏 6 · 开班登录
| 态 | 表现 |
|---|---|
| 正常 | 收银员头像 + PIN 盘 |
| 空 | 无收银员:"请联系老板在后台添加收银员" |
| 加载 | 提交 PIN 时键盘锁 + 转圈 |
| 出错 | PIN 错→抖动+清空+`pos.pin_invalid`;网络错→"连接失败 [重试]" |

## 屏 2 · 升级税票
| 态 | 表现 |
|---|---|
| 正常 | 买方表单 |
| 加载 | 开具时按钮转圈、防重复提交 |
| 出错 | 税号无效 inline 红字;已升级→`pos.already_upgraded` 提示并关闭 |
| 边界 | 必填未齐→开具按钮 disabled |

## 屏 3 · 挂单/取单
| 态 | 表现 |
|---|---|
| 正常 | 挂单卡网格 |
| 空 | "暂无挂起单"(插画) |
| 加载 | 本地数据,基本无加载 |
| 出错 | 取回的单商品已下架→提示"部分商品已变更" |

## 屏 4 · 退货
| 态 | 表现 |
|---|---|
| 正常 | 原单 + 可选行 |
| 空 | 未输小票号:引导"扫描或输入小票号" |
| 加载 | 查单转圈 |
| 出错 | 小票不存在→"找不到此小票";超退→`pos.over_refund` |
| 边界 | 未选行→确认按钮 disabled;已全退过→提示 |

## 屏 5 · 交班日结
| 态 | 表现 |
|---|---|
| 正常 | 班次汇总 + 点钞 |
| 加载 | 拉汇总骨架;提交转圈 |
| 出错 | 提交失败 [重试];无 open 班次→"当前没有进行中的班次" |
| 边界 | 差异≠0→红色高亮但允许提交(差异入记录) |

## 屏 7 · 库存后台
| 态 | 表现 |
|---|---|
| 正常 | 库存表 + 汇总卡 |
| 空 | 无商品:"还没有商品,去添加/导入";筛选无结果:"没有符合条件的" |
| 加载 | 表格骨架行 |
| 出错 | "加载失败 [重试]";入库提交失败→弹窗内提示 |
| 边界 | 缺货行红标;近效期黄标 |

## 屏 8 · 开通收银
| 态 | 表现 |
|---|---|
| 正常 | 业态卡 + 预览面板 |
| 加载 | 开通提交转圈(批量建表/建账可能稍久) |
| 出错 | 开通失败→明确提示 [重试];已开通过→直接进收银台 |
| 边界 | 收银员 PIN 不足位→提示 |

## 屏 9 · 销售报表
| 态 | 表现 |
|---|---|
| 正常 | KPI + 图 + 榜 |
| 空 | 区间无数据:"这段时间还没有销售" |
| 加载 | KPI/图骨架 |
| 出错 | "加载失败 [重试]" |

## 验收(对应 10)
每屏四态逐一截图/断言;**重点查"出错态"真能出现**(注入失败看是否友好提示而非白屏/崩溃)——这是销项 500 坑的根治验法。
