## §1 第一等公民:回复护栏 + 文案口径(防机器人胡说)

**核心原则:模型只"听懂",不"开口"。**
- 理解层(意图分类、字段抽取)= 模型,**结构化输出**(固定枚举 + 固定字段名),产出数据、**不产出给用户看的话**。
- 回复层(用户看到的每一句)= **产品自己的 i18n 模板槽位填值**,**禁止把 LLM 自由文本直接回给用户**。机器人物理上只能"挑模板 + 填数",无法自由发挥 → 无法胡说。

**硬约束(逐条,验收要查)**:
1. 文案一律走 `services/line_binding/line_i18n.py`(4 语,泰语先);LLM 生成文本严禁直达用户。
2. LLM 输出只进 schema(意图枚举 + 字段),**永不**作 user-facing 文案。
3. **问答类**必须接已有"客户知识中心 / 问答带出处":有来源才答、附出处;**查不到 → 诚实"没找到 + 指路",绝不编造**。
4. **用户数据类问题**(本月花了多少 / 上一笔)走 **DB 真实查询**,不让模型凭记忆给数字。
5. **范围闸**:只答「本产品功能 + 用户自己的数据」;越界(闲聊 / 通用税务咨询 / 法律意见)→ 礼貌挡回 + quick reply 引导,**不硬答**。税务/合规不让模型自由出建议(责任风险)。
6. **不确定就问,不猜**:意图/字段低置信 → quick reply 澄清;破坏性动作(撤销/删除/确认入账)→ 按钮二次确认。
7. 面向用户文案**不出现"AI/人工智能"**字眼(用 智能/自动/系统);口吻对齐设计语言;**四态诚实**(失败/缺字段绝不显"成功")。

## §2 意图分层路由(代码 vs 模型,分界标清)

**原则:能用消息类型 / 正则 / 字典确定的,全走代码(免费/毫秒/确定/可测);只有真自然语言才丢给 L2 模型。**

| 层 | 判断 | 靠 | 例子(泰语优先) |
|---|---|---|---|
| **L0 消息类型** | 图片/文件 vs 文本 | **代码**(LINE message type) | 图片/PDF → 走 09 的 OCR 路;文本 → 进本文 |
| **L1 确定性规则** | 结构化文本意图 | **代码**(正则/字典) | 6 位码→绑定;`ขอ drive`/`ขอ sheet`→给链接;`含金额(+量词/分类词)`→记账;`ยกเลิก`/`แก้ไข`→撤销/改 |
| **L2 LLM 兜模糊** | 真自然语言意图 | **模型**(便宜 LLM + 固定意图枚举 + 结构化输出·仅 L1 兜不住时) | `เดือนนี้จ่ายไปเท่าไหร่`→查询;`VAT คิดยังไง`→问答(接知识中心);`เมื่อวานซื้อกาแฟ 60`→记账 |

- 意图枚举固定:`{expense, query, question, command, bind, link, unclear}`。
- L2 低置信 → **LINE Quick Reply 按钮**(`บันทึกค่าใช้จ่าย / สอบถาม / อัปโหลดบิล`)让用户一点定夺,**不猜**。
- **"上传票据"不需要判断**:消息类型就是图片(L0 代码定);"这张图是不是票"在 OCR 下游 `is_not_invoice`(09),不在路由层。

## §3 模糊 → 字段映射(记账意图内)

**确定性规则打底 + LLM 兜模糊**,不全丢 LLM。

| 用户输入(泰语优先) | 线索 | 落到字段 |
|---|---|---|
| `327849723847` 长串数字 | ≥10 位、无小数、可带字母前缀 | 13 位纯数字 → **卖家税号**;其余 → **发票号**(原样含前缀,简式不强制·见 09 §5) |
| `10` / `10 บาท` / `฿10` | 带币种或有小数 | **金额**(默认含税总额) |
| `5 ชิ้น` / `5个` / `x5` | 量词/×号 | **数量** |
| `@45` / `ต่อหน่วย 45` | 单价标记 | **单价**,自动算 total = qty × 单价 |
| `ค่าน้ำ` / `购物` / `taxi` | 关键词命中字典(泰语为主) | **分类 / 子分类**(字典 + 学习) |
| `13/06/26` / `วันนี้` | 日期格式/相对词 | **日期**(佛历 25xx−543·缺省=今天) |
| 余下认不出的文字 | 兜底 | **卖家名 / 备注** |

- **可选标签做强信号**:`เลขที่ 327... จำนวนเงิน 10 จำนวน 5` → 标签直接定字段;没标签就靠线索推。**两者都支持。**
- **金额约定(钉死)**:裸数字 = **总额**;要单价用 `@` / `ต่อหน่วย`;**确认卡上永远显示算出的总额**,用户一眼核对。
- 映射结果对齐 09 / OCR 的同一套字段 schema(ThaiInvoice / ExpenseDraft),下游 dedup/入账共用。

## §4 引擎(L1 规则 + L2 LLM·同 spec 14 哲学)

1. **L1 确定性**:正则 + 泰语字典,字段齐 → 直接出确认卡(0 成本/毫秒)。
2. **L2 结构化输出**:逼模型把模糊文本映射进**固定字段名**,别自由发挥;扛多语言/缩写/无标点。
3. **多轮澄清**:歧义(两个裸数字分不清)/ 缺必填(金额)→ 反问一句 → 用户补 → 刷新卡。state 存 `line_pending_entry`(短 TTL)。
4. **学习**:用户纠正过的 keyword→category 记住,下次直接用。

---

## §5 智能回复 = 提议确认,不静默入账(死穴)

- 任何记账 → **确认卡**(字段 + 算出的总额)→ 用户确认才落账;**绝不**凭模型猜的数静默入账。
- 问答 → 知识中心(带出处);数据问题 → DB 查询。
- 与图片路(09)共享:**同字段 schema + 同确认卡 + 同 dedup/入账下游**。

## §7 别做(防跑偏)

- ❌ 别把 LLM 生成的自由文本直接回用户(§1)——这是头号红线。
- ❌ 别全用 LLM 路由:类型/正则/字典能定的别调模型(成本+延迟+不确定)。
- ❌ 别静默入账 / 别执行破坏性动作不确认。
- ❌ 别只做中文:**泰语优先**。
- ❌ 别和 09 的字段 schema / 确认卡 / 下游分叉:必须同一套。
- ❌ 越界问题(通用税务/法律/闲聊)别硬答:挡回 + 引导。

## §8 验收

- 泰语真句子:记账/查询/问答/绑定/链接/撤销 路由正确;模糊 → 澄清而非误丢。
- 模糊字段:§3 样例映射对、总额算对、确认卡正确。
- **护栏**:用户可见文案全部来自 i18n 模板(grep 确认无 LLM 直出);问答查不到 → 诚实兜底;越界 → 挡回;无"AI"字眼。
- 与 09 共享下游 dedup/入账一致;真账号 LINE 端 E2E + prod 验。
